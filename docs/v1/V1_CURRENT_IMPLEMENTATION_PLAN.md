# V1 当前实现规划

日期：2026-06-10

## 1. 为什么要调整

执行计划已经从“做销售漏斗”调整为：

> 普通用户定位 + 趋势/需求输入 + 自营账号自然播放收益验证。

因此，实现规划也必须调整。下一阶段不能只做一个播放数据页面，也不能继续把所有逻辑堆进 `src/main.tsx`。

当前实现必须同时支撑三条能力：

1. 趋势/需求输入：记录市场热点、用户痛点、搜索问题和竞品样本。
2. 趋势融合生成：把趋势改造成原创内容，而不是搬运热点。
3. 播放变现复盘：记录发布、播放、互动、收益和趋势效果。

详细任务计划见：

`docs/superpowers/plans/2026-06-10-playback-trend-implementation-plan.md`

## 2. 当前研发顺序

### 第一步：拆基础边界

先从 `src/main.tsx` 抽出：

- `src/types.ts`
- `src/storage.ts`

原因：

- `src/main.tsx` 已接近 4000 行。
- 趋势、播放、收益、案例、生成都继续放在一个文件里会快速失控。
- 后续普通用户版本需要更清晰的模块边界。

### 第二步：趋势/需求卡片

先做手动录入，不做自动抓取。

必须支持：

- 来源平台。
- 热词或样本标题。
- 用户需求。
- 热度原因。
- 内容角度。
- 风险提示。
- 状态流转。

### 第三步：趋势融合生成

趋势卡片可以进入创作向导。

生成内容时必须保留：

- 使用了哪条趋势。
- 热度改写逻辑。
- 原创化说明。
- 风险提示。

### 第四步：播放变现页

播放变现页必须支持：

- 平台账号资格。
- 14 天 28 条发布目标。
- 30 天 60 条发布目标。
- 24 小时、48 小时、7 天数据。
- 10 元和 100 元收益目标。
- 趋势融合内容单独复盘。

### 第五步：首页和复盘导出

首页要从旧销售指标切换为：

- 趋势数量。
- 可生成趋势数量。
- 已排期内容。
- 已发布内容。
- 累计播放。
- 累计收益。
- 当前主动作。

导出能力：

- 14 天播放复盘。
- 30 天流量资产复盘。
- 趋势/需求复盘。

## 3. 当前不做

- 自动抓热点。
- 自动抓评论。
- 自动发布。
- 自动读取平台后台数据。
- 支付和 SaaS。
- 客户线索/报价/销售漏斗增强。

这些都必须等自营账号跑出内容和流量证据后再考虑。

## 4. 验收标准

- [ ] `npm run build` 通过。
- [ ] 可以录入 10 条趋势/需求卡片。
- [ ] 可以用趋势卡片生成原创内容包。
- [ ] 可以把生成项目加入播放排期。
- [ ] 可以记录 24 小时、48 小时、7 天播放和收益。
- [ ] 首页显示 14 天 10 元和 30 天 100 元目标进度。
- [ ] 能导出播放复盘和趋势复盘。

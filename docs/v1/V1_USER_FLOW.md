# V1 用户流程

日期：2026-06-06

## 1. 设计原则

V1 只服务一件事：

> 外行用户从一句话想法开始，在 30 分钟内通过系统生成一条可发布的故事类短视频素材包。

V1 主流程不得暴露以下专业概念：

- Prompt。
- token。
- temperature。
- 模型路由。
- 备用模型。
- 工作流编排。
- 多模型评测。

这些能力可以存在于管理员设置或后台，但不能出现在新手主流程里。

## 2. V1 主流程

1. 用户点击“开始创作”。
2. 用户选择创作目标。
3. 用户选择目标平台。
4. 用户选择题材。
5. 用户输入一句话想法。
6. 系统生成故事类短视频素材包。
7. 用户在结构化编辑台修改内容。
8. 用户运行发布前质检。
9. 用户按建议一键修复或手动修改。
10. 用户导出可发布内容包。
11. 用户手工发布到平台。
12. 用户回到系统记录发布数据。
13. 系统给出下一条内容建议。

## 3. 首次生成必填项

首次生成前只允许 4 个必填项：

- 创作目标。
- 目标平台。
- 题材。
- 一句话想法。

## 4. 创作目标

V1 保留 3 个入口：

1. 故事短视频。
2. 小说转短视频。
3. 故事草稿。

默认推荐：

> 故事短视频

原因：

- 产物短，容易发布。
- 能验证钩子、冲突、反转、标题和发布文案。
- 可复用到小说转短视频、口播、短剧和图文。

## 5. 目标平台

V1 默认支持中文平台选项：

- 抖音。
- 视频号。
- 小红书。
- B站。

平台选择只影响：

- 脚本语气。
- 视频时长建议。
- 标题风格。
- 封面文案风格。
- 发布文案风格。
- AI 内容标识提醒。

V1 不做自动发布。

## 6. 题材

V1 题材：

- 都市逆袭。
- 悬疑反转。
- 情感故事。
- 职场复仇。
- AI 科幻。

每个题材应提供：

- 一句话解释。
- 适合人群。
- 常见冲突。
- 示例开头。
- 风险提醒。

## 7. 生成结果

系统生成的素材包必须包含：

- 开头钩子。
- 完整脚本。
- 分镜表。
- 字幕稿。
- 标题选项。
- 选中标题。
- 封面文案选项。
- 选中封面文案。
- 发布文案。
- 发布前质检评分。
- 修改建议。

## 8. 编辑动作

用户不需要自己写 Prompt，只通过按钮修改。

V1 编辑按钮：

- 开头更吸引人。
- 改得更短。
- 改得更爽。
- 改得更悬疑。
- 改得更口语。
- 换一组标题。
- 换一个封面文案。
- 重写结尾。
- 降低 AI 味。
- 更适合当前平台。

## 9. 发布前质检

导出前必须经过发布前质检。

如果评分低于发布阈值，系统应：

- 标出问题。
- 给出修改建议。
- 提供一键修复按钮。

用户可以强制导出，但必须填写原因。

## 10. 导出

V1 默认导出：

- Markdown 内容包。
- 纯文本字幕。
- 分镜表。
- 发布文案。

高级导出：

- CSV。
- JSON。

高级导出不作为默认入口。

## 11. 手工发布和复盘

发布后用户回到系统填写：

- 平台。
- 发布时间。
- 内容链接。
- 播放。
- 点赞。
- 评论。
- 收藏。
- 涨粉。
- 备注。

V1 复盘只回答 3 个问题：

1. 哪个题材更好？
2. 哪个钩子更好？
3. 下一条应该怎么改？

## 12. 成功标准

- 新用户不看教程，5 分钟内完成首次生成。
- 用户 30 分钟内导出一条可发布素材包。
- 首次生成前必填项不超过 4 个。
- 每个页面只有一个最突出的下一步按钮。
- 用户不需要手写复杂 Prompt。

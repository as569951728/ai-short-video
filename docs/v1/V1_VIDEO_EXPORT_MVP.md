# V1 自动成片 MVP

## 1. 当前目标

V1 自动成片能力先解决一个问题：

> 不懂剪辑的人，能不能从系统里拿到一份可继续渲染、导入剪辑工具或对外展示的视频结构包。

当前不追求精美大片，也不接剪映、Canva、Runway、Creatomate 或 Shotstack API。

## 2. 当前已实现

本地 API：

- `POST /api/export/video-plan`

输入：

- 当前项目 `project`。
- 项目中的 `generated.storyboard`。
- 标题、封面文案、字幕、分镜。

输出：

- 9:16 视频计划。
- 1080x1920 分辨率建议。
- 每个镜头的开始时间、结束时间、预计时长。
- 镜头场景、旁白、画面描述、字幕。
- `subtitles.srt` 字幕文本。
- `ffmpeg-draft.sh` 渲染草稿。

前端导出页：

- 新增“自动成片 MVP”面板。
- 新增“生成成片计划”按钮。
- 展示比例、分辨率、预计时长。
- 支持下载 SRT 字幕。
- 支持下载渲染草稿。
- 支持复制字幕。
- 展示镜头计划表。

样片生成：

- 新增 Swift 原生样片脚本：`scripts/create-demo-video.swift`。
- 当前样片目录：`outputs/demo-2026-06-09-v2/`。
- 样片包含原创小说、分镜、SRT 字幕和 720x1280 MP4。

## 3. 当前边界

当前不是完整视频生成平台。

尚未实现：

- 自动生成配音音频并合入视频。
- 自动选择真实图片或 AI 图像素材。
- 自动生成写实画面。
- 自动上传发布平台。
- 剪映/Canva API 对接。
- 云端视频生成 API 对接。

这些能力都必须等真实验证需要再做。

## 4. 当前技术判断

原计划优先考虑 Remotion + FFmpeg。

实际测试发现：

- 本机 Homebrew `ffmpeg` 因 `libunistring.2.dylib` 缺失无法启动。
- 两个应用内置 `ffmpeg` 二进制也会崩溃。
- 为了不中断验证，当前新增 Swift + AVFoundation 样片生成脚本作为本机降级路线。

这不是长期架构替代，只是当前机器环境下的可运行验证路径。

长期路线仍然是：

1. 先用系统导出稳定视频计划、SRT 和分镜。
2. 再用 Remotion/FFmpeg 或其他稳定渲染器生成 MP4。
3. 有付费意向后再考虑 Creatomate、Shotstack、Runway 等付费 API。

## 5. 为什么先做视频计划

它直接服务 V1 的两条铁律：

1. 简单易用：用户即使不会剪辑，也能看到每个镜头应该放什么、字幕在什么时间出现。
2. 能赚到钱：对外演示时，可以说明系统不只生成文案，还能把内容整理到成片前的执行结构。

它不会明显拖慢 100 元验证：

- 不需要新增第三方付费工具。
- 不需要用户申请 API key。
- 不需要设计复杂模板。
- 不改变当前“先触达 5 个真实对象”的商业节奏。

## 6. 下一步候选

只有当“不会剪辑”明显卡住验证，才继续增强自动成片。

优先级：

1. 把视频计划能力接入真实项目导出流程并验证页面可用。
2. 增加更贴合小说内容的基础可视化模板。
3. 增加 TTS 配音或音频上传。
4. 用 Remotion + 可用渲染器替换 Swift 样片脚本。
5. 有付费意向后接 Creatomate 或 Shotstack。

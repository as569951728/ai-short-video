# V1 模型生成质量准入测试

日期：2026-06-07

## 1. 目的

模型连接成功不等于内容可售。

V1 允许替换大模型，但任何模型在对外演示、收费生成或沉淀案例前，必须先通过同一批生成质量准入测试。

## 2. 当前固定测试样例

### 2.1 职场逆袭

- 平台：抖音。
- 题材：职场复仇。
- 想法：程序员发现项目失败不是技术问题，而是有人故意隐藏关键数据。
- 关注点：强冲突、反击动作、结尾悬念。

### 2.2 情感故事

- 平台：视频号。
- 题材：情感故事。
- 想法：一个总是退让的人，终于在家庭聚餐上说出真实想法。
- 关注点：情绪递进、口语化、真实细节。

### 2.3 小说转短视频

- 平台：小红书。
- 题材：悬疑反转。
- 想法：女主每晚都会收到一条来自三年前自己的短信，提醒她不要相信最亲近的人。
- 关注点：反转线索、画面可执行、标题吸引力。

## 3. 通过规则

每条测试都必须满足：

- 必须由真实模型生成成功，不能使用本地 Mock 回退结果。
- 质检总分不低于 38/50。
- 分镜不少于 5 段。
- 脚本正文不少于 180 个非空白字符。
- 标题候选不少于 3 个。
- 开头钩子必须包含明确问题或悬念。
- 内容中不能出现“作为一个 AI”“抱歉”“无法满足”等模型身份或拒答痕迹。

只有 3 条全部通过，模型才允许进入真实案例生产。

## 4. 失败处理

如果准入未通过，不要对外演示，不要开始收费。

处理顺序：

1. 先确认 Base URL、API Key、模型名是否配置正确。
2. 如果是结构缺失，优先修改模型网关提示词和 JSON 校验。
3. 如果是内容质量差，先换模型或调整生成提示词，不要继续扩功能。
4. 如果 3 条测试仍无法通过，保持本地 Mock 只用于产品流程演示，不能用于真实销售案例。

## 5. 当前状态

系统设置页已经提供“3 条生成质量准入测试”入口。

当前无真实模型配置时，系统会明确显示 `准入未通过：0/3 条达标`，并提示缺少模型配置。

# 小说环节产品验收预检记录（任务包 1～7）

## 结论

需求主控组织 4 个只读评审视角对小说环节进行产品验收预检：

- 产品旅程 / 小白用户闭环。
- 状态、网关和数据追溯。
- 前端交互和可用性。
- QA 风险和验收策略。

综合结论：**可接受风险，可以进入小说环节产品验收**。

但验收口径必须限定为：

- 验小说主链路，不验任务包 8。
- 使用 mock / in-memory 口径验收产品流程。
- 不验真实视频生产、TTS、字幕、渲染、发布。
- 不把 mock full review 当成真实模型审稿质量证明。
- 不把真实 MySQL / Prisma 持久化作为本轮产品验收准入。

如果本轮验收要求连接真实 MySQL / Prisma、刷新后仍可追溯、或按准生产环境验收，则当前结论应升级为**阻塞**。

## 验收边界

本轮产品验收的终点是：

> 小说从创建和创作主链路走到全书审稿、确认完成、确认进入待视频化，并生成可被后续视频模块引用的 `VideoReadinessSnapshot`。

本轮不验：

- 创建视频项目。
- 视频生成工作台。
- 旁白、TTS、字幕、渲染。
- 视频发布、数据回填。
- 任务包 8 及之后的研发内容。

任务包 8 只能在小说环节产品验收完成、用户明确确认后再派发。

## 产品验收建议路径

1. 先声明验收边界：本轮只验小说主链路，不验任务包 8，不验真实视频生产，不验真实模型质量。
2. 从 `/novels` 验小说列表：状态、下一步、主操作仍是“详情”，列表不直接创建视频。
3. 验结构资产主链路：方向、设定、全书大纲、阶段大纲、章节目录均为候选确认后才成为正式资产。
4. 验试写链路：第 1 章候选、选择候选、第 2/3 章、试写总评、风险继续原因、策略快照。
5. 验章节工作台：章节正文、重写候选、采用候选、影响评估、失败和重试入口。
6. 验批量正文：默认分批、硬失败暂停、已完成章节保留、中高影响未关闭阻塞全书审稿。
7. 验全书审稿：生成 `ReviewReport`、`FullReviewGate`、问题卡，低分不能静默完成。
8. 验完成确认：确认后保持 `completion_confirm`，生成 `CompletionDecisionRecord` 和待视频化检查，不直接进入 `video_ready`。
9. 验待视频化确认：检查通过后生成 `VideoReadinessSnapshot`，小说进入 `video_ready`。
10. 验边界：小说侧只展示待视频化和承接入口，不创建视频，不触发 TTS / 字幕 / 渲染 / 发布。

## 现场重点坏路

- 缺正文、缺章节特性卡、影响案例未关闭时，不能发起全书审稿。
- 低分全书审稿未强制通过时，不能确认完成。
- 强制通过必须填写原因并确认风险。
- 待视频化检查失败时，不能生成视频化快照。
- `sourceVersionRefs` 过期时必须阻断确认。
- 同一待视频化确认幂等键和同一请求指纹应复用同一快照。
- 同一幂等键但不同请求指纹应返回 `IDEMPOTENCY_CONFLICT`。
- 视频、TTS、字幕、渲染、发布后端能力在本轮边界内应保持不可用或不触发。
- 页面、响应、控制台和普通日志不应暴露 API key、token、DB URL、完整 prompt 或完整模型响应。

## 主控建议

可以进入小说环节产品验收，但建议分两层：

1. **产品流程验收**：立即开始，按 mock / in-memory 口径验证用户是否能走完整小说闭环。
2. **准生产技术验收**：后置，必须补齐真实 MySQL / Prisma 写路径、幂等指纹、状态摘要事实源和安全自动化抽样后再做。

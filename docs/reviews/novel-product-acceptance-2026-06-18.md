本次验收只覆盖小说环节产品闭环：

```text
小说列表 -> 创建小说 -> 方向 -> 设定 -> 大纲 -> 章节目录 -> 试写 -> 章节详情 -> 批量正文 -> 全书审稿 -> 完成确认 -> 待视频化确认
```

不验收：

- 任务包 8。
- 创建视频项目。
- 视频引用异常完整闭环。
- TTS、字幕、渲染、发布。
- 真实 MySQL / Prisma 准生产写路径。
- 真实模型审稿质量。

验收口径：

- 使用 mock / in-memory 产品流程口径。
- 重点判断小白用户是否能看懂当前状态、下一步动作和失败恢复路径。
- 若发现阻塞项，则先记录并回研发修复；若为可接受风险，则进入下一段验收。

## 第 4 段：批量正文、章节重写、影响评估

### 验收目标

- 正文策略快照生成后，用户可以按批次生成正文。
- 批量正文遵守幂等键和批次范围，不重复覆盖已有正文。
- 每章生成正文后保留正文版本、特性卡、单章审稿和长篇记忆。
- 章节重写先生成候选，不直接覆盖正式正文。
- 采用重写候选后生成影响评估；中等或严重影响未处理前阻止继续批量正文和全书审稿。
- 即使当前阶段被阻塞，用户仍能进入详情页查看原因和处理入口。

### 验收结果

结论：通过。

已通过的部分：

- `POST /novels/novel_000001/chapters/batch-generate` 已按策略快照和幂等键执行第一批正文生成。
- 第 4 章执行 `rewrite` 成功，生成候选正文 `ccv_000187`，候选状态为 `candidate`，未直接覆盖正式正文。
- 采用重写候选后生成影响案例 `impact_000193`，影响等级为 `medium`，状态为 `waiting_decision`，`blocksFullReview=true`。
- 在影响案例未关闭时再次请求批量正文，写侧正确返回 `GATE_BLOCKED`，没有放松生成门禁。

阻塞问题：

- 重写采用并产生 open impact case 后，小说列表仍能加载，行状态为“当前阶段被阻塞”，推荐动作是“进入详情”。
- 但 `GET /novels/novel_000001` 返回 HTTP 409，导致用户无法进入详情页处理阻塞。
- 错误信息为：`本批范围已有正式正文，不能通过批量入口重复覆盖；如需调整请进入章节详情重写`。
- 这会形成产品闭环断点：系统要求用户进详情处理影响案例，但详情页本身打不开。

主控初步定位：

- 详情读侧在组装 `bodyGeneration` 时复用了批量正文写侧范围计算：`buildBodyGenerationState -> calculateBodyBatchRange(chapters,{})`。
- 重写采用后第 4 章状态不再按 `completed` 计入下一批范围，导致读侧把下一批起点算到第 4 章，范围覆盖第 4-8 章。
- 第 5 章已有正式正文，写侧范围函数抛出 `GATE_BLOCKED`，进而污染详情读接口。

已采取动作：

- 已向全栈研发会话派发《小说环节产品验收阻塞修复包》。
- 修复边界：只修详情读侧 409 和对应测试；写侧批量正文、影响案例阻断、重复覆盖门禁不能放松；不进入任务包 8。
- 研发会话已完成修复并汇报：详情读侧改用展示用批次范围计算，不再触发写侧重复覆盖门禁；写侧 `POST /chapters/batch-generate` 仍保持 open impact case 未关闭时 409。
- 已向测试会话派发《小说环节产品验收阻塞修复包正式复验指令》。

复验结果：

- 产生 open impact case 后，`GET /novels/novel_000001` 返回 200，不再返回 409。

### 验收状态

执行前置：

- 第 4 段影响案例读侧阻塞已完成研发修复和测试复验，结论为通过。
- 影响案例处理或关闭后，正文阶段可以进入“待全书审稿”。

### 验收目标

- 全书审稿只能在正文完整、章节特性卡完整、单章审稿可追溯、无未关闭中高影响案例时发起。
- 全书审稿作为独立异步任务执行，生成全书审稿报告和完成门禁，不直接确认小说完成。
- 全书审稿报告包含总分、评级、主要问题、优势、建议、视频化建议、首条视频建议和来源版本快照。
- 用户确认小说完成是独立动作，必须基于有效全书审稿报告；低分或警告风险需要提示并记录原因。
- 完成确认后只进入 `completion_confirm` / 待视频化检查阶段，不能直接进入 `video_ready`。
- 待视频化检查必须展示检查清单；检查失败时保留小说完成状态但不能进入待视频化。
- 确认待视频化是独立动作，成功后生成 `VideoReadinessSnapshot`，小说进入 `creationStage=video_ready`。
- `video_ready` 后推荐动作是“去视频列表”，不是“创建视频”。
- 相同幂等键重复确认待视频化应复用已有快照；不同请求指纹应返回幂等冲突。

- 缺正文、缺章节特性卡、缺单章审稿、未关闭中高影响案例时仍能发起全书审稿。

- 全书审稿未完成、已过期或基于旧版本时仍能确认完成。

日期：2026-07-12

状态：五类专业复盘已完成，停止新增业务研发，等待路线决策

复盘范围：小说系统、视频系统、模型调用、异步任务、数据持久化、前后端交互、测试验收、Git 与多会话协作。

## 1. 复盘目的

本次复盘不回答“做了哪些模块”，而回答以下五个具体问题：

1. 用户要解决的真实问题是否已经解决。
2. 哪些能力只在合同、单测、内存或 mock 环境成立，哪些已经在真实环境成立。
3. 用户人工验收为什么反复发现自动验收未发现的问题。
4. 哪些返工来自单点 bug，哪些返工来自需求、架构、测试和主控机制缺失。
5. 下一阶段必须增加哪些门禁，才能避免继续用代码量替代可用结果。

本次复盘期间不启动 `P10-R1`、真实 MySQL、真实模型、真实 TTS、字幕、渲染、上传或发布研发。

所有“完成”必须附带证据等级，禁止跨级表述。

### 4.3 小说模块结论

小说系统已经有完整产品骨架、候选版本模型和大量状态处理，但“完整流程存在”不等于“可以稳定生成一本小说”。当前最关键的开放能力是：

1. 真实模型长输出必须支持预算、分片、checkpoint、JSON repair 和片段重试。
2. 后端任务必须成为刷新、多标签页、主卡片、任务抽屉和最近任务的唯一真相源。
3. 所有候选资产统一支持查看、编辑、优化指令、采用、历史版本和唯一当前版本。
4. 必须从空项目开始跑到小说完成，而不是按页面和接口分别验收。

| in-memory 代替数据库 | 唯一当前版本、幂等、事务和刷新恢复未被证明 |
| 对任务最终状态做断言 | 中间进度误导、取消/重试不一致 |
| 开发自测后直接报完成 | 独立测试来不及验证用户原始反馈 |
| 验收报告写边界但结论只写“通过” | 用户只收到强结论，忽略附录中的 mock 限制 |

### 6.2 测试机制问题

1. 验收单元以“研发包”为中心，而不是以“用户任务”为中心。
2. 缺少固定小说 fixture、固定模型响应集和一键恢复环境。
3. 缺少浏览器动作结果断言，尤其是滚动定位、弹窗、状态恢复和下一步导航。
4. 缺少真实 provider 的最小金丝雀；模型集成问题直到用户操作才暴露。
5. 缺少真实 MySQL smoke；Prisma static 结果被频繁当作持久化完成。
6. 缺少失败注入矩阵：超时、截断、非法 JSON、部分成功、保存失败、刷新和重复点击。
7. 验收结论没有固定输出 `evidence_level`、`not_proven` 和 `user_goal_status`。

## 9. 根因树

### RC-01 完成定义错位

- 表层：按钮、接口、候选和状态已实现。
- 第二层：研发包按技术层关闭。
- 根层：没有把“用户能稳定完成一本小说/生成一个真实视频”作为唯一价值流完成定义。

### RC-02 mock 与真实能力之间没有强制证据标签

- 表层：fake JSON、内存仓储和占位路径通过。
- 第二层：测试覆盖了预期路径，但没有真实延迟、脏输出和持久化。
- 根层：证据等级只在文档附录出现，没有进入页面、状态台账和验收结论。

### RC-03 AI 动作没有统一产品合同

- 表层：融合、优化、生成、查看和采用行为不一致。
- 第二层：每页独立实现按钮与 loading。
- 根层：需求没有固定定义输入、后台任务、结果位置、版本、编辑、采用、下一步、取消和失败恢复。

### RC-04 长任务架构按同步请求思维设计

- 表层：超时、非法 JSON、卡住、刷新丢状态。
- 第二层：一次生成太多内容，结果最后才持久化。
- 根层：缺 token 预算、分片、checkpoint、heartbeat、repair 和片段级重试。

### RC-05 测试围绕模块，不围绕用户旅程

- 表层：大量单测和 API 测试通过，浏览器按钮仍无效果。
- 第二层：验证数据存在而不是用户可继续。
- 根层：没有固定 fixture 和从空状态到最终成品的金丝雀流程。

### RC-06 主控台账与代码事实脱节

- 表层：旧巡检文案、迟滞派发、超大检查点。
- 第二层：状态靠会话描述维护。
- 根层：缺单一项目台账、自动证据收集和包级交付纪律。

## 11. 下一阶段硬门禁

任何新研发包必须同时满足：

1. 需求由产品、架构、测试至少三个独立角色评审。
2. 授权单写明 in-scope、out-of-scope、真实环境、成本、安全与回滚。
3. UI 上首先标明能力是 mock、占位还是真实。
4. 研发交付必须 commit 并 push，工作树归因清楚。
5. 研发自测不能替代独立测试。
6. 测试必须包含用户原始问题的浏览器动作结果断言。
7. 长任务必须覆盖刷新、多标签页、重复点击、取消、超时、脏 JSON 和保存失败。
8. 数据库能力必须有 E6；模型/媒体能力必须有 E5；没有则只能按 E3/E4 表述。
9. 验收结论首句必须同时写 `user_goal_status`、`evidence_level` 和 `not_proven`。
10. 主控必须给出唯一推荐动作，未授权不得预执行。

### 14.2 独立产品结论

1. 6 月 18 日验收记录本身已承认多处没有浏览器可视化点击验收，因此后来集中暴露按钮无效果并非偶然。
2. 接口成功不等于用户完成任务；结果定位、差异说明、当前版本切换和下一步导航都属于动作合同。
3. “取消本页等待”与“取消后端任务”必须拆开命名和实现，不能继续混用。
4. 真实模型的结构化输出不是底层细节，而是方向、设定、大纲、目录、试写和正文共同依赖的核心产品能力。
5. 视频当前准确状态是“产品结构走通、生产能力未走通”。

### 14.3 产品侧新增验收标准

- 无真实进度事件时，只能显示不确定进度条和当前阶段，禁止虚构百分比。
- AI 结果接口必须返回并使用 result id；页面自动滚动、高亮并展示来源与差异。
- 所有编辑必须生成新版本、保留原版本，并在刷新后保持。
- 所有优化必须保存 baseVersionId、instruction 和新版本，并展示目标达成说明。
- 第二标签页必须在 300ms 内获取并显示同一服务端 taskId；冲突参数必须阻断。
- 80 章目录需连续成功至少 3 次；单块失败只能重试该块，已完成块不得重算。
- 每种 taskType 的“查看结果”必须断言 URL、步骤、候选 ID、滚动位置和高亮状态。
- 真实视频验收必须同时证明文件存在、MIME/时长正确、可播放、可下载和服务重启后仍可取回。

| N-AI-08 | `previousMemory` 传入 provider，但没有进入模型 payload | 连续性记忆未生效 | 下一章请求包含更新后的长期记忆，并有冲突/连续性测试 |
| N-AI-09 | 真实产物来源仍可能标记为 `mock_ai`，current 唯一性缺数据库强约束 | 来源审计与并发唯一性部分关闭 | 记录 provider/model/promptVersion/token，真实并发采用只保留一个 current |

模型超时、限流、空响应和基础 JSON 错误的安全分类已有 19 项聚焦测试通过；该结论只属于 E2，不代表真实长文本稳定。

### 16.3 P9-real 金丝雀门禁

- 明确 provider、成本上限、密钥边界和失败降级。
- 从空 MySQL 执行 P8-P9 migration，并验证真实 Prisma 事务、并发和唯一 current。
- TTS 输出真实音频，验证 MIME、时长与浏览器播放。
- 字幕基于真实音频生成带时间戳 SRT/VTT，并抽样验证同步。
- 渲染输出真实 MP4；下载必须返回 200、`video/mp4`、非零文件、正确时长并可播放。
- 任务必须支持进程重启、刷新、多标签、取消、重试和旧版本回写阻断。

## 19. 主控最终裁决

### 19.1 当前准确完成度

- 小说：mock/in-memory 主流程可演示；多个已知交互故障已定向修复；没有可重复的全链浏览器 E2E；真实 DeepSeek + Prisma 完本链不可验收；全书审稿当前不可信。
- 视频：产品结构、状态流和版本工作台已形成；没有真实可播放音频、真实字幕时间线、真实 MP4 或真实下载文件。
- 测试：低层回归健康，但浏览器、真实 DB/provider/media 和内容质量证据缺失。
- 治理：授权与安全阻断有效；提交、证据分级、状态同步和派发 SLA 仍需制度化。

### 19.2 优先级

1. P0：补全小说 Prisma 后半程并建立真实 MySQL 完整 E2E。
2. P0：修正全书审稿输入，使其真正基于正文与连续性记忆。
3. P0：统一服务端持久任务、worker、checkpoint、真实重试与并发防重。
4. P0：纳管浏览器 E2E 与失败 fixture。
5. P1：真实 DeepSeek 质量回归、字数、记忆、JSON 与章节连续性门禁。
6. P1：P9-real 金丝雀，至少产出一个可播放音频与一个可下载 MP4。
7. P1：包级提交、状态台账与派发 SLA 自动门禁。
8. 暂缓：P10-R1，直到用户在“小说真实完本”和“P9-real”之间确认优先路线。

### 19.3 推荐路线

按照项目原始战略“小说质量优先”，主控建议下一阶段先做“小说 real-flow 修复包”，关闭两个 P0 并形成真实 DeepSeek + MySQL 的小规模完本金丝雀；随后做 P9-real 单视频金丝雀；P10-R1 排在真实视频文件之后。

该推荐只是一项复盘结论，不是研发授权。

# V1 工具与软件服务使用阶段

## 1. 选型原则

工具只解决当前瓶颈，不解决想象中的规模化问题。

当前 V1 的瓶颈不是工具数量，而是：

1. 生成内容是否能做成可展示样片。
2. 是否能触达到真实潜在用户。
3. 是否能收到明确反馈、报价机会或 100 元收入信号。

因此，任何新工具必须满足至少一个条件：

- 让外行更容易完成内容生产。
- 让我们更快完成 5 个公开/半公开真实触达。
- 让我们更快拿到 1 次明确报价或 100 元收入信号。

如果工具只是让系统更酷、更自动化、更像完整 SaaS，但不能推进上述目标，V1 暂缓。

## 2. 阶段总览

| 阶段 | 目标 | 使用工具 | 调用方式 | 是否当前需要 |
| --- | --- | --- | --- | --- |
| 阶段 0：系统生成 | 生成脚本、分镜、标题、发布文案 | DeepSeek、AIShortvideo 本地系统 | 系统 API 自动调用 | 已需要，已接入 |
| 阶段 1：首批样片 | 把演示案例做成可发布短视频 | 剪映/CapCut、Canva | 用户人工使用，我提供内容包和操作清单 | 当前需要 |
| 阶段 2：公开触达 | 联系 5 个真实触点 | 小红书、抖音、B站、视频号、社群 | 用户本人账号人工触达，我提供话术和筛选标准 | 当前需要 |
| 阶段 3：需求收集 | 收集对方需求和反馈 | Tally、飞书表单、腾讯问卷 | 可先人工建表单，后续可 webhook/API | 有反馈后需要 |
| 阶段 4：线索管理 | 跟进报价、回复、成交状态 | 系统收入页、飞书表格、Airtable | 先用系统内置；数据变多后再接表格/API | 暂不扩展 |
| 阶段 5：发布排期 | 管理多平台内容排期和基础分析 | Buffer、Metricool | 后续可 API；当前人工发布更稳 | 暂缓 |
| 阶段 6：视频生成增强 | 生成更强视觉素材 | Runway、可灵、Pika | 后续可 API 或人工生成 | 暂缓 |
| 阶段 7：自动化运营 | 表单、CRM、发布、通知自动流转 | Zapier、Make、n8n | 后续自动化 | 暂缓 |

## 3. 当前 7 天内的推荐组合

### 3.1 DeepSeek

用途：

- 故事脚本生成。
- 分镜生成。
- 标题、封面文案、发布文案生成。
- 质检辅助。

调用方式：

- 系统服务端通过 OpenAI-compatible API 自动调用。
- 用户不需要直接操作 DeepSeek 控制台。

当前动作：

- 继续使用 `deepseek-v4-flash` 作为 V1 默认模型。
- 只有当 3 条质量准入失败，才重新测试其他模型。

### 3.2 剪映/CapCut

用途：

- 导入脚本和分镜。
- 生成字幕。
- 配音或文字朗读。
- 套用基础模板。
- 导出第一批样片。

调用方式：

- 当前按人工工具处理。
- 我可以给出逐步操作清单、素材清单、字幕文本、镜头说明。
- 用户需要自己登录账号、选择素材、检查画面、导出视频。

为什么不现在做 API：

- 当前目标是 2 周内赚到 100 元，不是做全自动剪辑平台。
- 成片质量需要人工审美检查。
- 发布前需要确认没有不合适素材、版权风险或明显错字。

当前动作：

- 免费版先做 1 到 3 条样片。
- 如果遇到水印、导出限制、字幕样式限制，再评估是否付费。

### 3.3 Canva

用途：

- 封面图。
- 标题图。
- 简单贴片素材。
- 统一样片视觉风格。

调用方式：

- 当前按人工工具处理。
- 我可以生成封面文案、封面布局建议和配色建议。
- 用户需要自己在 Canva 中确认模板、导出图片。

API 判断：

- Canva 有 Connect APIs，可用于把 Canva 能力接入 Web 应用，支持资产、设计、导出等能力。
- 但当前不接 Canva API，因为 V1 不需要把封面制作自动化。

当前动作：

- 免费版做封面即可。
- 不为了首批验证购买 Canva Pro。

### 3.4 小红书/抖音/B站/视频号/社群

用途：

- 找公开/半公开真实触点。
- 发布或私信演示案例。
- 收集评论、回复、意向。

调用方式：

- 用户本人账号人工操作。
- 我可以提供筛选标准、评论回复、私信话术、报价话术。

为什么必须用户操作：

- 平台账号和手机号、实名、社交关系有关。
- 自动化私信或批量抓取容易违规。
- 早期验证需要真实上下文判断，不能机械化群发。

当前动作：

- 用项目专用手机号/账号。
- 关闭通讯录同步。
- 从公开评论区和公开主页筛选 5 个触点。

## 4. 有反馈后再引入的工具

### 4.1 Tally / 飞书表单 / 腾讯问卷

触发条件：

- 已经有 3 到 5 个对象愿意看演示或表达需求。

用途：

- 收集对方账号类型、内容方向、希望生成的故事主题、预算意向。

调用方式：

- 第一版人工创建表单。
- 后续可以用 Tally webhook 把提交同步到系统。

API 判断：

- Tally 支持 webhook，表单提交后可以把 JSON 数据推送到我们的接口。
- 当前暂不接，等至少有 5 条真实需求后再接。

### 4.2 Airtable / 飞书多维表格

触发条件：

- 系统收入页线索管理不够用。
- 线索超过 20 条，或需要多人协作。

用途：

- CRM。
- 报价记录。
- 跟进状态。
- 复盘转化率。

调用方式：

- 初期人工维护。
- 后续可以用 API 或飞书能力同步。

当前动作：

- 暂时继续使用系统收入页，不迁移。

### 4.3 Buffer / Metricool

触发条件：

- 已发布 5 到 10 条内容。
- 需要排期、跨平台复盘、基础分析。

用途：

- 内容排期。
- 多平台账号管理。
- 发布记录。
- 数据复盘。

API 判断：

- Buffer 有开发者 API，可用于内容排期和自动化。
- 当前不接，因为第一阶段人工发布更可控。

## 5. 暂缓工具

### 5.1 Runway / 可灵 / Pika

触发条件：

- 已证明脚本方向能吸引真实用户。
- 用户明确反馈视觉质量是成交障碍。
- 已经有收入或付费试点预算。

用途：

- 生成更强视频素材。
- 做更接近正式短剧/故事视频的视觉包装。

API 判断：

- Runway 有官方 API，可把生成模型接入应用。
- 但视频生成成本和调试成本更高，当前暂缓。

### 5.2 Zapier / Make / n8n

触发条件：

- 表单、线索、报价、发布记录已经形成重复流程。

用途：

- 自动同步表单到 CRM。
- 自动提醒跟进。
- 自动生成周报。

当前动作：

- 暂缓。没有真实流程前，不做自动化。

### 5.3 自动发布和批量私信工具

触发条件：

- V1 不建议引入。

原因：

- 违规风险高。
- 容易被平台限流或封号。
- 会掩盖真实产品价值判断。

## 6. 用户和系统的分工

系统 / 我可以做：

- 生成内容包。
- 生成剪映操作清单。
- 生成封面文案。
- 生成发布文案。
- 生成触达话术。
- 生成报价话术。
- 记录客户反馈。
- 后续接入 DeepSeek、Tally webhook、Buffer API、Runway API 等。

用户必须做：

- 注册和维护项目专用手机号、平台账号。
- 登录剪映、Canva、发布平台。
- 人工检查成片质量。
- 人工发布内容。
- 人工触达真实对象。
- 收款和确认收入。
- 判断是否愿意为某个工具付费。

## 7. 当前执行结论

当前不新增 API 对接。

当前只做三件事：

1. 系统继续生成高质量内容包。
2. 用户用剪映/Canva 免费版把首个演示案例做成样片。
3. 用样片和演示案例去触达 5 个公开/半公开真实对象。

只有出现以下信号，才进入下一轮工具接入：

- 有人愿意填写需求表。
- 有人愿意付 29 元生成服务。
- 有人愿意付 99 元诊断。
- 有人愿意付 100 元以上试点定金。
- 多人反馈同一个工具缺口阻碍成交。

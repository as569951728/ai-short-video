# V1 方向一致性复盘

日期：2026-06-07

## 1. 当前判断

当前所做事情和项目大方向相符。

原因：

- 项目大方向是做 AI 小说/短视频内容生产系统，并最终能通过内容发布、系统生成服务、系统诊断或试点赚到钱。
- 当前 V1 没有急着做完整大系统，而是收缩到故事类短视频脚本生产主线。
- 当前已完成从 Mock 闭环到真实模型准入的关键过渡，`deepseek-v4-flash` 已通过 3 条模型质量准入测试。
- 当前系统已经具备从一句话想法到内容包、质检、导出、案例库、收入验证、客户跟进的最小商业闭环工具。

当前所做事情和阶段目标也相符。

阶段目标不是立刻做完整 SaaS，而是：

1. 证明外行能更简单地产出可发布内容。
2. 证明真实模型能稳定生成可用内容。
3. 用系统产出 1 条可展示案例。
4. 用案例完成真实沟通和报价。
5. 在 14 天内拿到 100 元收入或等价付费试点。

其中第 1、2 步已经基本完成，第 3、4、5 步仍未完成。

## 2. 当前没有跑偏的证据

### 2.1 V1 范围被收住了

当前没有优先做自动发布、自动剪辑、模板市场、SaaS、多租户、支付、完整小说平台。

这些仍放在后置项里，避免拖慢 100 元验证。

### 2.2 技术实现服务于验证

当前技术动作不是为了炫技，而是为了降低真实验证风险：

- 模型网关支持 OpenAI-compatible Provider。
- 服务端加入 JSON mode / `response_format` 和 `max_tokens`。
- 增加 ModelRun 本地记录。
- 设置页增加 3 条模型质量准入测试。
- DeepSeek 真实模型已经通过准入。

这些动作直接服务于“稳定产出可演示内容”。

### 2.3 商业验证工具已经前置

系统不是只做内容生成，还做了：

- 20 个触点清单。
- 今日联系清单。
- 可发送话术。
- 演示材料摘要。
- 客户回复、异议类型、下次跟进。
- 0 元演示、29 元生成服务、99 元诊断、100 元以上试点定金。

这些动作直接服务于“赚到 100 元”。

## 3. 当前最重要的三个成功关键动作

### 3.1 用真实模型生成 3 条候选内容，并人工挑出 1 条演示案例

这是当前最关键动作。

原因：

- 没有演示案例，就无法让潜在客户理解系统交付价值。
- 没有人工确认，就容易拿低质量内容对外演示，伤害信任。
- 这个案例是后续私聊、报价、诊断、试点的核心销售材料。

执行标准：

- 使用 `deepseek-v4-flash` 生成 3 条候选。
- 人工挑 1 条最像真实可发布内容的。
- 人工修改标题、钩子、脚本和分镜。
- 跑发布前质检。
- 在案例库设为对外演示案例。
- 收入页演示材料摘要能引用该案例。

### 3.2 用演示案例联系 5 个真实对象，并记录真实回复

这是从研发进入商业验证的关键动作。

原因：

- 项目能不能赚钱，不由功能数量决定，而由真实客户是否愿意看、愿意试、愿意付费决定。
- 5 个真实对象可以快速暴露案例质量、话术、目标客户选择是否有问题。
- 如果没有真实沟通，继续研发会越来越像闭门造车。

执行标准：

- 从收入页生成今日 5 个联系对象。
- 用系统话术联系真实对象。
- 每个对象记录状态、客户回复、异议、下次跟进。
- 至少发出 1 个明确报价：29 元、99 元或 100 元试点定金。

### 3.3 根据真实反馈只修最影响成交的点

这是避免跑偏的关键动作。

原因：

- 真实客户反馈前，不知道问题在内容质量、使用体验、话术、报价还是目标客户选择。
- 一旦有反馈，研发应该服务于成交障碍，而不是继续按想象扩功能。

执行标准：

- 如果客户说内容不够像人写，优先改提示词、质检和人工编辑流程。
- 如果客户说不知道怎么用，优先改首页看板、向导和导出说明。
- 如果客户说愿意试但不想付费，优先改报价、交付边界和样例。
- 如果客户没有回复，优先改触点选择和开场话术。

## 4. 最可能导致失败的三个原因

### 4.1 继续做功能，但没有真实客户沟通

失败表现：

- 自动剪辑、自动发布、模板库、SaaS 页面越做越多。
- 收入页没有真实回复。
- 访谈记录还是 0。
- 没有人看过演示案例，也没有人听过报价。

为什么危险：

- 这会把项目从商业验证变成技术练习。
- 系统可能越做越复杂，但没有证明任何人愿意付钱。

纠偏动作：

- 任何新增功能前，先问是否已经完成 5 个真实沟通。
- 如果没有，停止扩功能，先联系客户。

### 4.2 演示案例质量不够，导致客户第一眼不信

失败表现：

- 内容像 AI 套话。
- 钩子不强，冲突不清楚。
- 分镜不可执行。
- 客户看完不知道能拿去做什么。

为什么危险：

- 这个项目卖的不是模型，而是“减少内容生产难度”的结果。
- 第一条演示案例如果不可信，后续报价很难成立。

纠偏动作：

- 演示案例必须人工确认。
- 不拿测试内容对外演示。
- 每条演示内容必须能在 3 分钟讲清楚交付价值。

### 4.3 把目标客户选错，导致验证信号失真

失败表现：

- 联系完全不做内容的人。
- 联系只想免费试工具、不愿付费的人。
- 联系没有短视频/小说/图文生产痛点的人。
- 客户反馈变成泛泛夸奖或礼貌拒绝。

为什么危险：

- 错误客户会让我们误判产品无价值，或者误判功能方向。
- 100 元验证需要的是有内容生产痛点的人，不是泛泛 AI 爱好者。

纠偏动作：

- 优先找正在做内容但更新困难的人。
- 优先找想做副业账号但不知道发什么的人。
- 优先找小说、故事、情感、职场、AI 工具相关内容创作者。
- 每次复盘都检查客户是否真的有内容生产痛点。

## 5. 下一阶段硬约束

下一阶段不以新增功能数为进度。

只看以下结果：

1. 真实模型已通过准入：已完成。
2. 生成 3 条候选内容：未完成。
3. 选出 1 条演示案例：未完成。
4. 联系 5 个真实对象：未完成。
5. 至少发出 1 个报价：未完成。
6. 拿到 100 元收入或明确付费试点：未完成。

如果这些结果没有推进，继续开发新模块就是跑偏。

## 6. 后续复盘问题

每次复盘都必须回答：

1. 当前动作是否让外行更容易产出内容？
2. 当前动作是否让我们更接近 100 元收入？
3. 当前动作是否来自真实客户反馈？
4. 当前是否在做阶段 2、3、4 的事情？
5. 如果今天只能做一件事，是生成演示案例、联系客户，还是修成交障碍？

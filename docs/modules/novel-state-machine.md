# 小说完整状态机

本文档补齐 `GAP-P0-001 小说完整状态机`。

状态机设计不能只列枚举。每一种状态都必须同时说明：

- 通过什么业务事件进入。
- 允许通过什么业务事件离开。
- 流转前需要满足什么条件。
- 流转时会产生什么副作用。
- 是否需要用户确认、填写原因或写操作日志。
- 失败后回到哪里，下一步推荐动作是什么。

## 核心结论

小说系统需要多维判断，但核心保存状态不宜过多。

建议真正作为小说主表核心状态维护的只有 3 个：

| 字段 | 作用 | 是否核心保存 |
| --- | --- | --- |
| `lifecycleStatus` | 项目是否仍在创作池里 | 是 |
| `creationStage` | 小说创作推进到哪一步 | 是 |
| `stageStatus` | 当前阶段正在发生什么 | 是 |

其余状态优先通过数据推导：

| 状态 | 来源 | 是否核心保存 |
| --- | --- | --- |
| 质量门禁 | 最新审稿报告、用户强制继续记录 | 否 |
| 章节汇总 | 章节表、章节审稿、章节影响记录 | 否 |
| 视频准备 | 小说完成确认、全书审稿、视频引用记录 | 否 |
| 页面展示状态 | 核心状态 + 派生状态 + 当前任务 | 否 |
| 推荐动作 | 状态规则统一推导 | 否，接口可返回快照 |

可以保存 `lastTaskId`、`lastReviewReportId`、`lastConfirmedAt` 等引用字段，但这些不是状态本身。

试写、章节重写影响、全书审稿和待视频化属于主链路门禁，完整规则分别见：

- `docs/modules/novel-trial-writing-loop.md`
- `docs/modules/chapter-rewrite-impact-handling.md`
- `docs/modules/novel-full-review-gate.md`
- `docs/modules/novel-video-readiness.md`

## 状态流转设计格式

后续所有状态流转都按统一格式设计：

| 字段 | 说明 |
| --- | --- |
| 业务事件 | 用户或系统触发的动作，例如生成设定、确认大纲、归档小说 |
| 来源状态 | 当前允许触发事件的状态 |
| 目标状态 | 事件成功后进入的状态 |
| 前置条件 | 触发前必须满足的条件 |
| 副作用 | 需要创建任务、保存版本、生成候选结果、更新摘要、标记影响等 |
| 用户确认 | 是否需要二次确认或填写原因 |
| 日志记录 | 是否写操作日志、任务日志、版本记录或审稿记录 |
| 失败处理 | 失败后进入什么状态，用户下一步能做什么 |

禁止通过通用编辑接口随意修改核心状态。核心状态只能通过明确业务事件流转。

## 核心状态一：项目生命周期 `lifecycleStatus`

项目生命周期表达小说是否还在当前创作池里。

| 状态 | 含义 | 是否可继续创作 |
| --- | --- | --- |
| `active` | 进行中 | 是 |
| `paused` | 已暂停 | 否 |
| `archived` | 已归档 | 否 |
| `deleted` | 已删除 | 否 |

### 生命周期流转

| 业务事件 | 来源状态 | 目标状态 | 前置条件 | 副作用 | 用户确认 |
| --- | --- | --- | --- | --- | --- |
| 暂停小说 | `active` | `paused` | 小说未删除 | 保存暂停原因；保留当前创作阶段和任务上下文 | 需要原因 |
| 恢复小说 | `paused` | `active` | 暂停项目存在；用户确认恢复 | 恢复暂停前推荐动作；检查热点、设定、大纲是否过期 | 需要确认 |
| 归档小说 | `active` / `paused` | `archived` | 小说未删除 | 保存归档原因；默认移出进行中列表；保留复盘数据 | 需要原因 |
| 恢复归档 | `archived` | `active` | 用户确认恢复 | 恢复归档前创作阶段；重新计算推荐动作和阻塞原因 | 需要确认 |
| 删除小说 | `active` / `paused` / `archived` | `deleted` | 用户二次确认；已视频引用需强提醒 | 软删除或彻底删除；写删除原因和影响范围 | 强确认 |

场景规则：

- 暂停和归档不改变 `creationStage` 和 `stageStatus`，只是暂停主流程展示和操作。
- 如果有正在执行的 AI 任务，暂停时需要让用户选择“等待任务完成后暂停”或“取消任务并暂停”。
- 已被视频引用、已发布或已有复盘价值的小说，删除前优先建议归档。

## 核心状态二：创作阶段 `creationStage`

创作阶段表达小说主链路推进到哪一步。

| 状态 | 小白文案 | 阶段目标 |
| --- | --- | --- |
| `draft` | 草稿 | 创建项目，保存热点来源和基础偏好 |
| `direction` | 选小说方向 | 生成、比较、融合或确认小说方向 |
| `setting` | 小说设定 | 生成并确认人物、世界观、爽点、雷区 |
| `outline` | 大纲结构 | 生成并确认全书主线和阶段结构 |
| `chapter_plan` | 章节目录 | 生成并确认每章目标、冲突、爽点和钩子 |
| `trial` | 试写调试 | 试写前 1-3 章并完成审稿调试 |
| `body` | 正文生成 | 批量生成正文并处理章节问题 |
| `full_review` | 全书审稿 | 全书评分，判断是否适合视频化 |
| `completion_confirm` | 完成确认 | 用户确认接受审稿结果 |
| `video_ready` | 待视频化 | 小说进入视频系统承接 |

### 创作阶段流转

| 业务事件 | 来源阶段 | 目标阶段 | 前置条件 | 副作用 | 失败处理 |
| --- | --- | --- | --- | --- | --- |
| 创建草稿项目 | 无 | `draft` | 有热点来源或默认推荐配置 | 保存用户偏好和生成参数 | 创建失败则停留在创建页 |
| 生成小说方向 | `draft` | `direction` | 项目未暂停/归档；无冲突任务 | 创建方向生成任务 | 方向生成失败，阶段保持 `direction`，`stageStatus=failed` |
| 确认方向 | `direction` | `setting` | 已有至少一个方向；低分方向需记录原因 | 保存当前方向版本；下一步推荐生成设定 | 确认失败则保持 `direction` |
| 确认设定 | `setting` | `outline` | 设定已生成；必要字段完整；风险已确认 | 锁定当前设定版本；生成可用于大纲的设定摘要 | 未确认则保持 `setting` |
| 确认全书大纲 | `outline` | `outline` | 全书大纲已生成；长篇结构可支撑目标章节数 | 锁定全书大纲版本；推荐生成阶段大纲 | 未确认则保持 `outline` |
| 确认阶段大纲 | `outline` | `chapter_plan` | 阶段大纲已生成；阶段数量、章节范围和阶段衔接合理 | 锁定阶段大纲版本；记录阶段数量；允许生成章节目录 | 未确认则保持 `outline` |
| 确认章节目录 | `chapter_plan` | `trial` | 章节目录完整；每章有目标、冲突、爽点和钩子 | 锁定章节计划；准备试写上下文 | 未确认则保持 `chapter_plan` |
| 确认试写通过 | `trial` | `body` | 前 1-3 章已生成正文、摘要和审稿；试写总评允许通过或用户确认风险 | 保存试写审稿结论；复制试写经验到正文策略 | 未通过则保持 `trial` |
| 正文全部完成 | `body` | `full_review` | 所有计划章节已生成；无待处理章节；无未关闭影响案例 | 创建全书审稿任务 | 有待处理章节或未关闭影响则保持 `body` |
| 全书审稿完成 | `full_review` | `completion_confirm` | 全书审稿报告有效；完成门禁为通过、警告或允许强制继续 | 生成完成建议和视频化建议 | 审稿失败、过期或低分阻塞则保持 `full_review` |
| 用户确认完成 | `completion_confirm` | `completion_confirm` | 用户接受审稿结果；低分或警告需确认原因 | 写入完成决策记录；生成或刷新待视频化检查；主推荐动作变为“确认进入视频化”或“处理视频化阻塞” | 未确认则保持待确认完成 |
| 用户确认待视频化 | `completion_confirm` | `video_ready` | 已有有效完成决策；待视频化检查通过；引用快照可生成 | 保存视频化快照；标记小说可视频化；开放视频模块引用入口 | 检查不通过则保持 `completion_confirm`，推荐处理阻塞来源 |

回退和重做规则：

- 修改上游正式资产，例如方向、设定、大纲、章节目录，必须评估对后续产物的影响。
- 轻微影响可以同步摘要和长篇记忆，不改变 `creationStage`。
- 中等影响需要标记受影响章节，通常回到 `body` 并 `stageStatus=blocked`。
- 严重影响需要用户选择：清空后续正文重新生成、逐章处理受影响章节、或返回修改当前内容。

## 核心状态三：阶段状态 `stageStatus`

阶段状态表达当前阶段正在发生什么。它只描述当前 `creationStage` 的状态。

为简化维护，阶段状态只保留 6 个：

| 状态 | 含义 | 展示示例 |
| --- | --- | --- |
| `not_started` | 当前阶段尚未开始 | 待生成设定 |
| `processing` | 当前阶段有任务进行中 | 生成中、审稿中、重写中 |
| `waiting_user` | 系统已给出结果，等待用户确认 | 待选择方向、待确认大纲 |
| `blocked` | 当前阶段被问题阻塞 | 章节待处理、审稿未通过 |
| `failed` | 当前阶段最近任务失败 | 生成失败、审稿失败 |
| `completed` | 当前阶段完成 | 已完成 |

“生成中、审稿中、重写中”不单独作为 `stageStatus`，由当前任务 `taskType` 表达。

### 阶段状态流转

| 业务事件 | 来源状态 | 目标状态 | 前置条件 | 副作用 | 失败处理 |
| --- | --- | --- | --- | --- | --- |
| 开始 AI 任务 | `not_started` / `waiting_user` / `failed` / `blocked` | `processing` | 无冲突任务；前置产物完整 | 创建任务；记录任务触发来源 | 创建任务失败则 `failed` |
| AI 任务成功且需要确认 | `processing` | `waiting_user` | 任务输出可解析且通过结构校验 | 保存候选结果；生成摘要和风险说明 | 结构校验失败则 `failed` |
| AI 任务成功且可自动完成 | `processing` | `completed` | 任务类型允许自动采用 | 保存结果和版本记录 | 保存失败则 `failed` |
| AI 审稿不通过 | `processing` / `waiting_user` | `blocked` | 审稿结果低于阻塞阈值 | 保存审稿报告；生成问题卡片和推荐动作 | 无 |
| 用户确认采用 | `waiting_user` | `completed` | 用户确认；低分需原因 | 候选版本变正式版本；写操作日志 | 采用失败则 `failed` |
| 用户放弃结果 | `waiting_user` | `not_started` 或 `blocked` | 用户确认放弃 | 保留候选版本为历史；重新计算推荐动作 | 无 |
| 用户处理阻塞问题 | `blocked` | `processing` 或 `waiting_user` | 用户选择处理动作 | 创建优化/重写/同步任务，或进入确认 | 处理失败则回到 `blocked` |
| 用户重试失败任务 | `failed` | `processing` | 失败原因允许重试 | 创建新的重试任务并关联原失败任务；原失败任务保留历史 | 新重试任务失败则保持 `failed` |

## 派生状态

派生状态用于判断和展示，不建议作为小说主表核心字段人工维护。

### 质量门禁

从最新审稿报告和用户确认记录推导：

- 未审稿。
- 审稿中。
- 通过。
- 警告，可继续但提示风险。
- 不通过，阻塞下一步。
- 用户强制继续，必须有原因。

流转场景：

- 审稿任务开始：未审稿 -> 审稿中。
- 审稿任务完成且高分：审稿中 -> 通过。
- 审稿任务完成且中等风险：审稿中 -> 警告。
- 审稿任务完成且低分：审稿中 -> 不通过，并让 `stageStatus=blocked`。
- 用户强制继续：不通过 -> 用户强制继续，写原因和操作日志。

### 章节汇总

从章节表、章节审稿和影响记录推导：

- 没有章节正文。
- 只有试写章节。
- 正文生成中。
- 存在待处理章节。
- 存在未审稿章节。
- 所有计划章节已完成。

流转场景：

- 开始批量生成正文：没有章节正文/只有试写章节 -> 正文生成中。
- 某章生成失败或低分：正文生成中 -> 存在待处理章节。
- 章节重写影响后文：所有计划章节已完成 -> 存在待处理章节。
- 待处理章节全部处理完：存在待处理章节 -> 所有计划章节已完成。

### 视频准备

从小说完成确认、全书审稿、章节状态和视频引用记录推导：

- 不可视频化。
- 可候选视频化。
- 待视频化。
- 已创建视频项目。
- 修改后需同步。

流转场景：

- 全书审稿完成但用户未确认：不可视频化 -> 可候选视频化。
- 用户确认完成：待确认完成 -> 待视频化候选。
- 用户确认待视频化：待视频化候选 -> 待视频化。
- 视频模块创建视频项目：待视频化 -> 已创建视频项目。
- 已引用章节被修改且影响视频：已创建视频项目 -> 修改后需同步。

## 页面展示状态

页面展示状态由后端统一生成，不手动维护。

| 展示状态 | 推导条件示例 | 展示重点 |
| --- | --- | --- |
| 待生成方向 | `creationStage=draft` | 引导点击生成方向 |
| 待选择方向 | `creationStage=direction` + `stageStatus=waiting_user` | 展示方向卡片和 AI 推荐 |
| 待生成设定 | `creationStage=setting` + `stageStatus=not_started` | 说明设定用于后续大纲和正文 |
| 待确认设定 | `creationStage=setting` + `stageStatus=waiting_user` | 展示设定摘要和风险 |
| 待生成大纲 | `creationStage=outline` + `stageStatus=not_started` | 说明全书结构 |
| 待确认章节目录 | `creationStage=chapter_plan` + `stageStatus=waiting_user` | 展示章节总数和阶段分布 |
| 试写中 | `creationStage=trial` + `stageStatus=processing` | 展示当前任务进度 |
| 试写需优化 | `creationStage=trial` + `stageStatus=blocked` | 展示核心问题和优化入口 |
| 正文生成中 | `creationStage=body` + `stageStatus=processing` | 展示章节生成进度 |
| 章节待处理 | 章节汇总为存在待处理章节 | 展示低分或受影响章节入口 |
| 待全书审稿 | `creationStage=body` + `stageStatus=completed` | 引导全书审稿 |
| 全书需优化 | `creationStage=full_review` + `stageStatus=blocked` | 展示全书问题 |
| 待确认完成 | `creationStage=completion_confirm` + `stageStatus=waiting_user` | 展示评分和是否进入视频化 |
| 待视频化 | `creationStage=video_ready` | 引导进入视频模块 |
| 已暂停 | `lifecycleStatus=paused` | 显示暂停原因和恢复入口 |
| 已归档 | `lifecycleStatus=archived` | 显示归档原因和恢复/复制入口 |

## 推荐动作

`recommendedAction` 不作为人工维护状态。

它由以下信息推导：

- `lifecycleStatus`
- `creationStage`
- `stageStatus`
- 当前主任务状态
- 最新审稿结果
- 章节汇总结果
- 视频引用结果
- 是否存在阻塞原因

接口可以返回 `recommendedAction` 快照，方便列表展示。但每次状态或关键产物变化后，后端需要重新计算。

## 必须阻塞的场景

以下情况必须阻塞小说进入完成确认或待视频化：

- 有进行中的主链路任务。
- 有失败任务没有处理，且该任务影响当前阶段产物。
- 设定未确认。
- 全书大纲或章节目录未确认。
- 试写章节未通过或用户未确认风险。
- 计划章节未全部生成。
- 存在章节主状态为待处理或处理中。
- 存在章节未生成摘要/特性卡片。
- 存在章节未审稿，且策略要求章节审稿。
- 存在未关闭的中等或严重章节影响案例。
- 全书 AI 审稿未完成。
- 全书 AI 审稿结果为不通过，且用户没有强制继续原因。
- 全书 AI 审稿报告已过期。
- 待视频化检查不通过时，只阻塞进入 `video_ready`，不阻塞用户确认小说完成。
- 小说被归档、暂停或删除。

## 和关联模块的关系

本文档定义核心状态、派生状态和状态流转机制。具体页面、任务和门禁细节由以下模块承接：

- `docs/modules/novel-recommended-actions.md`：每个展示状态对应的主按钮和承载位置。
- `docs/modules/novel-state-gate-action-contract.md`：状态、门禁、推荐动作总矩阵和字段口径统一。

- `docs/modules/novel-task-concurrency-contract.md`：任务并发、批量任务和上游版本变化处理。
- `docs/modules/novel-detail-workbench.md`：单本小说完整工作台。
- `docs/modules/novel-chapter-workbench.md`：单章正文、审稿、候选版本和影响处理。
- `docs/modules/generation-task-lifecycle.md`：AI 任务状态、失败、重试、确认和互斥。
- `docs/modules/novel-video-readiness.md`：完成确认后进入视频承接的判定。

# 热点分析系统模块设计

## 系统定位

热点分析系统是内容生产链路的上游情报系统。

它不是小说创建页面里的一个功能，而是独立负责收集、整理、分析市场热点，并对外提供热点报告、题材趋势、爽点趋势和创作机会点。

完整服务化设计见 `docs/modules/hotspot-service-system.md`。本文保留热点分析系统的基础模块边界，服务化文档补充定期报告、手动触发、对外接口、报告结构、机会点结构、小说创建引用和效果回流。

## 第一阶段目标

第一阶段不做复杂自动爬虫，先做可控的半自动热点分析：

- 用户手动录入或粘贴短视频标题、文案、链接、评论摘要、榜单信息。
- AI 总结这些素材里的题材、爽点、开头钩子、受众和风险。
- 系统生成热点分析报告。
- 用户可以从热点报告一键进入创建小说流程。
- 小说系统可以调用热点分析系统，获取默认创作建议。
- 如果用户没有提供热点素材，系统不能卡住，需要使用内置热点题材库生成默认热点报告。

## 核心能力

1. 热点素材管理
   - 保存标题、链接、文案、评论摘要、来源平台、备注。
   - 支持按时间、平台、题材打标签。

2. 热点报告生成
   - 支持手动触发。
   - 支持定期生成，例如每日或每周。
   - 支持从一组热点素材生成结构化报告。
   - 支持在无用户素材时，根据内置热点题材库生成默认报告。

3. 热点趋势总结
   - 热门题材。
   - 热门爽点。
   - 常见主角开局。
   - 高频冲突。
   - 开篇钩子。
   - 目标受众。
   - 可改编成小说的机会点。
   - 同质化和平台风险。

4. 创作机会输出
   - 输出 3-10 个可用于小说创作的机会点。
   - 每个机会点说明适合的题材、爽点、受众、风险和推荐理由。
   - 机会点只能作为原创参考，不能直接抄袭热点内容。

5. 对外服务
   - 获取最新热点报告。
   - 获取指定时间范围内的热点报告列表。
   - 获取热门题材、热门爽点、开篇钩子和受众趋势。
   - 根据目标用户或题材偏好推荐小说创作机会点。
   - 根据一批素材生成临时热点分析结果。
   - 为小说创建流程提供默认选题建议。

## 无素材兜底机制

第一版热点分析系统必须支持没有外部素材的情况。

兜底顺序：

1. 用户提供热点素材时，优先分析用户素材。
2. 用户没有素材时，使用内置热点题材库生成默认热点报告。
3. 后续接入第三方热点服务后，可以优先使用第三方服务生成报告。

内置热点题材库可以先包含：

- 男频：都市逆袭、重生复仇、战神归来、系统暴富、修仙升级。
- 女频：重生复仇、豪门追妻、替嫁、甜宠、虐恋反转。
- 悬疑：规则怪谈、民间异闻、探案反转、无限流。
- 短剧感：身份曝光、前任后悔、家族打脸、误会反转。

默认热点报告需要明确标记来源为“内置题材库”，避免和真实市场热点混淆。

默认热点报告的目标不是准确反映实时市场，而是保证系统能在没有素材时继续生成相对靠谱的小说方向。

6. 与小说系统联动
   - 创建小说时可以选择一个热点报告。
   - 小说方向需要记录引用了哪个热点报告。
   - 后续发布数据可以反哺热点报告，判断哪些热点真的有效。

## 后续扩展

- 接入第三方热点分析服务。
- 接入榜单、数据平台或自建采集任务。
- 自动生成周期性热点报告。
- 将视频发布数据、完播率、点赞、评论、转化数据回流到热点系统。
- 支持多个平台分别分析，例如抖音、快手、小红书、番茄、短剧平台。
- 未来可作为独立能力售卖或开放给外部系统。

## 设计原则

- 第一阶段优先半自动，避免在爬虫和平台限制上投入过多。
- 热点分析结果必须服务创作，而不是只做数据展示。
- 报告要让小白能看懂，避免复杂运营术语。
- 输出必须有“推荐做什么”和“为什么推荐”，不能只列热门词。
- 所有热点都要经过原创化处理，避免版权和平台风险。

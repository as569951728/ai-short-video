# 失败和异常处理

本文档补齐 `GAP-P1-004`：小说系统在生成失败、审稿失败、解析失败、模型超时、输出不合格、用户取消、状态变化、候选过期等异常场景下，如何给用户清晰提示和下一步动作。

本文档只定义需求规则，不讨论具体实现方案。

## 设计目标

- 失败时用户知道发生了什么。
- 失败时用户知道下一步可以点什么。
- 技术错误不直接暴露给小白。
- 不让失败任务把小说卡死在不可理解状态。
- 所有失败都能被复盘。

## 失败分类

用户可见失败分类保持通俗：

| 用户看到的分类 | 典型技术原因 | 推荐动作 |
| --- | --- | --- |
| 模型暂时不可用 | 模型接口异常、限流、超时 | 稍后重试、换模型重试 |
| 内容太长 | 上下文过长、章节太长 | 缩短范围、分段生成 |
| 结果格式不对 | 结构化输出解析失败 | 重新整理结果、重试 |
| 质量不达标 | 审稿低分、输出跑题 | 优化输入、重新生成 |
| 内容有风险 | 平台或安全风险 | 调整设定或剧情 |
| 状态已变化 | 上游版本变化、候选过期 | 基于最新内容重新生成 |
| 有任务冲突 | 同一对象已有任务进行中 | 查看当前任务、等待或取消 |
| 用户已取消 | 用户主动取消任务 | 回到上一推荐动作 |

完整技术错误码只在高级任务详情中展示，并且必须脱敏。

## 失败展示规则

失败提示必须包含：

- 失败发生在哪个动作。
- 通俗失败原因。
- 是否影响当前小说进度。
- 推荐下一步。
- 是否可以重试。

失败提示不能只显示：

- 失败。
- 系统异常。
- undefined。
- 原始模型错误。
- 原始堆栈。

## 推荐动作

不同失败场景需要给出明确动作：

| 场景 | 推荐动作 |
| --- | --- |
| 临时模型异常 | 重试 |
| 多次模型异常 | 换模型重试 |
| 上下文太长 | 缩短范围后重试 |
| 输出格式不合格 | 重新整理结果 |
| 内容质量低 | 优化输入或重新生成 |
| 内容风险 | 返回设定/大纲调整 |
| 候选过期 | 基于最新内容重新生成 |
| 任务冲突 | 查看当前任务 |
| 用户取消 | 回到上一推荐动作 |

每次失败页面或抽屉只能突出一个主推荐动作，其余动作作为次要选项。

## 任务失败与业务阻塞

任务失败不一定等于小说阶段失败。

规则：

- 如果失败任务影响当前阶段核心产物，小说当前阶段显示失败或阻塞。
- 如果失败任务只是次要优化任务，小说主流程不一定阻塞。
- 如果失败任务已有可用旧版本，系统可以提示继续使用旧版本。
- 如果失败任务导致关键产物缺失，必须阻塞下一步。

示例：

- 设定生成失败：阻塞进入大纲。
- 章节局部优化失败：可继续使用当前正式章节。
- 全书审稿失败：阻塞确认完成。
- 视频引用重新检查失败：不一定阻塞小说完成，但阻塞创建新视频项目。

## 重试规则

重试前需要说明：

- 是否会产生新结果。
- 是否会增加成本。
- 是否基于最新内容。
- 是否会覆盖当前内容。

重试结果仍遵守 AI 产物确认规则：

- 关键资产生成候选。
- 用户确认后才采用。
- 不静默覆盖当前正式内容。

多次失败后，系统应建议：

- 换模型重试。
- 缩短输入范围。
- 返回上游修改设定、大纲或章节摘要。
- 暂停当前项目。

## 取消规则

用户取消任务后，应回到可理解状态。

取消后展示：

- 已取消什么任务。
- 是否保留已生成部分结果。
- 当前小说下一步是什么。

取消不能造成：

- 按钮一直 loading。
- 小说状态停在处理中。
- 用户不知道还能不能继续。

## 候选过期

候选结果基于旧版本生成时，必须提示过期。

过期候选默认动作：

- 放弃旧候选。
- 基于最新内容重新生成。
- 查看旧候选摘要。

如果用户仍想采用过期候选，必须进入高风险确认，并说明可能影响设定、大纲、章节或视频引用。

## 空数据和异常数据

如果页面缺少必要数据，不能显示空白页。

常见提示：

- 当前小说缺少设定，请先生成设定。
- 当前章节还没有正文，请先生成正文。
- 当前审稿报告已过期，请重新检查。
- 当前任务已不存在，请刷新状态。
- 当前项目已归档，恢复后才能继续创作。

每个空状态都要有下一步动作。

## 承载位置

- 列表展示失败摘要和主推荐动作。
- 任务抽屉展示失败原因、重试和取消。
- 弹窗用于高风险确认。
- 任务详情展示完整排查信息。
- 小说详情工作台展示项目级失败历史。

## 需求验收标准

- 所有失败都能映射到用户可理解分类。
- 所有失败都有下一步动作。
- 失败不会让小说停在不可操作状态。
- 重试不会静默覆盖正式内容。
- 候选过期有明确提示和处理动作。
- 技术错误和密钥不会暴露给小白用户。

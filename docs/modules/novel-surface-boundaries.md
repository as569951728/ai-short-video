# 列表、抽屉、弹窗、详情边界

本文档补齐 `GAP-P1-001`：小说系统已经确认“列表驱动 + 工作台承接”的产品方向，但不同操作到底留在列表、进入抽屉、打开弹窗、进入小说详情工作台，还是进入章节详情工作台，需要形成稳定规则。

本文档只定义需求和交互承载边界，不讨论实现计划。

## 核心结论

小说列表是小白用户的默认驾驶舱，详情工作台和章节工作台是复杂问题的维修间。这个原则已经在 `docs/modules/novel-p0-visible-main-flow.md` 中明确。

P1-001 进一步规定：

- 列表优先展示状态、筛选和详情入口，避免承载过多创作操作。
- 轻量确认和摘要查看进入抽屉或弹窗。
- 长文本、复杂版本、复杂影响、全书结构问题进入工作台。
- 单章问题进入章节详情工作台。
- 项目级问题进入小说详情工作台。
- 视频项目创建属于视频模块，小说模块只展示视频引用状态和跳转入口。

## 承载容器定义

| 容器 | 定位 | 用户感知 |
| --- | --- | --- |
| 小说列表 | 当前所有小说的驾驶舱 | 我下一步点什么 |
| 行展开区 | 列表里的轻量摘要 | 为什么推荐我这样做 |
| 抽屉 | 不离开列表的轻量处理区 | 看结果、确认、重试、接受风险 |
| 弹窗 | 强确认或单一决策 | 确认是否继续 |
| 小说详情工作台 | 单本小说完整维修间 | 结构、审稿、版本、任务、视频引用 |
| 章节详情工作台 | 单章维修间 | 正文、问题、重写、影响后文 |
| 任务详情页 | 高级排查入口 | 为什么任务失败、用了什么配置 |

## 总体边界规则

### 留在小说列表

满足以下条件的动作可以留在列表：

- 用户只需要知道当前状态和主推荐动作。
- 操作不需要阅读长文本。
- 操作不需要比较多个版本。
- 操作不会直接改变大量正式内容。
- 操作结果可以通过任务进度或轻量确认表达。
- 不需要进入视频项目创建、正文生成进度、归档原因等复杂场景。

典型场景：

- 查看小说状态。
- 查看最近任务进度摘要。
- 查看当前阻塞原因摘要。
- 查看章节进度和待处理数量。
- 筛选待处理、生成中、可视频化、已归档小说。
- 点击“详情”进入小说详情工作台。

### 进入行展开区

满足以下条件的内容适合行展开区：

- 是列表当前行的补充说明。
- 不需要用户做复杂决策。
- 内容可以在 1-2 屏内读完。
- 主要用于解释“为什么推荐这个动作”。

行展开区适合展示：

- 当前推荐动作原因。
- 最近任务摘要。
- 最新审稿摘要。
- 章节统计摘要。
- 待处理章节数量。
- 视频引用风险摘要。

行展开区不适合展示：

- 完整正文。
- 完整审稿报告。
- 版本对比。
- 多章节影响明细。
- 复杂高风险决策。

### 进入抽屉

抽屉适合“轻量但需要看一点信息才能确认”的操作。

满足以下条件的动作进入抽屉：

- 用户需要查看摘要后确认。
- 内容比弹窗多，但不需要完整工作台。
- 操作后仍能回到列表继续推进。
- 结果有明确的采用、放弃、重试或接受风险动作。

抽屉适合：

- 确认小说设定摘要。
- 确认全书大纲摘要。
- 确认章节目录摘要。
- 查看试写总评摘要。
- 查看任务进度和失败原因。
- 查看候选结果摘要。
- 查看全书质量检查摘要。
- 确认进入待视频化。
- 接受低分风险继续。

抽屉不适合：

- 长正文编辑。
- 精细版本对比。
- 多章节批量影响处理。
- 全书结构大调整。
- 手动维护复杂设定字段。

### 进入弹窗

弹窗只用于单一、强约束、短决策，不承载复杂内容。

弹窗适合：

- 删除确认。
- 归档原因填写。
- 暂停原因填写。
- 低分接受风险继续。
- 清空后续章节强确认。
- 放弃候选结果确认。
- 进入待视频化确认。

弹窗必须说明：

- 用户将要做什么。
- 会影响哪些对象。
- 是否可以回退。
- 是否需要填写原因。

弹窗不适合：

- 展示完整大纲。
- 展示完整正文。
- 展示完整审稿报告。
- 展示长版本差异。

### 进入小说详情工作台

小说详情工作台承载项目级复杂问题。

以下场景必须进入小说详情工作台：

- 大纲结构需要调整。
- 章节目录需要批量调整。
- 全书质量检查发现结构性问题。
- 需要查看完整审稿报告。
- 需要查看全书版本历史。
- 需要查看任务历史和失败复盘。
- 需要查看视频关联和引用异常明细。
- 需要项目复盘、归档原因、成功/失败总结。

小说详情工作台默认仍应先展示：

- 当前状态。
- 当前推荐动作。
- 阻塞原因。
- 最近任务。
- 最新审稿摘要。

不应该默认把全部区块展开。

### 进入章节详情工作台

章节详情工作台只处理单章问题。

以下场景必须进入章节详情工作台：

- 阅读或编辑单章正文。
- 查看单章完整审稿问题。
- 一键重写或局部重写章节。
- 采用或放弃章节新改稿。
- 对比章节正文版本。
- 处理当前章影响后续章节的问题。
- 修改已被视频引用的章节。

章节详情工作台不处理：

- 全书大纲调整。
- 阶段数量调整。
- 全书质量检查汇总。
- 视频项目详情。
- 项目级复盘。

## 按主流程划分承载位置

| 主流程节点 | 默认承载 | 说明 |
| --- | --- | --- |
| 创建小说 | 创建小说页面 | 创建页完成方向确认 |
| 生成设定 | 小说详情工作台主推荐动作 | 任务完成后在详情页或结果抽屉确认设定摘要 |
| 确认设定 | 抽屉 | 只展示小白摘要、风险和确认按钮 |
| 生成全书大纲 | 小说详情工作台主推荐动作 | 生成任务进度留在详情页任务区或任务抽屉 |
| 确认全书大纲 | 抽屉 | 展示主线、阶段数、风险摘要 |
| 生成章节目录 | 小说详情工作台主推荐动作 | 生成任务进度留在详情页任务区或任务抽屉 |
| 确认章节目录 | 抽屉 | 展示章节数量、阶段分布、前几章摘要 |
| 试写前三章 | 小说详情工作台主推荐动作 | 任务进度在详情页任务区或任务抽屉 |
| 查看试写结果 | 抽屉 | 展示总分、Top 3 问题、是否可继续 |
| 优化试写章节 | 章节详情工作台 | 需要看正文和问题时进入 |
| 批量生成正文 | 小说详情工作台章节管理区 | 列表不提供正文批量生成入口 |
| 处理待处理章节 | 章节详情工作台 | 单章问题进入单章维修间 |
| 全书质量检查 | 小说详情工作台主推荐动作 | 结果摘要先进入详情页审稿区或抽屉 |
| 全书结构性优化 | 小说详情工作台 | 涉及大纲、多个章节或全书问题 |
| 确认小说完成 | 弹窗或抽屉 | 有风险摘要时用抽屉，纯确认可弹窗 |
| 检查是否可以做视频 | 抽屉 | 展示检查清单和创建视频入口 |
| 创建视频项目 | 视频模块 | 小说模块只展示可视频化和引用状态 |

## 按问题类型划分承载位置

| 问题类型 | 默认承载 | 说明 |
| --- | --- | --- |
| 当前任务处理中 | 任务抽屉 | 展示进度和取消条件 |
| 当前任务失败 | 任务抽屉 | 展示失败原因、重试、换模型或返回修改 |
| 设定低分 | 设定确认抽屉 | 展示风险和优化入口 |
| 大纲低分 | 大纲确认抽屉；严重时小说详情工作台 | 轻问题可确认，结构问题进工作台 |
| 章节低分 | 章节详情工作台 | 需要查看正文和问题 |
| 试写低分 | 试写结果抽屉；单章问题进章节详情 | 先摘要，再处理具体章节 |
| 中等章节影响 | 影响摘要抽屉；处理进入章节详情 | 展示受影响章节数量和推荐处理 |
| 严重章节影响 | 章节详情工作台 | 三选一决策需要上下文 |
| 全书审稿不通过 | 全书结果抽屉；结构问题进小说详情 | 先展示 Top 3 问题 |
| 视频引用异常 | 视频引用抽屉；复杂时小说详情视频区 | 已发布内容不自动覆盖 |
| 暂停/归档/删除 | 弹窗 | 必须填写或确认原因 |

## 推荐动作目标规则

`recommendedAction.target` 需要与承载边界一致。

| target | 使用场景 | 示例 |
| --- | --- | --- |
| `inline` | 当前页面内直接触发轻量动作 | 详情页内生成设定、生成大纲、试写、全书质量检查 |
| `drawer` | 需要看摘要和确认 | 确认设定、确认大纲、查看任务失败 |
| `modal` | 单一强确认 | 归档、删除、接受风险继续 |
| `page` | 复杂工作台 | 打开小说详情、打开章节详情 |
| `task` | 进入任务抽屉或任务中心 | 查看任务 |
| `external` | 跳转下游模块 | 去视频列表 |

规则：

- `inline` 动作如果会创建 AI 任务，需要立即转为任务状态展示。
- 小说列表行不使用 `inline` 承载创作生成动作；列表行只提供“详情”进入对应工作台。
- `drawer` 不能承载长正文和复杂版本对比。
- `modal` 不能承载多步骤流程。
- `page` 用于复杂问题，不作为小白默认推进方式。
- `task` 用于任务详情或任务抽屉，不直接改变创作资产。
- 列表行主要按钮统一为“详情”；归档、暂停、删除等高风险动作进入详情后处理。
- 视频创建按钮不出现在小说列表行操作中。

## 页面跳转规则

为了避免用户迷路，跳转必须保持返回路径。

从列表进入：

- 进入小说详情后，返回小说列表时保留筛选条件。
- 进入章节详情后，返回小说详情或列表时保留章节筛选和小说行位置。
- 从任务抽屉进入任务详情后，返回时回到原小说或原任务抽屉。

从详情进入：

- 从小说详情进入章节详情，返回时定位到章节管理区。
- 从小说详情进入任务详情，返回时定位到任务区。
- 从小说详情进入视频项目，返回时定位到视频关联区。

异常返回：

- 如果目标对象已被删除、归档或版本过期，需要回到可解释页面，不显示空白错误。
- 如果用户无权限或状态已变化，需要展示“状态已更新，请按新的推荐动作继续”。

## 高风险动作承载规则

高风险动作不能只靠普通按钮。

高风险动作包括：

- 删除小说。
- 归档小说。
- 暂停小说。
- 清空后续章节。
- 接受低分风险继续。
- 手动确认中等或严重影响无问题。
- 采用会影响后续章节的新改稿。
- 修改已被视频引用章节。

承载要求：

- 必须有弹窗或抽屉确认。
- 必须说明影响范围。
- 必须说明是否可回退。
- 需要原因时必须提供原因输入或原因选项。
- 确认后需要回到新的推荐动作，而不是停留在旧按钮。

## 小白默认隐藏规则

以下内容默认隐藏，只有用户主动展开或进入高级入口才展示：

- 完整审稿维度。
- 完整模型信息。
- 完整提示词信息。
- token 和成本明细。
- 完整任务事件时间线。
- 全量版本历史。
- 复杂影响评估明细。
- 项目复盘数据。

默认可见内容应限制在：

- 当前状态。
- 当前推荐动作。
- 核心风险。
- Top 3 问题。
- 一个最推荐处理动作。

## 需求验收标准

本模块设计完成后，后续原型和需求评审至少需要能回答：

- 每个主推荐动作点击后打开哪里。
- 每个高风险操作是否有确认和原因。
- 哪些内容绝不能出现在列表首屏。
- 哪些复杂问题必须进入详情工作台。
- 从工作台返回列表时，用户是否能回到原来的上下文。
- 小白用户是否可以只通过列表主按钮走完整条主链路。

如果某个操作无法明确归属到列表、抽屉、弹窗、小说详情或章节详情，说明这个操作的产品边界还没设计清楚。

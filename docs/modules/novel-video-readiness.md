# 待视频化判定

本文档补齐 `GAP-P0-012`：小说完成后，系统如何判断是否可以进入视频系统承接，如何生成待视频化状态，如何处理小说修改、章节范围、视频引用和低分风险。

待视频化不是“小说已完成”的同义词。它表示小说已经通过必要门禁，允许被视频模块引用，用于生成音频、字幕、解压视频合成或后续更复杂的分镜视频。

待视频化判定需要引用 `docs/modules/novel-hit-content-planning.md` 中的短视频旁白单元、首条视频钩子、首屏字幕和推荐切片标准，避免只判断“能不能引用正文”，却没有给运营测试提供可用切片。具体字段和小白展示按 `docs/modules/novel-hit-content-integration-matrix.md` 的待视频化落点执行。

需求口径统一为两步：先由用户确认小说完成，生成完成决策记录；再进行待视频化检查并由用户确认进入视频化。待视频化检查不通过时，小说可以保持“已确认完成但暂不可视频化”，不能直接开放正式视频项目创建。

## 设计目标

- 明确小说什么时候可以出现在视频创建入口。
- 避免未完成、低质量、待处理或版本过期的小说被视频系统误用。
- 给视频模块提供稳定的引用范围和内容快照。
- 小说修改后能识别视频引用异常。
- 初期支持简单视频方案，后续可扩展到分镜、短视频集和发布数据回流。

## 判定结论

视频准备状态建议作为派生状态，不作为小说主表核心状态人工维护。

| 状态 | 含义 | 主推荐动作 |
| --- | --- | --- |
| `not_ready` | 不可视频化 | 返回当前阻塞步骤 |
| `candidate` | 全书审稿完成，等待用户确认 | 确认进入视频化 |
| `ready` | 可被视频系统引用 | 去视频列表 |
| `video_created` | 已创建至少一个视频项目 | 查看视频项目 |
| `reference_stale` | 小说修改导致视频引用可能过期 | 查看引用异常 |

`creationStage=video_ready` 表示主链路已经进入视频承接阶段；视频引用异常仍通过派生状态和视频引用记录表达。

## 前置条件

进入待视频化至少满足：

- 小说生命周期为 `active`。
- 小说已完成全书 AI 审稿。
- 全书审稿报告未过期。
- 完成门禁为通过、警告通过或用户强制通过。
- 用户已确认小说完成。
- 所有计划章节有当前正式正文。
- 所有计划章节有章节摘要/特性卡片。
- 不存在 `pending` 或 `processing` 章节。
- 不存在未处理的中等或严重影响案例。
- 不存在阻塞性的内容安全高风险。
- 不存在会影响正文的进行中任务。

如果只想把部分章节做成正式视频项目，需要另行创建视频项目引用范围，小说本身仍建议先满足全书完成门禁。

试写阶段允许生成“首条测试视频”，它是运营验证分支，不代表小说进入 `video_ready`，也不允许引用未确认完成的全书内容创建正式视频项目。

## 视频化检查清单

系统在确认待视频化前生成检查清单：

| 检查项 | 通过要求 |
| --- | --- |
| 完成确认 | 用户已确认小说完成 |
| 审稿有效 | 最新全书审稿基于当前正文版本 |
| 章节完整 | 计划章节都有正式正文 |
| 章节状态 | 无待处理、处理中章节 |
| 影响处理 | 无未关闭的影响案例 |
| 内容风险 | 无强阻塞安全风险 |
| 视频引用 | 当前未存在冲突引用，或已提示风险 |
| 内容快照 | 能生成稳定的章节引用快照 |

检查失败时，推荐动作必须指向具体阻塞来源，例如“处理待处理章节”“重新全书审稿”“确认小说完成”。

## 试写后首条测试

试写总评通过或警告通过后，系统可以生成首条测试卡，帮助用户尽早验证选题和开篇吸引力。

首条测试卡包含：

- 推荐引用章节范围，默认前 1-3 章或试写中最强冲突片段。
- 标题钩子 2-3 个。
- 前 3 秒旁白钩子 2-3 个。
- 首屏字幕 2-3 个。
- 结尾悬念。
- 简单视频生成方式，默认“旁白音频 + 字幕 + 循环背景视频”。
- 发布平台和建议发布时间。
- 发布前检查。

规则：

- 首条测试视频只能引用试写章节的当前正文版本。
- 首条测试视频需要保存引用快照，但不改变小说 `creationStage`。
- 测试结果作为运营信号回流，不替代全书审稿和正式待视频化判定。
- 用户可以跳过首条测试，继续批量正文生成。

## 用户确认进入视频化

确认弹窗展示：

- 小说名称。
- 全书评分和评级。
- 章节数和总字数。
- 预计普通语速音频时长。
- 是否存在低分或警告风险。
- 视频化建议：整本、按阶段、按章节范围或先做开篇试投。
- 首条视频建议：推荐章节范围、前 3 秒旁白钩子、首屏字幕、标题钩子和结尾悬念。
- 确认后视频系统可引用的内容范围。

用户确认后：

- `creationStage` 进入 `video_ready`。
- 记录 `VideoReadinessSnapshot`。
- 生成或刷新视频引用可用范围。
- 小说列表展示“待视频化”和视频引用状态；创建视频项目从视频模块发起。

如果是低分强制通过，视频化快照必须记录风险来源和用户原因。

## 视频引用范围

创建视频项目时，用户可以选择引用范围：

| 范围 | 说明 | 适用场景 |
| --- | --- | --- |
| 整本小说 | 引用全部计划章节 | 后续拆成短视频小说集 |
| 阶段/分卷 | 引用某个阶段章节 | 按剧情阶段做系列 |
| 章节范围 | 引用第 N 到 M 章 | 先做开篇或某个高潮片段 |
| 单章 | 只引用一章 | 测试旁白、封面和视频模板 |

初期最简单方案可以只支持章节范围，并默认选中前 1-3 章或全书。

创建视频项目时需要保存内容快照：

- 引用章节 ID 列表。
- 引用章节版本 ID 列表。
- 章节摘要和正文更新时间。
- 全书审稿报告 ID。
- 视频化风险摘要。

视频项目后续不应直接依赖实时正文，否则小说修改会导致历史视频内容不可追溯。

## 小说修改后的引用异常

小说进入待视频化或已创建视频项目后，如果发生以下变化，需要标记引用异常：

- 被视频引用的章节正式正文变化。
- 被视频引用章节的摘要/特性卡片关键变化。
- 被视频引用范围内章节被清空或重新生成。
- 全书审稿报告过期。
- 小说从待视频化回退到正文生成、章节处理或全书审稿。
- 内容安全策略变化导致已引用内容风险升高。

引用异常分级：

| 等级 | 含义 | 推荐动作 |
| --- | --- | --- |
| `info` | 小说有轻微润色，视频可不处理 | 查看差异 |
| `warning` | 引用章节内容变化，建议重新生成音频/字幕 | 标记视频待同步 |
| `blocking` | 引用正文失效或风险升高 | 暂停继续生成视频 |

已发布视频不会被自动修改。系统只记录“小说已变更，平台已发布内容可能与当前小说不一致”。

## 简单视频方案承接

初期视频系统可以采用最低可行方案：

- 小说正文转语音。
- 生成字幕。
- 使用循环解压视频或固定素材。
- 合成短视频。
- 记录视频项目和引用章节范围。

待视频化判定只需要保证小说内容可引用、版本可追溯、修改后能提示异常，不要求一开始就具备自动分镜和高质量视频生成。

## 完成与回退

进入 `video_ready` 后，小说仍可能因为修改回退：

| 场景 | 处理 |
| --- | --- |
| 修改未被视频引用章节且不影响全书 | 保持待视频化，刷新风险提示 |
| 修改被视频引用章节 | 标记视频引用异常 |
| 章节重写造成中等或严重影响 | 回到正文问题处理，视频准备变为不可用 |
| 全书审稿过期 | 回到待全书审稿或完成确认前 |
| 用户归档小说 | 视频创建入口隐藏，已创建视频仍可查看 |

回退不是删除视频项目，视频项目保留引用快照和异常提示。

## 页面交互

小说列表：

- 展示视频引用状态。
- 待视频化时显示“可被视频引用”或“待视频化”。
- 已创建视频时显示已引用或引用数量摘要。
- 引用异常时显示引用异常标签。
- 列表主按钮仍为“详情”，不直接创建视频项目。

小说详情工作台视频关联区：

- 展示待视频化检查清单。
- 展示视频化快照。
- 展示已创建视频项目。
- 展示引用异常和受影响章节。
- 提供跳转视频列表创建视频项目的入口。

视频列表：

- 引用小说标题、章节范围、引用状态、生成状态、发布状态。
- 如果引用异常，显示原因和处理入口。

## 数据与接口边界

建议对象：

- `VideoReadinessSnapshot`：小说进入待视频化时的检查和内容快照。
- `VideoReference`：视频项目引用小说章节的版本快照。
- `VideoReferenceIssue`：小说修改后产生的引用异常。

建议接口：

- `GET /novels/:novelId/video-readiness`：获取待视频化判定结果和检查清单。
- `POST /novels/:novelId/confirm-video-ready`：确认进入待视频化。
- `POST /novels/:novelId/video-readiness/recheck`：重新检查视频化状态。
- `POST /videos`：创建视频项目并绑定小说章节范围。
- `GET /videos/:videoId/reference`：查看视频引用快照。
- `POST /videos/:videoId/reference/recheck`：重新检查引用异常。

确认待视频化和创建视频项目必须做版本校验，避免用户基于过期页面创建视频。

## 和其他模块的关系

- 状态机：待视频化对应 `creationStage=video_ready`。
- 全书审稿门禁：只有完成门禁通过并确认完成，才能进入待视频化。
- 章节重写影响处理：存在未处理影响时不可视频化；视频引用章节修改后产生引用异常。
- 视频系统：接收小说内容快照，不直接修改小说正文。
- 推荐动作：视频准备状态决定小说详情和视频模块的承接入口；小说列表只展示视频引用状态。
- 爆款内容策划体系：待视频化检查需要引用短视频旁白单元和首条验证建议。

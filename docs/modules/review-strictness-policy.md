# 审稿严格程度与策略

本文档补齐 `GAP-P1-003`：小说系统中方向审稿、试写审稿、章节审稿、全书审稿和视频化前检查，需要有统一的严格程度和策略口径，避免不同文档出现分数阈值不一致、阻塞规则不清晰的问题。

本文档只定义需求规则，不讨论配置后台如何实现。

## 设计目标

- 让审稿结果能稳定决定“能不能进入下一步”。
- 让小白用户看懂评分，不被复杂维度吓住。
- 允许不同严格程度，但默认策略必须统一。
- 低分能否继续、什么时候必须阻塞，需要有明确口径。
- 审稿策略调整后，历史审稿结果仍能解释。

## 审稿层级

审稿分为五类：

| 审稿类型 | 作用 |
| --- | --- |
| 方向审稿 | 判断小说方向是否值得继续 |
| 试写审稿 | 判断开篇、文风和爽点是否适合批量生成 |
| 章节审稿 | 判断单章是否可用于后续创作 |
| 全书审稿 | 判断整本小说是否可以完成 |
| 视频化前检查 | 判断小说是否可以被视频系统引用 |

## 默认评分档位

默认使用统一的标准策略，避免不同模块各自定义阈值。

| 分数 | 评级 | 默认含义 | 小白文案 |
| --- | --- | --- | --- |
| 80-100 | 通过 | 可以进入下一步 | 可以继续 |
| 70-79 | 警告 | 可以继续，但建议优化 | 建议优化后继续 |
| 60-69 | 需修改 | 默认阻塞，需要处理问题 | 建议先修改 |
| 0-59 | 不建议继续 | 强阻塞，除非策略允许接受风险 | 不建议继续 |

此前文档中出现的 85/75/65 只能作为严格策略的示例，不作为默认标准策略。

## 严格程度

系统支持四个严格程度：

| 严格程度 | 使用场景 | 行为 |
| --- | --- | --- |
| 宽松 | 早期试错、快速探索题材 | 低分多提示，少阻塞 |
| 标准 | 默认小说生产流程 | 使用统一 80/70/60 档位 |
| 严格 | 全书完成、视频化前检查 | 更重视一致性、风险和可视频化 |
| 极严格 | 样板内容、对外售卖前审核 | 低容忍风险，更多强阻塞 |

小白默认使用标准策略，不需要选择严格程度。

## 阻塞规则

分数不是唯一阻塞条件。

以下情况必须阻塞，不能通过接受风险继续绕过：

- 缺少正文。
- 缺少章节摘要/特性卡片。
- 存在待处理章节。
- 存在未关闭的中等或严重影响案例。
- 全书审稿基于旧版本。
- 待视频化快照基于旧版本。
- 内容安全强风险。
- 任务仍在处理中。

以下情况可以允许“接受风险继续”：

- 评分处于警告区间。
- 评分处于需修改区间，但没有数据完整性和安全强阻塞。
- 用户明确想先跑完整链路再回头优化。

接受风险继续必须填写原因。

## 各审稿类型的重点

### 方向审稿

重点：

- 市场匹配。
- 爽点是否清晰。
- 主角开局是否抓人。
- 长篇支撑力。
- 视频化潜力。
- 抄袭和同质化风险。

方向低分可以继续，但必须提示后续生成质量风险。

### 试写审稿

重点：

- 开篇吸引力。
- 前 1-3 章爽点兑现。
- 文风是否适合持续生成。
- 节奏是否拖沓。
- 旁白是否顺。

试写低分时，默认推荐优化试写章节，不建议直接批量生成。

### 章节审稿

重点：

- 本章目标完成。
- 冲突强度。
- 爽点密度。
- 人设一致性。
- 结尾钩子。
- 是否影响后文。

章节低分会让章节进入待处理，处理完成前不能进入全书质量检查。

### 全书审稿

重点：

- 全书完整度。
- 主线清晰。
- 前后连贯。
- 伏笔回收。
- 爽点持续性。
- 结尾完成度。
- AI 味和平台风险。
- 视频化适配。

全书审稿未通过时，默认不能确认完成。

### 视频化前检查

重点：

- 全书审稿是否有效。
- 章节版本是否稳定。
- 是否存在引用风险。
- 是否适合转语音和字幕。
- 是否存在强内容风险。

视频化前检查不是重新审整本小说，而是判断能否安全引用小说内容。

## 用户展示规则

小白默认只展示：

- 总分。
- 评级。
- 一句话结论。
- Top 3 问题。
- 推荐动作。
- 是否可以继续。

高级信息默认折叠：

- 全部分项评分。
- 权重。
- 策略版本。
- 模型信息。
- 原始审稿输出。

## 策略版本记录

每次审稿必须记录当时使用的策略版本。

记录目的：

- 后续解释为什么当时允许继续。
- 策略调整后不误判历史结果。
- 复盘不同策略是否提高小说质量。

页面上默认不展示策略版本，但高级入口可以查看。

## 需求验收标准

- 所有审稿类型使用同一套默认分数档位。
- 低分是否允许继续有明确规则。
- 数据完整性和内容安全强风险不能被分数规则绕过。
- 小白能从审稿结果中看到“能不能继续”和“下一步点什么”。
- 审稿结果能追溯当时的策略版本。

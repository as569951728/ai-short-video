# 系统自我成长完整设计

本文档补齐 `GAP-P2-002`：小说系统如何越用越好。当前阶段不追求模型训练或复杂推荐算法，而是先把生成、审稿、调试、采用、归档、视频表现和用户反馈沉淀为可分析的学习信号，再由系统给出策略、提示词、模型和内容策划优化建议。

系统自我成长不是小白用户每天要操作的主流程，而是产品稳定运行后用于提升小说质量、降低失败率、优化成本和提升视频转化的复盘系统。

## 设计目标

- 让每次生成、审稿、重写、采用、放弃和归档都有可复盘的数据。
- 判断哪些题材、方向、提示词、模型和策略更容易产出好小说。
- 识别常见失败原因，例如开篇弱、爽点重复、AI 味重、中后段水、视频化差。
- 给出可审核的优化建议，而不是自动修改系统核心配置。
- 让后续运营数据回流到小说生成和热点分析，形成闭环。

## 范围边界

自我成长模块负责：

- 收集学习信号。
- 统计质量和运营指标。
- 生成成功案例、失败案例和策略建议。
- 对模型、提示词、题材模板、审稿阈值和推荐动作做效果复盘。
- 为 AI 配置管理后台提供可发布的优化建议。

自我成长模块不负责：

- 初期自动训练大模型。
- 自动发布提示词或策略版本。
- 自动判定用户一定该写什么题材。
- 替代人工最终决策。
- 存储敏感原始提示词、完整模型响应或平台 token。

## 学习信号

学习信号是系统自我成长的基础数据。

### 创作链路信号

| 信号 | 来源 | 用途 |
| --- | --- | --- |
| 方向生成评分 | 方向审稿 | 判断题材、热点和方向模板质量 |
| 方向采用/放弃 | 用户选择 | 判断 AI 推荐是否准确 |
| 方向融合/优化次数 | 方向调试 | 判断初始方向是否稳定 |
| 设定评分 | 设定审稿 | 判断人设和长期冲突质量 |
| 大纲评分 | 大纲审查 | 判断长篇支撑能力 |
| 试写评分 | 试写总评 | 判断开篇和正文策略 |
| 章节重写次数 | 章节工作台 | 判断章节生成质量和提示词稳定性 |
| 审稿问题类型 | 章节/全书审稿 | 发现系统性质量问题 |
| 强制继续原因 | 用户决策 | 判断门禁是否过严或内容判断偏差 |
| 归档原因 | 归档流程 | 判断失败原因 |

### 视频和运营信号

| 信号 | 来源 | 用途 |
| --- | --- | --- |
| 是否创建视频 | 待视频化后动作 | 判断小说视频化吸引力 |
| 首条视频选择范围 | 视频项目 | 判断首条建议是否被采用 |
| 生成失败步骤 | 视频任务 | 优化视频流程和素材配置 |
| 发布状态 | 发布记录 | 判断内容是否进入真实验证 |
| 播放量 | 平台数据或手动录入 | 判断题材和开头吸引力 |
| 完播率 | 平台数据或手动录入 | 判断旁白节奏和钩子 |
| 点赞/评论/关注 | 平台数据或手动录入 | 判断情绪共鸣 |
| 付费/成交 | 手动录入或外部数据 | 判断商业化潜力 |

### 成本和稳定性信号

| 信号 | 来源 | 用途 |
| --- | --- | --- |
| 模型调用成功率 | 生成任务 | 选择稳定模型 |
| 格式错误率 | 输出结构校验 | 优化提示词和修复模板 |
| 平均耗时 | 任务日志 | 优化模型和并发 |
| 平均成本 | 成本记录 | 做成本质量平衡 |
| 重试率 | 任务生命周期 | 发现不稳定任务类型 |

## 学习对象

自我成长模块至少分析以下对象：

- 题材模板。
- 热点报告和创作机会点。
- 小说方向生成策略。
- 设定生成策略。
- 大纲生成策略。
- 章节正文生成提示词。
- 章节审稿提示词。
- 全书审稿提示词。
- 内容安全和原创化规则。
- 模型供应商和具体模型。
- 策略配置版本。
- 视频化推荐规则。

## 指标体系

### 小说质量指标

- 方向采用率。
- 试写通过率。
- 章节平均评分。
- 章节重写率。
- 全书审稿通过率。
- 强制通过率。
- 归档率。
- AI 味问题率。
- 一致性问题率。
- 爽点衰减问题率。

### 内容市场指标

- 热点报告引用率。
- 机会点采用率。
- 题材模板成功率。
- 前三章通过率。
- 待视频化转化率。
- 首条视频创建率。
- 首条视频表现分。

### 模型和提示词指标

- 任务成功率。
- 格式合格率。
- 平均审稿分。
- 用户采用率。
- 低分重写后提升幅度。
- 平均生成耗时。
- 平均成本。
- 单位成本质量分。

### 运营指标

- 视频发布率。
- 播放量。
- 完播率。
- 点赞率。
- 评论率。
- 关注率。
- 付费或成交转化。
- 已发布内容引用异常率。

## 成功案例和失败案例

### 成功案例

成功案例用于沉淀可复用经验。

入选条件示例：

- 试写评分高。
- 全书审稿高分。
- 用户低重写次数完成。
- 首条视频数据好。
- 归档率低、视频化率高的题材模板。

案例内容：

- 小说基础信息。
- 引用热点报告和机会点。
- 题材模板。
- 使用的模型、提示词和策略版本。
- 方向、设定、大纲、试写、全书评分。
- 关键成功标签。
- 可复用经验摘要。
- 不可复用或高风险部分。

### 失败案例

失败案例比成功案例更重要。

入选条件示例：

- 方向多次被放弃。
- 试写低分。
- 章节重写率高。
- 全书审稿阻塞。
- 用户归档。
- 视频发布后表现差。

案例内容：

- 失败阶段。
- 失败原因。
- 关联审稿问题。
- 关联归档原因。
- 使用的模型、提示词和策略版本。
- 是否来自热点误判。
- 建议后续规避方式。

## 优化建议

自我成长模块输出的是建议，不直接修改生产配置。

建议类型：

| 类型 | 示例 |
| --- | --- |
| 题材建议 | 都市逆袭近期试写通过率高，建议提高默认推荐权重 |
| 热点建议 | 某热点报告引用多但归档率高，建议降低置信度 |
| 提示词建议 | 章节正文生成经常解释过多，建议增加少解释、多事件约束 |
| 模型建议 | 某模型章节生成分高但成本高，适合质量优先策略 |
| 审稿建议 | 开篇审稿经常低估用户接受度，建议校准阈值 |
| 视频建议 | 前 1-3 章视频完播低，建议首条改用更强冲突章节 |

建议状态：

| 状态 | 含义 | 可做动作 |
| --- | --- | --- |
| `detected` | 系统发现异常或机会 | 查看证据 |
| `proposed` | 已生成优化建议 | 送审、忽略 |
| `reviewing` | 人工评审中 | 通过、驳回 |
| `approved` | 已确认可应用 | 生成配置草稿 |
| `applied` | 已应用到配置版本 | 查看效果 |
| `rejected` | 不采用 | 记录原因 |
| `archived` | 不再处理 | 查看历史 |

规则：

- 高风险建议不能自动应用，例如提高自动通过率、降低安全规则、发布提示词模板。
- 应用建议时要生成配置草稿，再走 AI 配置管理后台的测试、发布和回滚流程。
- 每个建议必须带证据，不允许只给结论。

## 页面设计

### 数据复盘首页

展示：

- 小说生产总览。
- 质量趋势。
- 失败原因排行。
- 题材表现排行。
- 模型和提示词效果摘要。
- 视频表现摘要。
- 最新优化建议。

小白模式不默认展示该页面；高级/管理员可见。

### 成功案例库

字段：

- 案例名称。
- 题材。
- 关键成功标签。
- 试写评分。
- 全书评分。
- 视频表现。
- 使用策略版本。
- 可复用经验。

### 失败案例库

字段：

- 案例名称。
- 失败阶段。
- 失败原因。
- 问题标签。
- 关联版本。
- 归档原因。
- 建议规避方式。

### 优化建议列表

字段：

- 建议标题。
- 建议类型。
- 影响模块。
- 证据摘要。
- 预期收益。
- 风险等级。
- 状态。
- 推荐动作。

### 模型和提示词效果

展示：

- 按任务类型的模型效果。
- 提示词版本效果。
- 策略版本效果。
- 成本和质量对比。
- 样本量和置信度。

## 数据规则

- 所有学习信号必须绑定租户、用户、业务对象和版本。
- 不记录完整 API Key、完整提示词、完整模型响应、平台 token。
- 低样本量数据不能直接生成强建议，必须标记“样本不足”。
- 用户手动反馈需要支持“好看、不好看、哪里怪、想继续看、适合视频、不适合视频”等简单选项。
- 系统建议必须记录来源指标、统计周期和样本量。

## 和其他模块的关系

- 热点系统：接收热点报告和机会点的实际使用效果。
- 小说系统：接收方向、设定、大纲、章节、审稿和归档信号。
- 视频系统：接收发布和数据表现。
- AI 配置管理后台：把优化建议转为提示词、策略或模型配置草稿。
- 权限租户：数据复盘和配置建议属于高级能力，需要权限控制。

## 实施分期建议

1. 先记录学习信号和归档原因，形成可查询数据。
2. 做基础数据复盘页：质量趋势、失败原因、模型效果、题材表现。
3. 做成功/失败案例库和优化建议列表。
4. 接入视频发布数据和热点效果回流。
5. 将建议和 AI 配置管理后台打通，支持生成配置草稿。

## 验收口径

- 系统能记录方向采用、试写结果、章节重写、审稿问题、强制通过和归档原因。
- 能按题材、模型、提示词、策略版本统计质量和失败原因。
- 能生成带证据的优化建议。
- 优化建议不能自动改生产配置，必须进入人工确认。
- 视频数据和热点引用效果能回流到复盘体系。

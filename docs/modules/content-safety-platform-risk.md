# 内容安全与平台风险

本文档补齐 `GAP-P1-008`：小说和后续视频内容可能涉及内容安全、平台规则、原创化和版权风险。系统需要在生成、审稿、完成确认和视频化前给出风险提示和阻塞规则。

本文档只定义需求规则，不提供法律意见，也不绑定某个短视频平台的实时规则。平台规则可能变化，后续需要通过可配置规则持续更新。

## 设计目标

- 避免生成明显不适合发布或变现的内容。
- 避免过度贴近已有作品或热点素材。
- 让用户知道风险来自哪里。
- 高风险内容不能直接进入视频化。
- 风险提示要通俗，不吓退正常创作。

## 风险类型

| 风险类型 | 说明 |
| --- | --- |
| 内容安全风险 | 暴力、色情、未成年人、仇恨、诈骗、违法等风险 |
| 平台发布风险 | 可能不适合短视频平台发布或审核 |
| 原创化风险 | 过度像已有作品、热点素材或搬运内容 |
| 版权风险 | 直接使用真实作品名、角色名、台词或设定 |
| 价值观风险 | 过度宣扬违法、伤害、歧视或极端行为 |
| 商业转化风险 | 内容可能影响后续带货、付费观看或账号安全 |

## 风险等级

| 等级 | 含义 | 默认处理 |
| --- | --- | --- |
| `info` | 轻微提醒 | 不阻塞，只提示 |
| `warning` | 有明显风险 | 建议修改，可接受风险继续 |
| `blocking` | 高风险 | 阻塞下一步，必须处理 |

内容安全强风险属于 `blocking`，不能通过“接受风险继续”绕过。

## 检查时机

内容安全与平台风险检查应出现在：

- 小说方向生成后。
- 设定档案生成后。
- 大纲和章节目录生成后。
- 章节正文生成后。
- 全书质量检查时。
- 待视频化检查时。
- 视频项目创建前。

不要求每次都展示完整报告，但风险摘要要能追溯。

## 原创化风险

系统需要避免直接复制热点素材。

风险场景：

- 书名过度像已有作品。
- 主角设定、核心冲突、世界观高度相似。
- 使用真实作品角色名。
- 直接复用短视频文案或爆款标题。
- 多个方向只是换名字，没有明显差异。

处理动作：

- 重新生成更差异化方向。
- 调整主角开局。
- 调整世界背景。
- 调整核心冲突。
- 降低对热点素材的直接引用。

## 平台发布风险

平台风险需要通俗提示：

- 这类内容可能不适合直接发布。
- 这段内容建议弱化表达。
- 这个设定可能影响后续投放或变现。
- 建议改成更安全的表达方式。

系统不承诺平台一定通过审核，只提供风险提醒。

## 风险展示规则

小白默认展示：

- 风险等级。
- 一句话说明。
- 影响范围。
- 推荐处理动作。

高级入口可展示：

- 命中的规则。
- 风险类型。
- 原始片段摘要。
- 处理记录。
- 规则版本。

不要在默认页面展示过多敏感内容原文。

## 阻塞规则

以下风险必须阻塞：

- 明显违法或危险行为指导。
- 涉及未成年人不当内容。
- 明显色情或极端暴力内容。
- 明显仇恨或歧视内容。
- 明显诈骗或诱导违法内容。
- 高度疑似搬运或直接复刻已有作品。

阻塞后推荐动作：

- 返回设定调整。
- 返回大纲调整。
- 重写风险章节。
- 重新生成更安全版本。

## 和审稿的关系

内容安全不是普通审稿分数的一部分。

规则：

- 普通质量分高，不代表内容安全通过。
- 内容安全强风险可以单独阻塞。
- 全书质量检查和待视频化检查都需要读取最新风险结果。
- 风险规则版本需要记录，便于后续解释。

## 和视频化的关系

待视频化前必须检查平台和内容风险。

如果存在 `blocking` 风险：

- 不能确认待视频化。
- 不能创建视频项目。
- 必须先处理风险内容。

如果存在 `warning` 风险：

- 可以提示用户接受风险继续。
- 必须记录原因。

## 需求验收标准

- 方向、设定、大纲、章节、全书和视频化前都有风险检查入口。
- 风险等级能决定是否阻塞。
- 内容安全强风险不能被质量分或接受风险绕过。
- 原创化风险能提示用户调整方向。
- 风险提示有通俗说明和推荐处理动作。

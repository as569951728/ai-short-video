# 成本与资源提醒

本文档补齐 `GAP-P1-007`：长篇小说生成会消耗模型 token、时间、并发资源和本地/服务器资源，系统需要在合适的时机提醒用户成本和耗时风险，但不能让小白被技术细节淹没。

本文档只定义需求规则，不讨论具体实现方案。

## 设计目标

- 长任务开始前让用户知道大概会花多久、消耗多少。
- 高成本、高失败风险任务需要提醒。
- 小白默认不看 token 和模型细节。
- 高级用户可以查看成本和资源摘要。
- 成本异常可以用于复盘和优化。

## 用户默认看到什么

小白默认只看到：

- 预计耗时。
- 当前进度。
- 是否可能比较耗资源。
- 失败后是否建议缩短范围或换模型。

默认不展示：

- 详细 token。
- 单价。
- 完整模型参数。
- 完整成本公式。

## 需要提醒的场景

以下场景需要成本或资源提醒：

- 批量生成正文。
- 长章节生成。
- 全书 AI 审稿。
- 批量重写受影响章节。
- 重新生成大量章节。
- 视频音频、字幕、渲染任务。
- 多次失败后重试。
- 选择高质量模型或长上下文模型。

## 提醒等级

| 等级 | 含义 | 展示方式 |
| --- | --- | --- |
| 普通 | 常规生成，无明显风险 | 任务进度中展示预计耗时 |
| 提醒 | 可能耗时较久或消耗较多 | 抽屉提示 |
| 警告 | 大批量、长文本或多次重试 | 需要用户确认 |
| 阻塞 | 超过系统限制或配置不可用 | 不允许继续，说明原因 |

阻塞只用于明确限制，例如模型不可用、额度不足、并发已满、文本超过上限。

## 提醒内容

提醒应使用小白文案：

- 这次可能需要几分钟。
- 这次会生成较多章节，建议保持页面打开查看进度。
- 这章内容较长，系统会分段生成。
- 已失败多次，建议缩短范围或换模型。
- 当前任务较多，需要排队。

高级入口可展示：

- 预计输入 token。
- 预计输出 token。
- 实际输入 token。
- 实际输出 token。
- 预计成本。
- 实际成本。
- 模型名称。
- 耗时。
- 失败率。

## 超长章节规则

每章系统上限为 20,000 字，但常规建议为 2,000-3,500 字。

当章节目标过长时：

- 提示“本章较长，系统会分段生成”。
- 提示可能增加耗时。
- 允许用户继续。
- 如果超过系统上限，必须阻塞并要求调整目标字数。

## 批量任务规则

批量任务需要展示：

- 总章节数。
- 已完成章节数。
- 当前正在处理章节。
- 预计剩余时间。
- 失败章节数量。
- 是否可暂停或取消。

批量任务失败时，不应让用户误以为全部失败。需要区分：

- 已完成。
- 失败。
- 未开始。
- 待确认。

## 重试成本提醒

以下重试需要提醒：

- 全书审稿重试。
- 批量正文重试。
- 大量章节重写。
- 视频渲染重试。
- 已连续失败多次的任务。

提醒文案：

- “这次重试会重新调用 AI，可能继续消耗额度。”
- “建议先查看失败原因，再决定是否重试。”

## 资源拥堵提醒

当系统任务较多时：

- 显示排队中。
- 显示预计开始时间或排队说明。
- 不允许重复点击创建同类任务。
- 提供取消排队入口。

不要让用户误以为按钮没反应。

## 需求验收标准

- 长任务开始前有耗时或资源提示。
- 批量任务能显示总数、已完成、失败和当前处理对象。
- 多次重试会提示可能继续消耗资源。
- 小白默认不看到复杂 token 和模型细节。
- 高级入口能查看成本和资源摘要。

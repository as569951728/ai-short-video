# 章节重写影响处理

本文档补齐 `GAP-P0-010`：章节重写、局部优化、手动编辑或采用候选版本后，系统如何判断对后续章节、长篇记忆、伏笔和视频引用的影响，并让用户选择安全的处理方式。

章节重写影响处理的核心原则是：不要因为改好当前章，悄悄破坏后面的剧情。

## 设计目标

- 让用户知道“当前章改动会不会影响后面”。
- 把轻微、中等、严重影响的处理流程标准化。
- 避免状态膨胀，影响等级和状态备注不变成新的主状态。
- 保留三种用户选择：自动调整、逐章处理、清空后续重来。
- 所有高风险操作都可追溯、可解释、可回退到历史版本。

## 触发场景

以下场景必须触发影响评估：

- 采用章节候选正文版本。
- 手动编辑章节并保存为正式版本。
- 局部重写改变关键事实、人物关系、伏笔或结尾钩子。
- 恢复旧章节版本。
- 调整章节摘要/特性卡片并影响正文。
- 修改已被视频引用的章节正文。

以下场景可以不触发完整影响评估，只记录轻量同步：

- 修正错别字、标点、排版。
- 不改变剧情事实的少量表达润色。
- 审稿报告或摘要文字的展示修正。

是否免评估必须由策略和差异检查共同判断，不能只由前端决定。

## 影响评估输入

AI 影响评估任务至少读取：

- 小说设定档案当前版本。
- 全书大纲、阶段大纲、章节目录当前版本。
- 当前章旧版本和新版本。
- 当前章章节摘要/特性卡片旧版本和新版本。
- 后续章节目录。
- 已生成后续章节摘要、正文版本和审稿报告。
- 长篇记忆、人物状态、关系状态、伏笔状态。
- 视频引用范围和发布状态。

输入给模型时应使用摘要和结构化差异，避免把整本正文全部塞入上下文。

## 影响评估输出

影响评估必须输出结构化结果：

| 字段 | 说明 |
| --- | --- |
| `impactLevel` | `none`、`minor`、`medium`、`severe` |
| `summary` | 小白能看懂的一句话结论 |
| `changedFacts` | 被改变的关键事实 |
| `affectedChapterIds` | 受影响章节范围 |
| `affectedMemoryItems` | 受影响的人物、关系、伏笔、地点或道具 |
| `videoReferenceImpact` | 是否影响已生成或已发布视频 |
| `recommendedHandling` | 推荐处理方式 |
| `blocksCompletion` | 是否阻塞小说完成 |
| `riskNotes` | 风险说明 |

影响评估报告自动保存，但不能自动执行高风险处理。

## 影响等级

| 等级 | 含义 | 主状态处理 | 是否阻塞完成 |
| --- | --- | --- | --- |
| 无影响 `none` | 只改表达，不影响事实和后文 | 当前章回到 `normal` | 否 |
| 轻微影响 `minor` | 改动局部情绪、表达或细节，可同步摘要解决 | 当前章回到 `normal`，备注可清空 | 否 |
| 中等影响 `medium` | 影响部分后续章节，但不需要推翻全书 | 受影响章节标记 `pending` | 是 |
| 严重影响 `severe` | 破坏后续主线、关键关系、伏笔或大量正文 | 当前影响记录进入待决策 | 是 |

规则：

- “轻微影响”和“受影响未调整”只作为备注或影响记录，不新增章节主状态。
- 章节主状态只保留 `normal`、`pending`、`processing`、`resolved`。
- 当章节重新变为 `normal` 时，当前 `statusNote` 可以清空。
- 历史影响原因保存在影响评估、版本记录和操作日志中。

## 轻微影响处理

轻微影响默认由系统自动同步：

- 更新当前章节摘要/特性卡片。
- 更新长篇记忆。
- 更新后续生成上下文。
- 刷新章节和小说推荐动作。

用户看到：

- “这次修改只影响摘要和记忆，系统已同步。”
- 可查看同步详情。

轻微影响不阻塞小说完成，也不需要用户逐章处理。

## 中等影响处理

中等影响表示后续部分章节需要调整，但不需要整体推翻。

系统展示：

- 受影响章节列表。
- 每章受影响原因。
- 影响的关键事实、人物状态、关系或伏笔。
- 推荐处理方式。
- 是否涉及视频引用。

用户可选动作：

| 动作 | 说明 | 结果 |
| --- | --- | --- |
| 生成调整方案 | AI 生成每章如何修的建议 | 生成处理方案，不覆盖正文 |
| 批量生成候选修正 | 为受影响章节批量生成候选版本 | 候选待确认 |
| 逐章处理 | 进入章节详情工作台逐章修 | 每章独立闭环 |
| 手动确认无影响 | 用户判断某些章节无需处理 | 需要填写原因 |

规则：

- 受影响章节主状态改为 `pending`，`statusNote=受影响未调整`。
- 批量生成只能生成候选版本，不能静默覆盖正式正文。
- 每章处理完成后进入 `resolved` 或 `normal`，是否需要复审由策略决定。
- 所有受影响章节处理完成前，小说不能进入完成确认。
- 用户手动确认无影响必须写操作日志。

## 严重影响处理

严重影响必须给用户三种选择：

1. 清空当前章之后的正文，后续章节重新生成。
2. 不清空正文，将所有受影响章节标记为 `pending` 后逐章处理。
3. 返回修改当前重写内容，让当前章尽量兼容后续章节。

### 选择 1：清空后续正文

适用：

- 当前章改动让后续主线大面积失效。
- 后续正文质量本身一般，重新生成成本可接受。
- 还没有视频引用，或用户接受视频引用异常。

规则：

- 必须二次确认并填写原因。
- 必须展示清空范围、章节数、字数、审稿情况、视频引用情况。
- 不删除历史版本，只取消后续章节当前正文引用或标记为不再当前使用。
- 后续章节正文状态变为缺失或待生成。
- 长篇记忆回滚到当前章后的安全点。

### 选择 2：逐章处理

适用：

- 后续正文已有可用价值。
- 用户不想重来。
- 视频引用或历史版本需要保留。

规则：

- 所有受影响章节标记 `pending`。
- 系统生成影响处理清单。
- 用户逐章进入章节详情处理。
- 处理完成前不能进入完成确认或待视频化。

### 选择 3：返回修改当前候选版本

适用：

- 当前章改动质量高，但可以微调以兼容后文。
- 用户不想动后续章节。
- 严重影响主要来自少数关键事实冲突。

规则：

- 当前候选版本不应直接采用。
- 系统基于影响报告生成“兼容后文优化建议”。
- 生成新的候选版本后重新复审和影响评估。

## 影响处理状态

建议影响记录使用以下状态：

| 状态 | 含义 |
| --- | --- |
| `assessing` | 正在评估影响 |
| `waiting_decision` | 已评估，等待用户选择处理方式 |
| `handling` | 已选择方案，正在处理 |
| `resolved` | 影响已处理完成 |
| `cancelled` | 用户放弃候选版本或回退到旧版本 |

该状态属于 `ImpactAssessment` 或 `ChapterImpactCase`，不是章节主状态。

## 完成判断

一次影响处理可以关闭，至少满足：

- 影响评估任务完成。
- 中等或严重影响已经选择处理方案。
- 所有受影响章节都不再是 `pending` 或 `processing`。
- 需要复审的章节已复审通过，或用户已确认风险。
- 长篇记忆已同步到新的当前版本。
- 视频引用异常已标记或确认不受影响。
- 高风险操作已有操作日志和原因。

关闭后，小说推荐动作重新计算。

## 页面交互

章节详情工作台展示影响评估和处理区。

小说详情工作台展示项目级影响摘要：

- 当前是否存在未处理影响。
- 影响来自哪一章。
- 受影响章节数量。
- 是否阻塞完成。
- 主推荐动作：处理受影响章节。

中等和严重影响需要有独立抽屉或页面区块，不能只用一个弹窗承载。

## 数据与接口边界

建议复用或新增对象：

- `ImpactAssessment`：影响评估报告。
- `ChapterImpactCase`：一次影响处理案例，用于管理决策和处理进度。
- `ChapterImpactItem`：受影响章节明细。
- `OperationLog`：高风险操作日志。

建议接口：

- `POST /novels/:novelId/chapters/:chapterId/impact-assessments`：发起影响评估。
- `GET /novels/:novelId/impact-cases/:impactCaseId`：查看影响处理案例。
- `POST /novels/:novelId/impact-cases/:impactCaseId/resolve`：首期关闭、忽略或取消影响案例。
- `POST /novels/:novelId/impact-cases/:impactCaseId/decide`：提交处理选择，后续扩展。
- `POST /novels/:novelId/impact-cases/:impactCaseId/generate-plan`：生成调整方案，后续扩展。
- `POST /novels/:novelId/impact-cases/:impactCaseId/generate-candidates`：批量生成候选修正，后续扩展。
- `POST /novels/:novelId/impact-cases/:impactCaseId/clear-after-chapter`：清空后续当前正文引用，后续扩展且必须二次确认。
- `POST /novels/:novelId/impact-cases/:impactCaseId/confirm-no-impact`：手动确认无影响，后续扩展。

清空后续章节、手动确认无影响、批量生成候选修正必须做幂等和版本校验。

## 和其他模块的关系

- 章节详情工作台：承载单章影响评估和受影响章节处理。
- 小说详情工作台：展示影响处理摘要和阻塞原因。
- AI 任务生命周期：影响评估和处理方案生成都通过任务管理。
- AI 产物确认：批量修正只能生成候选版本。
- 全书审稿门禁：存在未处理影响时，全书审稿可以查看，但不能确认完成。
- 待视频化判定：存在未处理影响或视频引用异常时，不能进入待视频化。

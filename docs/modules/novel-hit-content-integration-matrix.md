# 爆款内容策划落点矩阵

本文档承接 `GAP-P2-006`，用于说明 `docs/modules/novel-hit-content-planning.md` 中的题材模板、爆款拆解、开篇黄金段、爽点节奏、人设关系、伏笔台账、故事圣经和短视频旁白单元，如何落到小说创建和生产链路里的具体产物、页面展示、AI 审稿和用户动作。

它不是新的业务流程，也不是新的后台页面。它是一张贯穿方向、设定、大纲、章节、试写、全书审稿和待视频化的执行矩阵，避免内容策划只停留在“要爆款、要好看”的抽象要求上。

## 设计目标

- 让每个创作阶段都能明确引用哪些内容策划标准。
- 让 AI 生成结果有结构化字段承接，而不是只生成一段文字。
- 让 AI 审稿能按同一套内容标准扣分、给建议和阻塞。
- 让小白用户看到的是结论、风险和下一步动作，而不是专业创作术语。
- 为后续热点系统、自我成长系统和视频运营复盘沉淀统一标签。

## 设计取舍

### 方案 A：把字段分散补到各模块文档

优点是改动少；缺点是标准容易散落，后续提示词、审稿、页面和数据结构可能各做各的。

### 方案 B：建立独立落点矩阵

优点是所有阶段共用一张映射关系，后续研发、提示词设计、测试用例和原型检查都能按矩阵核对；缺点是多一个需求文档需要维护。

### 方案 C：等真实运营数据回来后再补

优点是更接近真实市场；缺点是前期没有统一内容标尺，系统可能先生产大量结构完整但不好看的小说。

推荐采用方案 B。当前阶段先定义完整落点，研发实现时再裁剪优先级。

## 小说方向卡落点

方向卡不是简单题材选择，而是小说是否值得继续投入的第一道内容判断。

### 强化字段

- `templateRefs`：引用的主题材模板和辅助题材模板。
- `benchmarkStructureRefs`：引用的爆款结构摘要，不保存原文。
- `readerEmotionPromise`：给读者的核心情绪承诺，例如“被看不起后连续反击”。
- `openingHookType`：开篇钩子类型，例如身份反差、危机压迫、强反转、强羞辱、秘密揭露。
- `firstThreeChapterPromise`：前三章预计兑现的第一个爽点。
- `firstVideoPotential`：是否适合做首条短视频验证。
- `commercialFit`：后续做流量或商品代理的匹配倾向。
- `differentiationSummary`：和对标素材的差异化说明。
- `homogenizationRisk`：同质化或抄袭感风险。
- `recommendedTestSlice`：建议后续优先验证的章节范围或剧情单元。

### 小白展示

默认只展示：

- 这个方向一句话。
- 为什么可能有人看。
- 前三章预计有什么看点。
- 适不适合先做视频测试。
- 最大风险和系统推荐动作。

### 审稿与动作

方向分低时，系统优先给出：

- 换题材模板。
- 调整主角开局困境。
- 强化开篇钩子。
- 增加前三章爽点承诺。
- 降低对标相似度。
- 重新生成方向。

如果方向没有清晰情绪承诺或前三章爽点承诺，不建议进入设定档案生成。

## 设定档案落点

设定档案负责把方向里的“看点”变成能支撑长篇的角色、关系、世界规则和冲突。

### 强化字段

- `characterRelationshipMatrix`：重要人物之间的关系张力。
- `appealUpgradePlan`：爽点从前期到后期如何升级。
- `storyBibleSeeds`：初始故事圣经事实。
- `foreshadowingPlan`：开局、中期、终局伏笔计划。
- `commercialFitNotes`：适合后续短视频和商业化表达的内容点。
- `midStoryWaterRisk`：中后段可能变水的原因。
- `originalityGuardrails`：避免过度像对标素材的规则。

### 小白展示

默认只展示：

- 主角是谁。
- 主角开局为什么惨或为什么有吸引力。
- 读者为什么愿意追。
- 这套设定能不能写长。
- 最大同质化风险和中后段风险。

### 审稿与动作

设定审稿应增加：

- 主角欲望是否足够强。
- 反派或阻力是否能持续升级。
- 人物关系是否有戏剧张力。
- 爽点是否有长期升级空间。
- 故事圣经初始事实是否稳定。

设定不通过时，推荐动作应优先指向“强化主角欲望”“强化反派压迫”“补人物关系张力”“重建爽点升级”，而不是泛泛地“优化设定”。

## 大纲和章节目录落点

大纲负责把内容策划标准转成可执行的长期生产计划。

### 全书大纲

需要强化：

- `stageAppealUpgrade`：各阶段爽点升级路线。
- `longConflictEngine`：支撑长篇的长期冲突发动机。
- `emotionalPeakPlan`：每个阶段的情绪峰值。
- `foreshadowingRoadmap`：伏笔埋设、推进和回收路线。
- `waterRiskByStage`：每个阶段可能变水的原因。

审稿重点：

- 是否能支撑目标章节数。
- 是否只有开头强、中后段弱。
- 阶段之间是否只是扩大同一种冲突。
- 终局回报是否和开篇情绪承诺一致。

### 阶段大纲

需要强化：

- 阶段主痛点。
- 阶段主爽点。
- 阶段反派压迫升级。
- 阶段关系变化。
- 阶段结尾诱饵。
- 和上一阶段的冲突差异。

审稿重点：

- 阶段目标是否清楚。
- 情绪回报是否足够。
- 是否连续多个阶段重复同一套路。
- 是否给后续章节留下明确生产空间。

### 章节目录

每章需要强化：

- `painPoint`：本章痛点。
- `pressure`：本章压迫。
- `counterattack`：本章反击。
- `appealPoint`：本章爽点。
- `twist`：本章反转或变化。
- `emotionPayoff`：本章情绪回报。
- `slicePotential`：是否适合短视频切片。
- `emotionPromise`：本章不能丢失的情绪承诺。
- `conflictDifferenceFromNeighbors`：和前后章的冲突差异。

章节目录不通过时，系统应给出“重写某几章目录”“调整阶段节奏”“补某章爽点”“替换重复冲突”等动作。

## 试写总评落点

试写是最关键的早期质量闸口，必须把市场感和正文手感同时纳入判断。

### 强化字段

- `openingGoldenSegmentScores`：首句、前 300 字、前 1 分钟旁白、第一章结尾、前三章爽点兑现评分。
- `firstVideoHook`：首条视频的前 3 秒旁白钩子。
- `firstScreenSubtitle`：首条视频建议首屏字幕。
- `firstThreeChapterPayoff`：前三章是否兑现第一轮情绪回报。
- `ttsRisk`：转语音后的听感风险。
- `batchGenerationReadiness`：是否适合进入批量生成。
- `returnUpstreamReason`：需要返回方向、设定、大纲或章节目录的原因。

### 小白展示

默认只展示：

- 前三章是否好看。
- 开头最需要改哪里。
- 能不能进入批量生成。
- 建议先做的一个动作。

### 审稿与动作

低分时优先动作：

- 优化首句。
- 重写前 300 字。
- 增强第一轮压迫。
- 提前第一次反击。
- 压缩设定解释。
- 强化第一章结尾钩子。
- 返回章节目录调整前三章。

如果开篇黄金段低分，不建议进入批量正文生成；用户强制继续必须填写原因。

## 章节详情工作台落点

章节详情工作台需要把内容策划标准转成单章可处理问题。

### 章节摘要/特性卡强化

在已有字段基础上，章节卡需要额外保留：

- 本章痛点。
- 本章压迫方式。
- 本章反击方式。
- 本章情绪回报。
- 本章切片点。
- 本章和前后章节的差异。
- 本章不能丢失的情绪承诺。

### 问题卡映射

| 审稿问题 | 小白文案 | 默认动作 |
| --- | --- | --- |
| 开头钩子弱 | 开头不够抓人，用户可能划走 | 优化开头 |
| 爽点重复 | 这一章的爽点和前面太像 | 调整本章反击方式 |
| 只有压迫没有回报 | 读者一直憋屈，看不到回报 | 补充情绪回报 |
| 解释过多 | 讲设定太多，事件推进太少 | 压缩解释性内容 |
| 结尾钩子弱 | 看完这一章不太想继续看 | 强化结尾悬念 |
| 故事圣经冲突 | 本章和前面已确认事实不一致 | 修复事实冲突 |
| 切片价值弱 | 这一章不太适合做短视频 | 标记不优先视频化或强化冲突 |
| AI 腔明显 | 读起来像模板话，不像真实故事 | 降低 AI 腔 |
| 主角代入感弱 | 主角目标和行动力不够，读者不容易站他 | 强化主角代入感 |

所有优化结果都进入候选版本，不能静默覆盖正式正文。

### 章节局部编辑动作

章节详情工作台应把审稿问题转成具体编辑动作，而不是只给“优化章节”。

可选动作：

- 删解释。
- 补事件。
- 换对话。
- 提前反击。
- 强化结尾钩子。
- 降低 AI 腔。
- 强化主角代入感。
- 爽点去重。

小白默认只看到 1 个系统推荐动作；其余动作折叠为可选项。

## 全书审稿落点

全书审稿必须从“完整性审稿”升级为“可投放前的内容质量门禁”。

### 强化维度

- `openingGoldenSegmentFinal`：最终开篇黄金段质量。
- `midLateAppealDecay`：中后段爽点衰减。
- `conflictRepetition`：冲突和打脸方式重复度。
- `foreshadowingClosure`：伏笔回收完整度。
- `storyBibleConflicts`：故事圣经冲突。
- `narrationUnitAvailability`：可用旁白单元数量。
- `benchmarkSimilarityRisk`：对标相似度和原创化风险。
- `videoTestRecommendation`：首条、测试条、高潮条推荐。
- `midStageReviewSignals`：阶段审稿发现的中段变水、重复和主角目标弱化信号。

### 小白展示

默认只展示：

- 这本小说能不能完成。
- 是否建议进入视频化。
- 最影响流量的 1-3 个问题。
- 系统建议先修哪里。

### 门禁规则

- 开篇黄金段严重低分时，不建议直接待视频化。
- 中后段大面积重复时，应建议返回阶段或章节批量优化。
- 伏笔未回收或故事圣经冲突属于完成门禁问题。
- 视频化适配低分不一定阻塞小说完成，但会影响待视频化建议。

## 待视频化落点

待视频化检查不只判断内容能不能引用，还要给运营测试一个可执行的起点。

### 强化字段

- `recommendedFirstVideoUnit`：推荐首条视频单元。
- `testUnit`：适合做测试的章节范围。
- `highlightUnit`：适合做高潮条的章节范围。
- `followUpUnit`：适合做追更条的章节范围。
- `titleHook`：标题钩子。
- `firstThreeSecondNarration`：前 3 秒旁白。
- `firstScreenSubtitle`：首屏字幕。
- `endingCliffhanger`：结尾悬念。
- `retentionRisk`：留存风险。

### 小白展示

默认只展示：

- 建议先做哪几章。
- 第一条视频开头怎么说。
- 视频标题可以突出什么。
- 结尾用哪个悬念。
- 如果内容不适合视频，原因是什么。

## 归档和复盘落点

归档原因是后续系统自我成长的重要输入，不能只写“不要了”。

建议归档原因映射到内容策划维度：

| 归档原因 | 归属维度 | 后续用途 |
| --- | --- | --- |
| 题材不吸引 | 题材模板 | 调整题材推荐权重 |
| 开头不好看 | 开篇黄金段 | 优化开篇提示词和审稿阈值 |
| 爽点弱 | 爽点节奏 | 优化章节目录和正文生成策略 |
| 主角不讨喜 | 人设关系 | 优化主角欲望和代入感规则 |
| 写不长 | 长篇支撑 | 优化大纲阶段规划 |
| 太像别的作品 | 对标拆解 | 提高原创化和差异化约束 |
| AI 味重 | 表达和样例 | 优化文风样例和审稿标准 |
| 不适合视频 | 旁白单元 | 优化视频化倾向判断 |

用户归档时小白默认选择通俗原因，高级记录映射维度和关联产物版本。

## 数据沉淀字段

各阶段至少要沉淀以下字段，供后续系统自我成长使用：

- 题材模板版本。
- 对标拆解版本。
- 方向卡评分和采用情况。
- 开篇黄金段评分。
- 前三章爽点兑现评分。
- 章节痛点、反击、爽点和钩子标签。
- 章节被重写、采用、放弃和强制确认记录。
- 全书审稿中后段衰减评分。
- 视频化推荐单元和用户是否采用。
- 归档原因和用户主观反馈。
- 后续视频播放、完播、点赞、评论、关注、付费或成交数据。

## 页面展示原则

小白默认视图只展示三类信息：

- 当前结论：好不好、能不能继续、值不值得做视频。
- 核心原因：最多 1-3 条，用通俗话说明。
- 下一步动作：系统推荐一个主动作，保留少量可选动作。

高级入口才展示：

- 题材模板详情。
- 对标拆解摘要。
- 爽点/痛点节奏表。
- 人设关系矩阵。
- 伏笔回收台账。
- 故事圣经。
- 短视频旁白单元。
- 分项评分和策略版本。

## 不做的内容

当前需求设计不把以下能力放进主路径：

- 自动抓取真实平台热点。
- 自动学习并改写题材模板权重。
- 复杂素材库管理。
- 专业编辑模式的深度剧作诊断。
- 自动生成平台投放策略。
- 在页面暴露完整提示词或完整模型响应。

这些能力可以在热点系统、自我成长系统、AI 配置管理后台和视频运营模块中继续细化。

## 验收口径

需求和原型层面满足以下条件即可认为本矩阵落地：

- 方向卡能展示市场理由、开篇钩子、前三章承诺和差异化风险。
- 设定档案能说明读者为什么追、能不能写长和最大中后段风险。
- 大纲能展示阶段升级、伏笔路线和重复风险。
- 章节目录或章节卡能记录痛点、压迫、反击、爽点、钩子和情绪承诺。
- 试写总评能按开篇黄金段给出具体问题和动作。
- 章节详情问题卡能把专业审稿问题翻译成小白可执行动作。
- 全书审稿能识别爽点衰减、冲突重复、故事圣经冲突和视频化潜力。
- 待视频化能给出首条视频单元、前 3 秒旁白、首屏字幕和留存风险。
- 归档原因能反向映射到内容策划维度。

## 和其他模块的关系

- 爆款内容策划体系：本矩阵是 `docs/modules/novel-hit-content-planning.md` 的执行层补充。
- 小说方向生成：方向卡字段需要按本矩阵强化。
- 设定档案：设定摘要和审稿需要引用人设关系、爽点升级和长篇风险。
- 大纲三层结构：全书、阶段和章节目录需要引用阶段升级、冲突去重和伏笔路线。
- 试写闭环：试写总评需要输出开篇黄金段和首条视频建议。
- 章节详情工作台：章节问题卡和一键重写需要引用章节卡的情绪承诺。
- 全书审稿门禁：完成前需要按本矩阵判断内容质量和视频化价值。
- 待视频化判定：视频承接需要消费本矩阵中的旁白单元和首条建议。
- 系统自我成长：本矩阵沉淀的标签、评分和归档原因是后续学习信号来源。

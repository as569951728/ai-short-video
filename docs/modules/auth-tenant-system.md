# 权限与租户完整设计

本文档补齐 `GAP-P2-004`：系统从单用户本地工具，扩展到家庭/小团队使用、打包售卖和后续 SaaS 化时，需要怎样设计用户、角色、租户、授权、数据隔离、操作日志和高风险权限。

权限和租户不是当前小说主链路的研发重点，但核心数据和后台导航必须提前预留边界，避免后续产品化时大面积返工。

## 设计目标

- 当前可以按单用户 owner 模式运行，降低初期复杂度。
- 核心业务数据从第一天开始预留 `tenantId`、`ownerId`、`createdBy`、`updatedBy`。
- 高风险操作要有权限、确认和原因记录。
- 后续支持给家人、合伙人、运营人员或客户开账号。
- 为系统打包售卖、授权到期、功能开关和数据隔离预留能力。

## 运行形态

| 形态 | 说明 | 权限复杂度 |
| --- | --- | --- |
| 单用户本地版 | 用户自己在本地或 Mac mini 上使用 | 默认 owner，最简单 |
| 家庭/小团队版 | 用户和家人或少量协作者共同使用 | 需要角色和操作边界 |
| 打包售卖版 | 每个客户独立部署一套系统 | 需要授权和默认租户 |
| SaaS 版 | 多租户共享一套服务 | 需要严格数据隔离和租户管理 |

当前研发可以优先支持单用户本地版，但需求设计按完整形态保留。

## 核心对象

### 租户

租户代表一套独立的数据空间。

字段建议：

- 租户名称。
- 租户类型：个人、家庭团队、客户部署、SaaS 客户。
- 状态：正常、暂停、到期、禁用。
- 授权状态。
- 授权到期时间。
- 功能套餐。
- 资源额度。
- 创建时间。

规则：

- 单用户模式使用默认租户。
- 普通业务查询默认按租户隔离。
- 跨租户操作只允许超级管理员或部署运维角色。

### 用户

字段建议：

- 用户名。
- 昵称。
- 手机或邮箱。
- 登录账号。
- 所属租户。
- 状态：正常、禁用、待激活。
- 默认角色。
- 最近登录时间。

规则：

- 单用户模式可以使用默认 owner 用户。
- 后续支持邀请用户加入租户。
- 禁用用户后不能登录，但历史操作日志保留。

### 角色

建议角色：

| 角色 | 定位 |
| --- | --- |
| owner | 系统拥有者，拥有全部权限 |
| admin | 管理员，可配置模型、策略、用户和大部分业务 |
| creator | 内容创作者，可创建和管理小说、视频 |
| reviewer | 审稿/质检人员，可查看审稿、处理问题和确认风险 |
| operator | 运营人员，可管理热点、视频发布和数据回流 |
| viewer | 只读查看 |

单用户模式下默认使用 `owner`。

### 权限点

权限点建议按模块和动作拆分。

模块：

- 热点系统。
- 小说系统。
- 章节工作台。
- 视频系统。
- 生成任务。
- 数据复盘。
- AI 配置。
- 系统设置。
- 用户和租户。

动作：

- 查看。
- 创建。
- 编辑。
- 删除。
- 归档。
- 发布。
- 强制通过。
- 清空后续章节。
- 发布提示词模板。
- 修改策略。
- 查看敏感信息。
- 导出数据。

## 权限矩阵

| 操作 | owner | admin | creator | reviewer | operator | viewer |
| --- | --- | --- | --- | --- | --- | --- |
| 创建小说 | 是 | 是 | 是 | 否 | 可选 | 否 |
| 生成/重写章节 | 是 | 是 | 是 | 可选 | 否 | 否 |
| 强制通过低分审稿 | 是 | 是 | 可选 | 是 | 否 | 否 |
| 清空后续章节 | 是 | 是 | 否 | 否 | 否 | 否 |
| 管理热点报告 | 是 | 是 | 可选 | 否 | 是 | 只读 |
| 创建视频项目 | 是 | 是 | 是 | 否 | 是 | 否 |
| 标记发布 | 是 | 是 | 否 | 否 | 是 | 否 |
| 查看数据复盘 | 是 | 是 | 可选 | 可选 | 是 | 只读 |
| 发布提示词模板 | 是 | 是 | 否 | 否 | 否 | 否 |
| 修改策略配置 | 是 | 是 | 否 | 否 | 否 | 否 |
| 管理用户 | 是 | 是 | 否 | 否 | 否 | 否 |
| 查看完整提示词 | 是 | 是 | 否 | 否 | 否 | 否 |

“可选”表示可由租户配置控制，默认不开放。

## 数据归属

核心表建议预留：

- `tenantId`
- `ownerId`
- `createdBy`
- `updatedBy`
- `deletedAt`

适用对象：

- 热点素材和报告。
- 小说项目。
- 小说方向、设定、大纲、章节、版本。
- 审稿报告。
- 生成任务。
- 视频项目和发布记录。
- 配置版本。
- 学习信号。
- 操作日志。

规则：

- 普通用户只能访问本租户数据。
- 默认列表不展示软删除数据。
- 归档不等于删除，归档数据仍保留复盘价值。
- 高风险导出需要权限和操作日志。

## 高风险操作

高风险操作必须同时满足：

- 用户有权限。
- 页面二次确认。
- 填写原因。
- 后端版本校验。
- 写入操作日志。

高风险操作包括：

- 删除小说。
- 清空后续章节。
- 强制通过低分全书审稿。
- 归档有视频引用的小说。
- 删除已生成或已发布视频。
- 发布提示词模板。
- 修改策略阈值。
- 降低内容安全或原创化严格程度。
- 停用模型供应商。
- 导出完整数据。

## 授权与套餐

如果系统未来打包售卖，需要授权能力。

建议授权对象：

- 授权码。
- 授权租户。
- 授权版本。
- 到期时间。
- 可用功能。
- 最大用户数。
- 最大小说数。
- 最大月任务数。
- 是否允许视频模块。
- 是否允许 AI 配置自定义。

授权状态：

| 状态 | 含义 | 系统行为 |
| --- | --- | --- |
| 正常 | 授权可用 | 正常使用 |
| 即将到期 | 临近到期 | 提示续费或更新授权 |
| 已到期 | 授权失效 | 限制新建和生成任务，保留查看和导出 |
| 已禁用 | 手动禁用 | 禁止登录或限制使用 |

本地单用户版可以不做在线校验，但数据结构保留授权状态。

## 登录和账号

完整产品可支持：

- 用户名密码登录。
- 邀请用户。
- 修改密码。
- 忘记密码。
- 登录日志。
- 会话过期。

当前本地版可以简化：

- 默认 owner 账号。
- 本地环境变量或初始化脚本创建账号。
- 暂不接入短信、邮箱和第三方登录。

## 页面设计

### 用户管理

字段：

- 用户名。
- 昵称。
- 角色。
- 状态。
- 最近登录时间。
- 创建时间。

操作：

- 新增用户。
- 修改角色。
- 禁用用户。
- 重置密码。

### 角色权限

展示：

- 角色列表。
- 模块权限。
- 动作权限。
- 高风险权限。

规则：

- 系统默认角色不能删除。
- 角色权限变更写操作日志。

### 租户设置

展示：

- 租户名称。
- 授权状态。
- 功能开关。
- 资源额度。
- 当前用户数。
- 数据统计。

### 操作日志

字段：

- 操作人。
- 操作对象。
- 操作类型。
- 操作原因。
- 操作前摘要。
- 操作后摘要。
- 是否高风险。
- 时间。

操作日志用于审计和复盘，不用于普通调试日志。

## 前端预留

即使权限后续研发，前端现在也应预留：

- 路由 `permissionCodes`。
- 按钮权限码。
- 菜单权限码。
- 高风险动作确认弹窗。
- 无权限提示页。
- 用户和租户上下文。

默认单用户模式下，权限判断可以全部通过，但结构不能缺失。

## 后端预留

后端需要预留：

- 请求上下文中的 `tenantId` 和 `userId`。
- service 层权限校验入口。
- repository 查询默认租户过滤。
- 操作日志服务。
- 高风险操作原因字段。
- 默认租户和默认 owner 初始化。

规则：

- route 层只解析身份和权限上下文，不写复杂权限逻辑。
- service 层判断业务动作是否允许。
- 数据访问层默认带租户过滤，避免漏查。

## 和其他模块的关系

- 小说系统：小说、章节、版本和审稿都归属租户和 owner。
- 热点系统：热点报告可以是系统默认，也可以是租户私有。
- 视频系统：发布记录和平台账号需要租户隔离。
- AI 配置：系统默认配置可全局共享，租户级配置可以覆盖。
- 自我成长：复盘数据默认租户内可见，跨租户汇总需要脱敏。

## 实施分期建议

1. 单用户默认租户和默认 owner，核心表预留归属字段。
2. 前端路由和按钮预留权限码，后端请求上下文预留用户和租户。
3. 增加用户管理、角色管理和操作日志。
4. 增加授权状态和功能开关。
5. 支持多租户数据隔离和租户级配置覆盖。

## 验收口径

- 当前单用户模式可以无感使用。
- 核心业务数据都能追溯租户、owner、创建人和更新人。
- 高风险操作有权限、二次确认、原因和操作日志。
- 前端菜单、路由和按钮预留权限码。
- 后续扩展多用户、多租户和售卖授权时不需要推翻核心数据结构。

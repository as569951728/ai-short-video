# AI 配置管理后台完整设计

本文档补齐 `GAP-P2-003`：系统如何管理模型供应商、模型配置、Agent 模型路由、任务模型映射、提示词模板、策略与程度配置、输出结构校验、测试发布和回滚。

AI 配置管理后台是高级/管理员能力，不进入小白创作主流程。小白用户默认只使用系统推荐配置；管理员可以在这里调整“哪个 Agent 用哪个大模型、哪个任务由哪个 Agent 执行、用哪个提示词、按什么审稿阈值、失败后怎么兜底”。

## 设计目标

- 让不同 AI 任务可以使用不同模型、提示词和策略。
- 让不同 Agent 可以绑定不同大模型能力，避免把所有生成、审稿、重写和视频任务压到同一个模型。
- 支持国内大模型优先，但不绑定单一供应商。
- 让提示词和策略可版本化、可测试、可发布、可回滚。
- 让审稿严格程度、内容安全、原创化、重写强度、市场导向、视频化倾向和自动处理程度不写死在代码里。
- 让每个生成任务和审稿报告都能追溯当时使用的模型、提示词和策略版本。
- 避免 API Key、完整提示词和完整模型响应泄露到普通页面和日志。

## 范围边界

AI 配置管理负责：

- 模型供应商配置。
- 模型能力和成本配置。
- Agent 能力配置和 Agent 模型路由。
- 任务到 Agent 的映射，以及 Agent 到模型策略的映射。
- 提示词模板和版本管理。
- 策略方案和程度项配置。
- 输出结构校验规则。
- 内容安全规则入口。
- 配置测试、发布、回滚和操作日志。
- 模型、提示词和策略效果复盘入口。

AI 配置管理不负责：

- 小白创作页面的交互。
- 直接修改已生成小说正文。
- 保存明文 API Key。
- 自动发布未经测试的生产配置。
- 用配置系统替代内容策划和审稿需求本身。
- 让普通用户每次生成都选择具体大模型。

## 配置对象

### 模型供应商

字段建议：

- 供应商名称。
- 供应商类型：OpenAI 兼容、本地模型、第三方 SDK、内部服务。
- API Base URL。
- API Key 环境变量名。
- 可用状态。
- 超时配置。
- 重试配置。
- 并发限制。
- 成本计算方式。
- 默认请求头配置。
- 安全说明。

规则：

- 页面只保存和展示 API Key 环境变量名，不展示明文 Key。
- 停用供应商属于高风险操作，需要提示影响哪些任务映射。
- 供应商配置变更后，需要重新测试相关模型。

### 模型配置

字段建议：

- 模型名称。
- 所属供应商。
- 模型类型：文本生成、长文、结构化输出、审稿、总结、TTS、视频生成。
- 上下文长度。
- 最大输出长度。
- 是否支持 JSON 模式或结构化输出。
- 推荐任务类型。
- 默认参数：温度、top_p、超时、最大重试。
- 成本单价。
- 质量标签：质量优先、速度优先、成本优先、稳定优先。
- 状态：启用、禁用、测试中、归档。

规则：

- 模型禁用前必须检查是否被任务映射使用。
- 模型成本变化不修改历史成本记录，只影响后续任务。

### Agent 能力配置

Agent 是任务执行角色，不等同于具体大模型。系统先根据任务选择 Agent，再由 Agent 的模型路由策略选择实际调用的大模型、提示词模板和兜底模型。

字段建议：

- Agent 编码，例如 `novel_planner_agent`、`novel_writer_agent`、`video_narration_agent`。
- Agent 名称。
- 所属模块：热点分析、小说、视频、运营复盘、系统自我成长。
- 职责说明。
- 能力标签：创意策划、长文写作、结构规划、审稿评分、影响评估、摘要记忆、旁白改写、字幕拆分、视频素材判断、运营复盘。
- 支持任务类型。
- 默认策略档位：系统推荐、质量优先、成本优先、速度优先、稳定优先。
- 默认提示词模板类型。
- 输出 schema。
- 是否需要复审。
- 是否允许换模型重试。
- 是否允许自动格式修复。
- 状态：测试中、启用、停用、归档。
- 当前 Agent 版本。
- 变更原因。

规则：

- Agent 职责发布后不能随意改语义，职责变化明显时需要新建 Agent 版本。
- 一个任务可以由一个主 Agent 执行，也可以由多个 Agent 串联，例如正文 Agent 生成、审稿 Agent 复审、格式修复 Agent 修结构。
- Agent 不保存 API Key，只引用模型路由配置。
- Agent 的输出不能绕过任务状态机、资产版本和用户确认规则。
- 普通用户不感知 Agent 和模型细节，只看到系统推荐动作、任务进度和结果质量。

### Agent 模型路由

Agent 模型路由决定“某个 Agent 在某个策略档位下实际调用哪个大模型”。研发阶段需要先内置默认路由，后续再开放后台调整和效果复盘。

字段建议：

- Agent 编码。
- Agent 版本。
- 适用任务类型。
- 策略档位：系统推荐、质量优先、成本优先、速度优先、稳定优先。
- 主模型。
- 备用模型。
- 失败重试模型。
- 质量复审模型。
- 格式修复模型。
- 默认参数：温度、top_p、最大输入 token、最大输出 token、超时。
- 上下文裁剪策略。
- 失败兜底策略。
- 成本上限。
- 并发限制。
- 是否允许用户高级覆盖。
- 状态：草稿、测试中、已发布、已停用。
- 路由版本。
- 测试样例和最近测试结果。
- 发布人、发布时间、变更原因。

路由规则：

- 生成任务运行时按 `taskType -> agent -> modelRoute -> modelConfig` 解析，不建议直接 `taskType -> model`。
- 同一个 Agent 在不同策略档位下可以使用不同模型，例如正文 Agent 的质量优先和成本优先可以走不同模型。
- 同一个任务可能记录多个 Agent 快照，例如正文生成任务需要记录正文 Agent 和审稿 Agent 的模型快照。
- 主模型不可用、输出格式错误、质量门禁失败时，按路由策略选择备用模型、格式修复模型或复审模型。
- 已发布路由不能原地修改，只能新建版本或回滚。
- 路由变更只影响后续任务，历史任务继续引用历史路由快照。

研发阶段默认 Agent 与模型建议：

| Agent | 主要职责 | 推荐模型能力 | 推荐模型族（研发默认） | 研发阶段默认建议 |
| --- | --- | --- | --- | --- |
| 热点分析 Agent | 总结热点、题材趋势、短视频爽点 | 信息抽取、摘要、市场归纳 | Qwen / 豆包，备用 DeepSeek | 优先选择稳定、低成本、结构化输出好的通用模型 |
| 小说策划 Agent | 方向、卖点、题材、爽点、人群判断 | 创意、市场理解、结构化输出 | Qwen / GLM，备用 DeepSeek | 质量优先，选择创意和中文表达较强的模型 |
| 小说结构 Agent | 设定、大纲、阶段大纲、章节目录 | 长结构规划、一致性、伏笔管理 | Qwen，备用 DeepSeek / GLM | 质量优先或稳定优先，选择长上下文和结构化能力强的模型 |
| 正文写作 Agent | 第 1 章试写、2-3 章试写、批量正文 | 长文本创作、文风稳定、情绪推进 | Qwen / 豆包，备用 GLM | 质量优先，正文批量可按成本优先提供备用模型 |
| 审稿评分 Agent | 方向审稿、章节审稿、全书审稿 | 批判、评分、问题诊断、风险识别 | DeepSeek / GLM，备用 Qwen | 与生成模型分开，优先选择推理和诊断能力强的模型 |
| 重写优化 Agent | 章节重写、局部补强、开篇优化 | 局部改写、风格保持、约束遵循 | Qwen / 豆包，备用 GLM | 稳定优先，避免自由发挥破坏上下文 |
| 长篇记忆 Agent | 摘要、角色状态、伏笔台账、长期记忆 | 信息抽取、结构化总结、事实一致性 | Qwen / DeepSeek，备用豆包 | 成本优先或稳定优先，要求结构化输出可靠 |
| 影响评估 Agent | 判断重写影响后续章节和视频引用 | 逻辑推理、一致性检查、风险分级 | DeepSeek / GLM，备用 Qwen | 稳定优先，必要时使用复审模型 |
| 视频策划 Agent | 判断适合视频化的章节段落和卖点 | 短视频理解、爽点提炼、节奏判断 | Qwen / 豆包 / GLM | 质量优先，先输出建议和引用范围，不直接生成视频 |
| 视频旁白 Agent | 将小说改写成短视频旁白 | 口语化、节奏、钩子、连续性 | 豆包 / Qwen，备用 GLM | 质量优先，后续接 TTS 前必须保存旁白版本 |
| 标题封面 Agent | 标题、封面文案、前三秒钩子 | 爆款标题、短文本 A/B、平台敏感度 | 豆包 / Qwen / GLM | 速度优先或质量优先，支持多候选 |
| 字幕拆分 Agent | 字幕分句、时长、可读性 | 文本切分、节奏控制、格式化 | Qwen / DeepSeek，备用豆包 | 稳定优先，要求格式严格 |
| 视频素材 Agent | 背景、素材、镜头、循环视频建议 | 视觉理解、素材匹配、风险判断 | 豆包 / GLM；涉及视觉时使用对应视觉模型 | 初期只做建议和规则判断，不接复杂生成模型 |
| 发布复盘 Agent | 24/48h 数据分析、下一步建议 | 数据归因、运营诊断、策略建议 | DeepSeek / Qwen，备用 GLM | 推理和结构化输出优先，后续接运营数据 |

具体模型版本不在需求文档里写死。研发阶段可以在 seed 配置里先用当前可用模型族落默认值，例如 Qwen、DeepSeek、豆包、GLM 或 OpenAI 兼容模型；实际接入时以供应商当前可用模型和成本为准。

### 当前模型接入阶段性建议（2026-06-18）

当前用户手上可用模型资源为 Kimicode、DeepSeek、GPT Pro。小说系统主链路已通过 mock / in-memory 产品验收，下一阶段需要先接真实大模型，但可以暂不实现完整模型配置后台。阶段性建议如下：

- 首个真实模型接入优先选 DeepSeek，作为小说主流程的默认模型族。
- GPT Pro / OpenAI 系列优先作为高价值复审、评分校准、提示词调试和关键质量诊断模型；只有具备 API Key / API 额度时才能接入系统后端，单纯 ChatGPT Pro 订阅不等于系统可调用 API。
- Kimicode 如果指代码/Agent 编程向能力，优先用于研发辅助，不作为小说正文主生成模型；如果实际有 Kimi 通用模型 API，可作为长上下文阅读、全书一致性检查和长篇记忆类任务的候选补充。
- 当前阶段先内置“固定路由 + 环境变量 API Key + 任务记录模型快照”，不先做可视化模型配置后台。

阶段性默认路由：

| Agent / 任务 | 当前默认模型建议 | 备用 / 复审建议 | 原因 |
| --- | --- | --- | --- |
| 小说策划、方向、设定、大纲、章节目录 | DeepSeek | GPT Pro / OpenAI API 可用时做少量复审 | 中文创作和结构化输出成本可控，适合先替换 mock 跑真实链路 |
| 第 1 章试写、2-3 章试写、批量正文 | DeepSeek | GPT Pro / OpenAI API 抽样复审；Kimi 通用 API 可作为长上下文候选 | 正文调用量最大，优先控制成本；质量通过评分与人工选择迭代 |
| 审稿评分、全书审稿、影响评估 | DeepSeek + GPT Pro / OpenAI 抽样校准 | Kimi 通用 API 用于长文一致性辅助 | 审稿最好和生成模型分离；现阶段可以先用 DeepSeek 跑全量，再用更强模型抽样校准 |
| 长篇记忆、章节摘要、伏笔台账 | DeepSeek | Kimi 通用 API | 需要稳定结构化和低成本；若 Kimi 长上下文可用，适合做长文读取 |
| 视频策划前置建议 | DeepSeek | GPT Pro / OpenAI API 抽样评估 | 视频模块还在需求设计期，先只做文本建议，不进入视频生产 |

首期不建议每次生成都让用户选模型。小白用户只看到“系统推荐配置”，研发在 seed / env 中固定默认路由；等真实生成质量、成本和失败模式跑出数据后，再进入 Agent 模型路由后台。

DeepSeek 首期接入任务包见：`docs/modules/model-integration-m1-deepseek-provider.md`。

### 任务模型映射

任务模型映射在新架构下主要决定每类任务由哪个 Agent 执行。具体大模型由 Agent 模型路由决定。

覆盖任务类型：

- 热点分析。
- 小说方向生成。
- 方向审稿。
- 方向融合/优化。
- 设定生成。
- 设定审稿。
- 全书大纲生成。
- 阶段大纲生成。
- 章节目录生成。
- 试写章节生成。
- 章节正文生成。
- 章节重写/优化。
- 章节审稿。
- 重写影响评估。
- 长篇记忆同步。
- 全书审稿。
- 视频化前检查。
- TTS。
- 字幕生成。
- 视频渲染或外部视频生成。

字段建议：

- 任务类型。
- 主 Agent。
- 辅助 Agent，例如复审 Agent、格式修复 Agent、记忆 Agent。
- 默认 Agent 路由策略档位。
- 可选高级覆盖范围。
- 兼容字段：主模型、备用模型、复审模型、格式修复模型。首期可以保留用于 seed 或旧实现过渡，但产品语义以 Agent 路由为准。
- 默认参数。
- 最大输入 token。
- 最大输出 token。
- 并发限制。
- 失败兜底策略。
- 适用策略方案。
- 状态。

失败兜底策略：

- 重试同模型。
- 降低上下文长度后重试。
- 切换备用模型。
- 切换同 Agent 的备用模型。
- 切换备用 Agent。
- 使用格式修复模型修复结构。
- 标记任务失败并提示用户。

## 提示词模板

提示词模板必须版本化，发布后不能原地修改。

模板类型：

- 热点报告生成。
- 创作机会点生成。
- 小说方向生成。
- 方向审稿。
- 设定生成。
- 大纲生成。
- 章节目录生成。
- 章节正文生成。
- 章节重写。
- 章节审稿。
- 影响评估。
- 试写总评。
- 全书审稿。
- 待视频化检查。
- TTS 旁白稿整理。
- 字幕拆分。

版本字段：

- 版本号。
- 状态：草稿、测试中、已发布、已停用、已归档。
- 角色定义。
- 任务目标。
- 输入变量说明。
- 上下文选择规则。
- 约束规则。
- 输出 schema。
- 质量规则。
- 风险规则。
- 示例输入。
- 示例输出。
- 变更原因。
- 创建人。
- 发布时间。

规则：

- 草稿可以编辑，已发布版本不能编辑，只能新建版本。
- 发布前必须通过至少一个结构化输出测试。
- 发布提示词属于高风险操作，需要填写变更原因。
- 普通小白页面不展示完整提示词。
- 任务日志只能记录提示词版本 ID 和输入摘要，不能记录完整提示词。

## 策略方案

策略方案承载不同创作偏好，例如：

- 标准创作策略。
- 严格质量策略。
- 快速试错策略。
- 短视频爽点策略。
- 成本优先策略。
- 高质量长篇策略。

策略版本字段：

- 审稿阈值。
- 分项权重。
- 阻塞规则。
- 低分推荐动作。
- 内容安全设置。
- 原创化设置。
- 重写强度设置。
- 市场导向设置。
- 视频化倾向设置。
- 自动处理设置。
- 适用任务范围。
- 变更原因。

规则：

- 策略版本发布后不能原地修改。
- 生成任务、审稿报告、影响评估、视频化检查都需要记录策略版本。
- 提高自动通过率、降低安全规则、降低原创化约束属于高风险变更。
- 策略回滚只影响后续任务，不修改历史报告结论。

## 程度项配置

程度项用于统一“宽松、标准、严格、极严格”等可选项。

建议程度项：

- AI 审稿严格程度。
- 内容安全严格程度。
- 原创化严格程度。
- 重写强度。
- 市场导向程度。
- 视频化倾向程度。
- 自动处理程度。
- 成本控制程度。
- 生成速度偏好。

规则：

- 程度项本身只提供可选等级和说明。
- 每个等级背后的阈值和动作由策略版本定义。
- 前端展示程度项时使用通俗说明，例如“更严格会减少低质量内容，但需要更多重写”。

## 输出结构校验

所有 AI 结构化产物都需要有输出结构校验。

对象：

- 热点报告。
- 小说方向卡。
- 设定档案。
- 大纲。
- 章节目录。
- 章节摘要/特性卡。
- 审稿报告。
- 影响评估。
- 视频化建议。

校验规则：

- 必填字段检查。
- 枚举值检查。
- 字数和数组长度检查。
- 分数范围检查。
- 高风险字段脱敏检查。
- 输出修复策略。

处理方式：

- 轻微格式错误：调用格式修复模型。
- 必填内容缺失：任务失败或重新生成。
- 安全风险输出：进入风险处理。
- 多次修复失败：任务失败并提示用户。

## 测试、发布和回滚

### 测试

测试对象：

- 模型连通性。
- 任务模型映射。
- 提示词模板版本。
- 策略版本。
- 输出 schema。

测试方式：

- 使用内置样例。
- 使用历史脱敏样例。
- 使用用户手动输入样例。
- 对比旧版本结果。

测试结果：

- 是否调用成功。
- 输出是否符合 schema。
- 审稿评分变化。
- 成本和耗时。
- 风险提醒。

### 发布

发布前检查：

- 是否通过测试。
- 是否影响主链路任务。
- 是否涉及安全和原创化规则。
- 是否有回滚版本。
- 是否填写变更原因。

发布后：

- 新任务使用新版本。
- 旧任务和历史报告继续引用旧版本。
- 写入操作日志。

### 回滚

回滚规则：

- 回滚到历史已发布版本。
- 回滚只影响后续任务。
- 回滚原因必须记录。
- 如果新版本已产生异常任务，应在复盘中关联这些任务。

## 权限和安全

- 只有管理员或 owner 可以发布配置。
- 普通创作者不能查看完整提示词和敏感配置。
- API Key 只通过环境变量引用。
- 完整模型输入输出不在普通页面展示。
- 配置发布、回滚、停用、降低安全策略都写操作日志。
- 导出配置时必须脱敏。

## 和其他模块的关系

- 生成任务：每个任务记录模型、提示词、策略和 schema 版本。
- Agent 模型路由：每个任务先记录 Agent、Agent 版本和路由版本，再记录实际模型快照。
- 审稿策略：审稿阈值和动作来自策略方案。
- 内容安全：安全规则可以作为策略版本的一部分发布。
- 系统自我成长：根据效果指标生成配置优化建议。
- 权限租户：不同租户可以使用系统默认配置，也可以有租户级覆盖配置。

## 实施分期建议

1. 先用种子数据内置默认供应商、模型、Agent、Agent 模型路由、任务映射、提示词和策略。
2. 管理后台只读展示当前配置，任务记录 Agent、路由、模型、提示词和策略版本。
3. 支持提示词草稿、测试、发布和回滚。
4. 支持 Agent 路由、策略方案和任务映射可配置。
5. 接入模型效果复盘、Agent 效果复盘和配置优化建议。

## 验收口径

- 每个 AI 任务都能追溯 Agent、Agent 版本、模型路由版本、实际模型、提示词模板版本、策略版本和输出 schema。
- 管理员能配置模型供应商、Agent 模型路由和任务到 Agent 的映射。
- 提示词和策略支持草稿、测试、发布、回滚。
- 发布和回滚高风险操作有原因和操作日志。
- API Key 和敏感配置不会出现在前端明文、普通日志和任务摘要中。

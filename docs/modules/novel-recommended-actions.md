# 小说列表推荐动作规则

本文档补齐 `GAP-P0-002 小说列表推荐动作规则`。

小说列表不是普通 CRUD 列表，而是小白用户管理小说项目的主入口。系统仍需要为每本小说计算推荐动作，但列表行的主要按钮统一收敛为“详情”，避免把生成、审稿、视频创建、归档等复杂操作都堆在列表里。

推荐动作不是人工维护状态，而是后端根据状态机、任务、审稿、章节和视频引用信息统一推导。

## 设计目标

- 每本小说仍计算一个当前最值得做的推荐动作。
- 小说列表首屏的主要按钮统一为“详情”。
- 推荐动作在列表中作为状态摘要、原因或详情页默认焦点，不一定直接渲染成主按钮。
- 前端不在页面里重复写状态判断，接口直接返回推荐动作对象。
- 复杂操作进入抽屉、弹窗、章节详情或小说详情，不让列表承载所有内容。
- 失败、阻塞、待确认场景优先于普通推进动作。
- 暂停、归档、删除等生命周期状态优先于创作流程状态。

## 推荐动作对象

列表接口建议返回 `recommendedAction`。

```ts
type NovelRecommendedAction = {
  code: string;
  label: string;
  intent: 'continue' | 'confirm' | 'review' | 'fix' | 'retry' | 'view' | 'restore' | 'create_video';
  priority: 'primary' | 'warning' | 'danger' | 'disabled';
  target: 'inline' | 'drawer' | 'modal' | 'page' | 'task' | 'external';
  targetRef?: string;
  taskType?: string;
  disabled?: boolean;
  disabledReason?: string;
  reasonText: string;
  confirmRequired?: boolean;
  confirmText?: string;
};
```

字段说明：

- `code`：动作编码，前端用于绑定按钮行为。
- `label`：按钮文案，必须是小白能理解的动词短语。
- `intent`：动作意图，决定按钮颜色和优先级。
- `priority`：视觉优先级。
- `target`：点击后打开哪里或触发什么。
- `targetRef`：抽屉类型、页面路由、任务 ID 或业务对象 ID。
- `taskType`：如果动作会创建 AI 任务，需要明确任务类型。
- `reasonText`：解释为什么现在推荐做这件事。
- `confirmRequired`：是否需要二次确认。

## 推导优先级

推荐动作按以下顺序推导，命中高优先级规则后不再继续向下判断。

1. 生命周期状态：暂停、归档、删除优先。
2. 当前主链路任务：处理中、待确认、失败优先。
3. 阻塞问题：章节待处理、审稿不通过、影响未处理优先。
4. 当前阶段待确认结果：方向、设定、大纲、章节目录、候选版本。
5. 当前阶段可开始任务：生成设定、生成大纲、试写、批量正文、全书审稿。
6. 视频准备状态：待视频化、已创建视频、视频引用异常。
7. 默认查看详情或打开小说工作台。

展示规则：

- 小说详情、章节详情和任务中心可以按推荐动作展示具体主按钮。
- 小说列表只展示推荐动作摘要，并以“详情”作为主要入口。
- 视频项目创建不作为小说列表行按钮，进入视频模块处理。

## 生命周期动作

| 条件 | 主动作 | 点击结果 | 说明 |
| --- | --- | --- | --- |
| `lifecycleStatus=paused` | 恢复小说 | 打开恢复确认弹窗 | 展示暂停原因和恢复后下一步 |
| `lifecycleStatus=archived` | 查看归档原因 | 打开归档信息抽屉 | 次操作提供恢复、复制 |
| `lifecycleStatus=deleted` | 无主动作 | 不在默认列表展示 | 仅管理员或回收站查看 |

生命周期状态会覆盖创作流程状态。暂停或归档小说不显示“生成设定”等创作动作。

## 当前任务动作

| 条件 | 主动作 | 点击结果 | 副作用 |
| --- | --- | --- | --- |
| 有主链路任务排队中 | 查看排队进度 | 打开任务抽屉 | 不允许重复触发同类任务 |
| 有主链路任务处理中 | 查看生成进度 | 打开任务抽屉 | 展示当前步骤、耗时、可取消条件 |
| 有任务待确认结果 | 确认生成结果 | 打开结果确认抽屉 | 用户采用后更新正式版本 |
| 主链路任务失败 | 查看失败原因 | 打开失败抽屉 | 提供重试、换模型、调整输入 |

任务动作优先于阶段动作。例如小说处于 `setting` 阶段，但设定生成任务正在处理中，主按钮显示“查看生成进度”，不是“生成小说设定”。

## 阻塞动作

| 条件 | 主动作 | 点击结果 | 说明 |
| --- | --- | --- | --- |
| 最新审稿不通过 | 查看审稿问题 | 打开审稿问题抽屉 | 展示问题卡片和优化入口 |
| 试写低分 | 优化试写章节 | 进入章节详情或打开优化抽屉 | 先处理开篇质量 |
| 存在待处理章节 | 处理待处理章节 | 进入章节详情列表或筛选视图 | 小说不能完成前必须处理 |
| 章节重写影响后文 | 查看影响范围 | 打开影响评估抽屉 | 让用户选择自动处理、逐章处理或清空后续 |
| 全书审稿不通过或已过期 | 优化全书问题 | 打开全书审稿报告 | 可返回章节、大纲或重新审稿 |
| 视频引用异常 | 查看引用异常 | 打开视频引用提示 | 不自动覆盖已生成视频 |

阻塞动作的按钮优先级为 `warning`。如果阻塞会导致无法完成或无法视频化，必须展示原因。

## 次要操作规则

次要操作放入更多菜单或行展开区，不抢主按钮。

常见次要操作：

- 编辑基础信息。
- 详情。
- 查看最近任务。
- 查看审稿报告。
- 查看版本记录。
- 复制项目。
- 暂停、归档、删除进入详情页处理，不在列表首屏直接暴露。

高风险次要操作：

- 归档小说：必须填写原因。
- 删除小说：必须二次确认。
- 清空后续章节：必须展示影响范围。
- 低分强制继续：必须填写原因。
- 修改已被视频引用章节：必须提示影响视频。

## 点击后的承载位置

| 动作类型 | 承载位置 | 说明 |
| --- | --- | --- |
| 简单确认 | 弹窗 | 例如确认完成、恢复小说 |
| 结果选择 | 抽屉 | 例如选择方向、确认设定、确认大纲 |
| 问题处理 | 抽屉或详情页 | 轻问题用抽屉，复杂章节问题进章节详情 |
| 长任务进度 | 任务抽屉 | 展示当前步骤、失败原因、重试入口 |
| 深度编辑 | 小说详情工作台 | 大纲、章节目录、复盘等复杂编辑 |
| 单章处理 | 章节详情工作台 | 正文、审稿问题、重写、候选版本、版本对比、影响评估 |
| 视频创建 | 视频模块 | 小说模块只提供待视频化和引用状态 |

## 推荐动作变更时机

以下事件发生后必须重新计算推荐动作：

- 小说核心状态变化。
- AI 任务创建、完成、失败、取消。
- 用户采用或放弃候选结果。
- 审稿报告生成。
- 章节生成、重写、审稿或影响处理完成。
- 用户强制继续或确认风险。
- 小说暂停、恢复、归档、删除。
- 视频项目创建。
- 已引用章节发生影响性修改。

## 空动作和禁用动作

有些场景不应该给用户可点击主按钮：

- 用户无权限操作。
- 小说已删除。
- 任务状态未知或数据异常。
- 当前状态缺少必要产物，且系统无法自动补齐。
- 系统配置缺失，例如模型不可用、策略不存在。

此时接口仍返回 `recommendedAction`，但 `disabled=true`，并提供 `disabledReason`，页面展示“无法继续”和原因。

## 和状态机的关系

本文档基于 `docs/modules/novel-state-machine.md`，并引用以下 P0 主链路规则：

- `docs/modules/novel-state-gate-action-contract.md`：推荐动作总矩阵、禁用原因和重算触发事件。

- `docs/modules/novel-trial-writing-loop.md`：试写低分、试写待确认、试写通过后的主动作。
- `docs/modules/chapter-rewrite-impact-handling.md`：章节重写影响后文、中等/严重影响未处理时的主动作。
- `docs/modules/novel-full-review-gate.md`：全书审稿未通过、审稿过期、完成确认时的主动作。
- `docs/modules/novel-video-readiness.md`：待视频化、已创建视频和视频引用异常时的主动作。

状态机回答“现在处于什么状态、允许怎么流转”。推荐动作规则回答“在列表上应该让用户点什么”。当列表主动作无法在抽屉里完成时，由小说详情工作台或章节详情工作台承接。

# 全书 AI 审稿完成门禁

本文档补齐 `GAP-P0-011`：正文全部生成后，系统如何进行全书 AI 审稿，如何根据评分、问题、用户确认和风险处理决定小说是否可以进入完成确认与待视频化。

全书审稿不是单章审稿的简单汇总。它要判断整本小说是否可用、是否适合视频化、是否存在长篇结构问题，以及是否值得继续投入视频生产。

全书审稿需要引用 `docs/modules/novel-hit-content-planning.md` 中的题材模板、开篇黄金段、爽点/痛点节奏、伏笔回收、长篇故事圣经和短视频旁白单元标准，避免只判断结构完整而忽略市场感和追更感。具体审稿维度、门禁和小白展示按 `docs/modules/novel-hit-content-integration-matrix.md` 的全书审稿落点执行。

## 设计目标

- 在小说进入视频化前做最后质量门禁。
- 发现单章审稿看不出的全书级问题。
- 把“能不能完成”从主观感觉变成可追溯的评分、问题和用户决策。
- 支持低分强制继续，但必须记录风险和原因。
- 为后续视频系统提供小说质量、分段建议和风险提示。

## 前置条件

全书审稿前必须满足：

- 小说处于 `body` 阶段或已完成正文生成。
- 计划章节全部有当前正式正文。
- 所有章节都有当前章节摘要/特性卡片。
- 不存在 `pending` 或 `processing` 章节。
- 不存在未处理的中等或严重影响案例。
- 不存在阻塞性的候选版本待确认。
- 当前没有冲突的正文生成、章节重写、影响评估或长篇记忆同步任务。

如果存在未审稿章节，系统可以根据策略先补章节审稿，再发起全书审稿；但不能跳过章节基础数据。

## 审稿输入

全书审稿应读取结构化摘要为主，正文全文按策略分批读取。

输入包括：

- 小说方向和设定档案。
- 全书大纲、阶段大纲、章节目录。
- 所有章节摘要/特性卡片。
- 章节正文摘要和必要正文片段。
- 章节审稿汇总。
- 长篇记忆、人物状态、伏笔状态。
- 影响评估历史。
- 试写总评和用户确认记录。
- 视频化倾向和目标平台风险规则。

长篇小说不建议一次把全部正文塞进一个模型请求。可以采用分段审稿、阶段汇总、全书汇总的多任务方式。

## 审稿维度

全书审稿必须输出以下维度：

| 维度 | 说明 |
| --- | --- |
| 全书完成度 | 是否完整、有始有终、没有明显缺章 |
| 开篇吸引力 | 前几章是否足够抓人 |
| 主线清晰度 | 主角目标、阻力、升级路线是否稳定 |
| 长篇连贯性 | 人物、关系、事实和伏笔是否前后一致 |
| 爽点密度 | 是否持续有爽点和情绪回报 |
| 爽点衰减 | 中后段是否重复、变水或情绪回报下降 |
| 节奏控制 | 是否拖沓、重复、断裂或高潮不足 |
| 人设稳定性 | 主角、反派、配角是否前后矛盾 |
| 结尾完成度 | 结局是否收束主要矛盾和伏笔 |
| AI 味风险 | 是否有明显模板化、废话、机械重复 |
| 原创化风险 | 是否过度像已有作品或热点素材 |
| 平台内容风险 | 是否存在敏感或不适合发布的内容 |
| 视频化适配 | 是否适合拆成短视频集和旁白 |
| 首条验证潜力 | 是否能提炼出首条短视频钩子和推荐切片 |

## 审稿输出

全书审稿报告必须包含：

- 总分。
- 评级。
- 分项评分。
- 核心优点。
- 主要问题。
- 阻塞问题数量。
- 建议优化动作。
- 是否允许进入完成确认。
- 是否建议进入视频化。
- 建议视频拆分方式。
- 平台风险提示。
- 低分继续风险。

报告保存为 `ReviewReport`，`reviewLevel=full_novel`。

## 完成门禁结果

全书审稿后，系统生成 `completionGateResult`：

| 结果 | 含义 | 推荐动作 |
| --- | --- | --- |
| `pass` | 达到策略门槛，可确认完成 | 确认小说完成 |
| `warning` | 可完成但有风险 | 查看风险后确认完成或继续优化 |
| `blocked` | 不建议完成，默认阻塞视频化 | 优化全书问题 |
| `forced_pass` | 用户低分强制通过 | 进入完成确认但记录风险 |

标准策略默认使用 `docs/modules/review-strictness-policy.md` 定义的 80/70/60 档位：

- 80 分及以上：`pass`。
- 70-79 分：`warning`。
- 60-69 分：`blocked`，允许填写原因强制继续。
- 60 分以下：强阻塞，默认建议返回章节或大纲调试。

85/75/65 可作为严格策略示例，不作为默认标准策略。最终阈值以后台策略版本为准。

## 问题处理

全书问题必须转成小白可执行的问题卡片。

常见问题类型：

- 开篇不够吸引。
- 中段重复或拖沓。
- 主角目标变弱。
- 人物关系前后不一致。
- 伏笔未回收。
- 反派压迫感不足。
- 爽点密度下降。
- 结尾仓促。
- AI 味明显。
- 视频旁白不顺。
- 内容或原创化风险。

每个问题卡片包含：

- 问题标题。
- 通俗解释。
- 严重程度。
- 影响范围。
- 推荐处理对象：章节、阶段、大纲、设定或全书策略。
- 推荐动作。
- 是否阻塞完成。

可执行动作：

- 定位到问题章节。
- 批量生成章节优化建议。
- 逐章进入章节详情处理。
- 返回大纲调整阶段节奏。
- 返回设定修正人物或世界规则。
- 标记问题可接受并填写原因。
- 重新发起全书审稿。

## 中段阶段审稿

长篇小说不应等全部正文完成后才第一次检查中后段质量。正文生成过程中建议每 10-15 章或每个阶段结束时生成一次阶段审稿。

阶段审稿重点：

- 爽点是否衰减。
- 冲突和打脸方式是否重复。
- 主角目标和行动力是否变弱。
- 人物关系是否偏离设定。
- 伏笔是否丢失或推进过慢。
- AI 味和解释性内容是否增加。
- 是否仍适合后续口播切片。

规则：

- 阶段审稿默认不替代全书审稿。
- 阶段审稿发现严重问题时，可以阻塞继续批量生成下一阶段。
- 阶段审稿问题需要转成章节或阶段问题卡。
- 用户接受风险继续时必须填写原因。

## 用户确认完成

全书审稿通过或用户强制继续后，进入完成确认。

确认弹窗展示：

- 全书评分和评级。
- 是否建议视频化。
- 阻塞问题是否已清零。
- 仍存在的风险。
- 章节数量、总字数、目标语音时长预估。
- 视频化建议范围。
- 用户确认后的结果：小说将进入待视频化，视频系统可引用。

低分、警告或存在未解决建议时，用户必须选择或填写原因：

- 我接受这些问题，先进入视频化测试。
- 我计划后续视频剪辑阶段再优化。
- 我认为当前内容已经可用。
- 其他原因。

确认完成后，系统写入完成决策记录和操作日志。

## 不允许完成的场景

以下情况不能通过强制继续绕过：

- 有进行中的主链路任务。
- 有 `pending` 或 `processing` 章节。
- 有未处理的中等或严重影响案例。
- 计划章节缺正文。
- 缺少章节摘要/特性卡片。
- 全书审稿报告不存在或已过期。
- 当前全书审稿基于旧版本，正文或大纲已被修改。
- 存在必须处理的内容安全高风险。

这些属于数据或安全门禁，不是单纯质量分低。

## 审稿过期

全书审稿完成后，如果发生以下变化，报告标记为过期：

- 任一章节正式正文被修改。
- 章节摘要/特性卡片发生关键变化。
- 设定、大纲或章节目录正式版本变化。
- 影响案例重新打开。
- 内容安全策略升级并要求复审。

过期报告不能用于完成确认。页面应提示“内容已变化，需要重新全书审稿”。

## 页面交互

小说详情工作台的“审稿报告区”和顶部概览栏承载全书审稿门禁。

建议展示：

- 全书评分卡。
- 完成门禁结果。
- 是否建议视频化。
- 阻塞问题列表。
- 分项评分雷达或表格。
- 按阶段/章节的问题分布。
- 推荐动作主按钮。
- 强制继续原因入口。
- 重新审稿入口。

列表页只展示摘要和主推荐动作，不展示完整报告。

## 数据与接口边界

建议对象：

- `ReviewReport`：保存全书审稿报告。
- `FullReviewGate`：保存完成门禁结果和审稿报告版本关系。
- `CompletionDecisionRecord`：保存用户确认完成、强制继续或返回优化的决策。

建议接口：

- `POST /novels/:novelId/full-review`：发起全书审稿。
- `GET /novels/:novelId/full-review/latest`：获取最新全书审稿报告。
- `POST /novels/:novelId/full-review/:reviewId/resolve-issue`：记录问题处理动作。
- `POST /novels/:novelId/full-review/:reviewId/force-pass`：低分强制继续并填写原因。
- `POST /novels/:novelId/confirm-completion`：确认小说完成。
- `POST /novels/:novelId/full-review/recheck-staleness`：检查审稿是否过期。

全书审稿是 AI 任务，接口返回任务 ID；确认完成是业务动作，需要版本校验和幂等控制。

## 和其他模块的关系

- 状态机：全书审稿对应 `creationStage=full_review`，完成确认对应 `creationStage=completion_confirm`。
- 推荐动作：全书审稿失败、过期、待确认完成都会影响列表主按钮。
- 章节重写影响处理：存在未处理影响时不能确认完成。
- AI 产物确认：全书审稿报告自动保存，但完成确认需要用户决策。
- 待视频化判定：只有完成门禁通过或强制通过后，才可以进入待视频化检查。
- 爆款内容策划体系：全书审稿需要引用开篇黄金段、爽点衰减、伏笔回收、故事圣经和旁白单元标准。

# 小说详情工作台信息架构

本文档补齐 `GAP-P0-003 小说详情工作台信息架构`。

小说详情工作台是单本小说的完整创作现场。小说列表负责告诉用户“下一步点什么”，小说详情工作台负责让用户看清“这本小说为什么处于这个状态、有哪些产物、哪里有问题、怎么调试、后续会影响什么”。

2026-06-19 交互方向已确认：小说详情默认不再把所有创作模块堆在一个长页面里，而是采用“小说概览页 + 大步骤详情页 + 子步骤条”的两层工作台。概览页负责全局状态、质量风险、最近任务和下一步建议；每个大步骤进入独立详情页处理该环节的草稿、候选、编辑、AI 优化、采用确认和异常。

P0 默认体验规则见 `docs/modules/novel-p0-visible-main-flow.md`。小说列表是小白用户的默认驾驶舱，小说详情工作台是完整能力承载层和复杂问题维修间；详情页默认仍应先展示概览和当前推荐动作，而不是把所有复杂区块一次性铺开。

页面承载边界见 `docs/modules/novel-surface-boundaries.md`。小说详情工作台只承接项目级复杂问题，列表、抽屉或弹窗能完成的轻量确认不应强行进入详情页。

版本、审稿策略、失败异常、归档恢复、视频引用、成本提醒和内容安全的完整规则分别见 `docs/modules/novel-version-management.md`、`docs/modules/review-strictness-policy.md`、`docs/modules/failure-exception-handling.md`、`docs/modules/archive-pause-restore.md`、`docs/modules/video-reference-exceptions.md`、`docs/modules/cost-resource-reminders.md` 和 `docs/modules/content-safety-platform-risk.md`。详情工作台只展示与当前小说相关的摘要和入口，复杂明细默认折叠。

## 设计目标

- 承载单本小说从方向到待视频化的完整创作资产。
- 始终展示当前状态、当前推荐动作、阻塞原因和下一步。
- 支持小白用户按推荐动作推进，也支持高级用户查看完整细节。
- 不做普通 CRUD 详情页，而是创作、审稿、调试、任务和复盘的工作台。
- 列表可以完成轻量推进；复杂编辑、版本对比、项目级复盘进入详情工作台。
- 默认不要让小白用户直接面对完整版本树、完整审稿维度、任务事件日志、模型成本和项目复盘。
- 默认详情页必须像“创作概览”，而不是“所有模块展开的长工作台”。
- 每个大步骤必须有独立处理页；大步骤内部如果存在多个确认点，需要用子步骤条或同等强提示承接，不能只用普通 tab 让用户自己猜是否需要确认。

## 页面入口

| 入口 | 进入后默认定位 |
| --- | --- |
| 小说列表点击小说名称 | 概览区 |
| 小说列表点击“详情” | 概览区 |
| 列表推荐动作需要深度处理 | 定位到对应工作区，例如大纲、章节、审稿 |
| 章节详情返回 | 定位到章节管理区，并保留章节筛选 |
| 任务详情返回 | 定位到任务记录区，并高亮任务 |
| 审稿报告入口 | 定位到审稿报告区 |

进入详情时可以携带 `focus` 参数，例如 `focus=outline`、`focus=chapters`、`focus=reviews`。

## 页面整体结构

推荐采用两层页面结构：

1. 小说概览页：展示书名、生命周期状态、质量评分、风险、最近任务、视频引用状态、当前推荐动作和全局大步骤条。
2. 步骤详情页：每个大步骤一个独立处理页，承载该步骤的子步骤条、草稿/候选/正式版本、编辑/优化/采用确认、任务进度、失败处理和影响提示。

概览页是小白默认入口，不能铺满全部创作模块。复杂信息可以作为摘要卡、任务抽屉或高级入口存在。步骤详情页才展开该环节的具体操作。

如果页面宽度不足，大步骤条和子步骤条可以横向滚动；辅助信息区可以变成抽屉或折叠区，但当前状态、下一步和主动作必须保持可见。

## 顶部概览栏

顶部概览栏必须一直让用户看清当前项目状态。

内容：

- 返回小说列表。
- 小说名称。
- 题材/频道。
- 热点来源。
- 生命周期状态：进行中、暂停、归档。
- 创作阶段和阶段状态。
- 当前 step。
- 章节进度。
- 市场潜力评分。
- 当前质量评分。
- 视频引用状态：未引用、可被引用、已引用、引用异常。
- 当前推荐动作主按钮。
- 更多操作：复制、暂停、归档、删除、查看操作日志。

顶部主按钮规则：

- 与小说列表的 `recommendedAction` 保持一致。
- 如果用户从列表点击某个推荐动作进入详情，顶部主按钮仍显示同一个动作。
- 如果详情页内状态发生变化，顶部主按钮需要刷新。
- 暂停、归档、删除等生命周期状态优先覆盖创作动作。

## 创作流程导航

创作流程导航用于展示完整流程，不用于让用户随意跳过前置步骤。

概览页建议使用 8 个用户可理解的大步骤，避免把工程任务拆得过细：

1. 方向确认。
2. 小说设定。
3. 大纲设计。
4. 章节目录。
5. 试写调试。
6. 批量正文。
7. 全书质检。
8. 待视频化。

每个 step 需要显示：

- 未开始、处理中、待确认、阻塞、失败、已完成。
- 该 step 的核心产物是否存在。
- 如果失败或阻塞，显示简短原因。
- 点击已完成 step 可以查看产物摘要。
- 点击未完成 step 只能查看说明，不能跳过流程。
- 点击可处理 step 进入对应步骤详情页，而不是在概览页展开完整模块。

## 步骤详情页与子步骤条

每个大步骤详情页都需要用子步骤条展示本环节有哪些内容要处理和确认。子步骤条不是普通 tab，它需要表达顺序、完成状态、当前处理点和门禁。

建议子步骤：

| 大步骤 | 子步骤 |
| --- | --- |
| 方向确认 | 草稿输入、候选生成、编辑优化、采用确认 |
| 小说设定 | 人物设定、世界规则、爽点禁忌、采用确认 |
| 大纲设计 | 全书主线、阶段大纲、节奏检查、采用确认 |
| 章节目录 | 章节表、单章摘要、批量调整、采用目录 |
| 试写调试 | 第 1 章候选、2-3 章试写、试写总评、策略快照 |
| 批量正文 | 批量生成、章节问题、重写影响、批次确认 |
| 全书质检 | 总评、Top 问题、处理建议、完成确认 |
| 待视频化 | 检查清单、视频建议、引用快照、去视频列表 |

子步骤交互要求：

- 当前子步骤需要明确说明“这一步要确认什么”。
- 未完成前置子步骤时，可以预览后续说明，但不能执行跳过式确认。
- 采用、完成、进入下一环节等动作需要二次确认，并说明影响范围。
- 手动编辑、AI 优化、候选采用不能静默覆盖正式资产，必须落到草稿、候选或新版本。
- 任务进度、失败原因、重试入口应在步骤详情页或任务抽屉中可见。

## 主工作区分区

主工作区不再采用“一个页面锚点导航 + 全模块分区全部铺开”的方式。新的承载方式是：概览页只展示全局摘要和大步骤入口；步骤详情页只展开当前大步骤的具体操作。

默认可见顺序仍然先服务主流程：概览、当前推荐动作、当前阶段产物和阻塞问题。版本、任务、视频关联、项目复盘等区块可以保留，但默认不抢主线。

### 1. 概览区

目标：让用户快速知道这本小说现在值不值得继续、卡在哪里、下一步做什么。

内容：

- 当前状态摘要。
- 当前推荐动作和推荐原因。
- 阻塞问题摘要。
- 最近任务。
- 最新审稿结论。
- 质量分变化。
- 章节进度。
- 视频引用状态。

主操作：

- 执行当前推荐动作。
- 查看阻塞问题。
- 打开最近任务。
- 查看最新审稿报告。

### 2. 方向方案区

目标：管理小说方向和方向版本。

内容：

- 当前已选方向。
- 初始生成的 3-5 个方向。
- AI 推荐方向理由。
- 方向评分和风险。
- 融合记录。
- 优化记录。
- 方向审稿报告。

主操作：

- 选择方向。
- 融合方向。
- 优化方向。
- 重新生成方向。
- 查看方向版本。

规则：

- 已确认方向后，普通用户不建议随意重选。
- 如果重选方向，需要提示会影响设定、大纲、章节和正文。
- 重选方向必须生成影响评估。

### 3. 小说设定区

目标：承载后续大纲和正文生成的基础设定。

完整规则见 `docs/modules/novel-setting-profile.md`。本区默认服务小白确认，高级结构字段按需展开。

内容：

- 设定摘要。
- 主角设定。
- 反派/阻力。
- 配角关系。
- 世界背景。
- 主线目标。
- 核心爽点。
- 雷区与禁用元素。
- 长篇记忆基础信息。
- 视频化倾向。

主操作：

- 生成设定。
- 确认设定。
- 优化设定。
- 编辑高级设定。
- 查看设定版本。

规则：

- 设定未确认前不能生成全书大纲。
- 设定确认后如果修改关键事实，需要影响评估。
- 小白模式默认只展示摘要、风险和主操作。
- 设定生成结果必须作为候选版本，用户确认后才成为当前正式设定。

### 4. 大纲管理区

目标：管理全书结构、阶段结构和章节目录。

完整规则见 `docs/modules/novel-outline-structure.md`。大纲管理区按全书大纲、阶段大纲、章节目录三层组织。

内容：

- 全书一句话主线。
- 开局、阶段目标、中期反转、终局高潮、结局方向。
- 阶段/分卷大纲。
- 阶段数量和章节范围。
- 章节目录。
- 每章目标、冲突、爽点、人物、钩子和伏笔。
- 大纲审稿问题。

主操作：

- 生成全书大纲。
- 确认全书大纲。
- 调整阶段数量。
- 生成章节目录。
- 确认章节目录。
- 局部重写阶段。
- 查看大纲版本。

规则：

- 大纲未确认前不能生成章节目录。
- 章节目录未确认前不能试写前三章。
- 调整阶段数量后，需要重新计算章节范围和冲突升级。
- 全书大纲、阶段大纲和章节目录都必须作为候选版本，用户确认后才成为当前正式结构。

### 5. 试写与开篇调试区

目标：先用前 1-3 章验证小说方向、文风、节奏和爽点。

完整规则见 `docs/modules/novel-trial-writing-loop.md`。试写区不是普通章节预览，而是批量正文生成前的质量验证和策略校准入口。

内容：

- 试写章节列表。
- 每章摘要和评分。
- 开篇吸引力评分。
- 爽点密度。
- AI 味风险。
- 试写审稿问题。
- 优化记录。

主操作：

- 试写前三章。
- AI 审稿。
- 优化试写章节。
- 确认试写通过。
- 放弃当前方向或返回大纲调整。

规则：

- 试写未通过，不建议批量生成正文。
- 低分强制继续必须填写原因。
- 试写优化后需要重新审稿。
- 试写总评必须给出通过、建议优化、需要修改或返回上游的结论。
- 试写通过后仍需要用户确认，才能进入批量正文生成。
- 试写优化产生的正文必须先作为候选版本，不能直接覆盖正式正文。

### 6. 章节管理区

目标：管理正文生成、章节状态、章节问题和章节入口。

内容：

- 章节列表。
- 章节序号、标题、所属阶段。
- 字数和目标字数。
- 主状态：正常、待处理、处理中、已处理。
- 状态备注。
- 章节评分。
- 正文状态。
- 审稿状态。
- 视频引用状态。
- 最近更新时间。
- 推荐操作。

主操作：

- 批量生成正文。
- 查看正文生成进度。
- 处理待处理章节。
- 打开章节详情工作台。
- 全书正文完成后触发全书审稿。

规则：

- 批量生成正文必须在章节管理区发起，用户能看见生成范围、起始章节和影响范围。
- 正文生成进度在最近任务、任务抽屉或生成任务区查看，不在小说列表行按钮中单独展开。
- “正文批量生成正在进行”这类提示必须带操作入口：查看任务进度、进入章节管理或查看最近任务。
- 单章正文、审稿问题、候选版本、重写和影响评估进入章节详情工作台。
- 章节重写影响处理完整规则见 `docs/modules/chapter-rewrite-impact-handling.md`。
- 只要存在待处理或处理中章节，小说不能进入完成确认。
- 中等或严重影响未关闭前，小说不能进入完成确认或待视频化。
- 已被视频引用章节修改时，必须提示影响。

### 7. 审稿报告区

目标：集中查看方向、章节和全书审稿结果。

内容：

- 方向审稿。
- 试写审稿。
- 章节审稿汇总。
- 全书审稿。
- 分项评分。
- 核心优点。
- 主要问题。
- 修改建议。
- 是否建议进入视频化。
- 用户强制继续记录。

主操作：

- 查看最新审稿报告。
- 按问题类型筛选。
- 执行审稿推荐动作。
- 重新审稿。
- 强制继续并填写原因。

规则：

- 审稿问题需要转成小白可执行的问题卡片。
- 全书审稿完成门禁完整规则见 `docs/modules/novel-full-review-gate.md`。
- 全书审稿未完成、审稿过期或门禁阻塞时，小说不能进入待视频化。
- 全书审稿不通过，默认不能进入待视频化；允许强制继续的场景必须填写原因，且不能绕过数据完整性和内容安全强阻塞。

### 8. 版本与调试区

目标：记录 AI 和用户对核心创作资产做过什么改动。

内容：

- 方向版本。
- 设定版本。
- 大纲版本。
- 章节目录版本。
- 章节正文版本。
- 候选版本。
- 评分变化。
- 修改原因。
- 使用模型和策略版本。

主操作：

- 查看版本详情。
- 对比版本。
- 恢复版本。
- 放弃候选版本。
- 采用候选版本。

规则：

- 重要创作资产不能静默覆盖。
- AI 重写默认生成候选版本，用户采用后才成为正式版本。
- 恢复旧版本需要做影响评估。

### 9. 生成任务区

目标：让用户和研发都能追踪 AI 长任务。

内容：

- 当前任务。
- 历史任务。
- 任务类型。
- 状态：排队中、处理中、待确认、完成、失败、取消。
- 任务进度。
- 失败原因。
- 重试入口。
- 关联业务对象。

主操作：

- 查看任务详情。
- 重试任务。
- 取消任务。
- 采用任务结果。
- 查看失败原因。

规则：

- 同一对象同一时间不能有冲突任务。
- 任务失败不能只显示失败，要提供下一步建议。
- 任务结果不能绕过用户确认直接覆盖正式资产。

### 10. 视频关联区

目标：展示小说和下游视频的引用关系。

内容：

- 是否待视频化。
- 已创建视频项目。
- 视频引用章节范围。
- 视频生成状态。
- 发布状态。
- 引用异常。
- 小说修改对视频的影响提示。

主操作：

- 查看视频项目。
- 查看引用异常。
- 标记需要重新生成视频。
- 跳转视频列表创建视频项目。

规则：

- 待视频化判定完整规则见 `docs/modules/novel-video-readiness.md`。
- 小说详情只展示待视频化检查、视频引用状态和跳转入口。
- 创建视频项目属于视频模块能力；视频模块创建时需要保存章节范围和章节版本快照。
- 已发布视频不被小说修改直接覆盖。
- 章节修改影响视频时，需要标记引用异常。

### 11. 项目复盘区

目标：为系统自我成长和后续优化沉淀经验。

内容：

- 项目关键时间线。
- 归档原因。
- 失败原因。
- 质量分变化。
- 用户采用/放弃记录。
- 哪些优化动作有效。
- 是否值得复制新项目。
- 视频反馈摘要。

主操作：

- 填写归档原因。
- 复制为新项目。
- 标记成功案例。
- 标记失败案例。

规则：

- 归档不是删除，归档后仍要保留复盘信息。
- 成功案例只能作为结构和经验参考，不能直接复制正文。

## 辅助信息区

辅助信息区用于减少小白用户迷茫。

建议展示：

- 当前推荐动作解释。
- 阻塞原因。
- 最近任务。
- 最新审稿问题。
- 风险提醒。
- 相关快捷入口。

辅助区不承担复杂编辑，复杂编辑进入对应主工作区或详情页。

## 小白模式与高级模式

小白模式默认展示：

- 当前状态。
- 当前推荐动作。
- 摘要和风险。
- 少量核心问题。
- 一键生成、一键优化、确认按钮。

高级模式展开：

- 完整设定字段。
- 完整大纲结构。
- 版本对比。
- 提示词和模型摘要。
- 任务日志。
- 策略版本。
- 成本和 token 摘要。

默认不展示 API Key、完整提示词和完整模型响应。

## 数据加载边界

小说详情工作台不应一次加载所有正文和大 JSON。

建议加载策略：

- 首屏加载小说概览、状态、推荐动作、评分摘要、章节统计、最近任务。
- 每个分区展开时按需加载详情。
- 章节正文只在章节详情工作台或章节预览中加载。
- 审稿报告默认加载摘要，完整报告点击后加载。
- 版本对比按需加载。

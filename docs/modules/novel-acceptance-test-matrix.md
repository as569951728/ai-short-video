# 小说系统验收测试矩阵

本文档补齐 `REVIEW-P1-006`：为 P0-P2 小说创作闭环建立跨模块验收矩阵，避免需求文档看起来完整，但进入原型、后端或测试时缺少统一的验收口径。

本文档只定义需求验收范围，不拆研发排期。

## 验收目标

- 验证小白用户能看懂当前状态和下一步动作。
- 验证状态、门禁、推荐动作在列表、详情、章节和任务入口一致。
- 验证 AI 任务不会重复创建、不会把旧上下文写入新资产。
- 验证候选采用、章节重写和高风险操作都有副作用处理。
- 验证视频引用、运营回填和 P2 数据口径不会破坏小说主链路。

研发首期验收需要同时对照 `docs/modules/novel-development-contract.md`、`docs/modules/novel-core-data-contract.md`、`docs/modules/novel-api-contract.md` 和 `docs/modules/novel-first-iteration-development-plan.md`。测试会话应按任务包逐包验收，不等全部研发完成后才集中测试。

## 首期任务包验收入口

| 任务包 | 必测主链路 | 必测异常链路 | 是否允许进入下一包 |
| --- | --- | --- | --- |
| 0 工程地基 | 前后端启动、健康检查、统一错误、共享枚举 | 后端不可用、错误响应结构、枚举缺失 | 基础能力全部通过 |
| 1 列表与草稿 | 创建草稿、列表展示、详情摘要 | 空列表、创建失败、暂停/归档不可操作 | 状态摘要稳定 |
| 2 方向候选 | 生成方向、候选展示、采用方向 | 重复点击、低分采用、候选过期 | 采用后能进入设定 |
| 3 结构资产 | 生成/采用设定、大纲、章节目录 | 上游未确认、下游过期、已有正文影响 | 能生成章节计划 |
| 4 任务中心 | 任务进度、失败、重试、取消 | 冲突任务、上游版本变化、参数幂等冲突 | 任务不再只靠 loading |
| 5 试写章节 | 试写生成、审稿、确认通过 | 低分阻塞、强制通过、缺章节目录 | 能进入正文阶段 |
| 6 批量正文 | 批量生成、章节重写、采用候选、影响评估 | 部分失败、影响未关闭、视频引用章节修改 | 全书审稿门禁可靠 |
| 7 全书与视频化 | 全书审稿、确认完成、待视频化快照 | 审稿过期、视频化检查失败、低分强制完成 | 可生成稳定快照 |
| 8 视频引用 | 创建视频项目、引用快照、引用异常 | 非 video_ready 引用、章节修改、已发布不覆盖 | 可进入自用验证 |
 
任务包验收结论必须给出：

- 通过用例。
- 失败用例。
- 是否阻塞下一任务包。
- 需要需求主控确认的问题。

## 状态、门禁、推荐动作矩阵

| 场景 | 前置数据 | 期望展示 | 期望动作 | 验收重点 |
| --- | --- | --- | --- | --- |
| 新建小说 | 无方向候选；无任务 | 待生成方向 | 生成小说方向 | 列表每行只突出一个主动作 |
| 有方向候选 | 方向候选未采用 | 待选择方向 | 选择小说方向 | 不展示内部任务状态为主状态 |
| 设定缺失 | 方向已确认；无设定 | 待生成设定 | 生成小说设定 | 禁用大纲生成并解释原因 |
| 试写低分 | 试写总评 60-69；无强风险 | 试写需优化 | 优化试写章节 | Top 3 问题优先展示 |
| 章节待处理 | 有章节低分或影响未关闭 | 章节待处理 | 处理待处理章节 | 阻塞全书审稿 |
| 全书审稿过期 | 正文已变化；旧全书审稿仍存在 | 全书审稿过期 | 重新全书审稿 | 不允许确认完成 |
| 待确认完成 | 全书审稿有效；完成门禁通过 | 待确认完成 | 确认小说完成 | 不直接进入待视频化 |
| 已确认完成但视频检查失败 | 完成决策已写；视频化检查失败 | 暂不可视频化 | 查看待视频化检查 | 小说完成状态不被回滚 |
| 待视频化 | 完成确认有效；视频化检查通过 | 待视频化 / 可被视频引用 | 去视频列表 | 小说列表不直接创建视频项目 |
| 视频引用异常 | 已有视频引用；章节当前版本变化 | 引用异常 | 查看引用异常 | 小说创作状态和视频异常分开解释 |

## AI 任务生命周期矩阵

| 场景 | 操作 | 期望结果 | 验收重点 |
| --- | --- | --- | --- |
| 重复点击生成 | 连续点击同一生成按钮 | 只返回同一任务或已有结果 | `idempotencyToken + requestHash` 生效 |
| 参数变化但幂等键相同 | 同一幂等键提交不同参数 | 返回参数冲突错误 | 不创建第二个任务 |
| 冲突任务存在 | 同章节已有正文重写中 | 新重写任务被拒绝 | 返回冲突任务摘要 |
| 上游版本变化 | 任务处理中设定被采用新版本 | 旧任务取消或结果标记过期 | 写入前校验 `sourceVersionRefs` |
| 批量正文部分失败 | 第 13 章失败，前 12 章成功 | 前 12 章保留，第 13 章显示失败 | 后续依赖章节默认暂停 |
| 子任务重试 | 单章生成失败后重试 | 创建新任务并关联原任务 | 原失败历史保留 |
| 用户取消批量任务 | 批量正文处理中取消 | 已完成子任务保留，未开始取消 | 页面重新计算推荐动作 |
| 任务成功待确认 | AI 生成候选完成 | 展示候选摘要和采用/放弃 | 不自动覆盖当前版本 |

## 高风险操作矩阵

| 操作 | 必须展示影响范围 | 必须二次确认 | 必须填写原因 | 必须写日志 |
| --- | --- | --- | --- | --- |
| 低分候选仍采用 | 是 | 是 | 是 | 是 |
| 采用过期候选 | 是 | 是 | 是 | 是 |
| 修改关键设定 | 是 | 是 | 是 | 是 |
| 恢复历史版本 | 是 | 是 | 是 | 是 |
| 清空后续章节 | 是 | 是 | 是 | 是 |
| 手动确认中等影响无问题 | 是 | 是 | 是 | 是 |
| 强制通过试写或全书审稿 | 是 | 是 | 是 | 是 |
| 修改已被视频引用章节 | 是 | 是 | 是 | 是 |
| 忽略视频引用异常 | 是 | 是 | 是 | 是 |

验收重点：

- 确认弹窗文案必须说明“会影响什么”和“下一步要做什么”。
- 原因要落到决策记录或操作日志，不能只停留在前端。
- 高风险动作完成后必须重新计算状态摘要。

## 视频引用异常矩阵

| 场景 | 期望状态 | 期望动作 | 验收重点 |
| --- | --- | --- | --- |
| 引用章节轻微润色 | 轻微引用风险 | 查看差异或确认无影响 | 不自动重生成视频 |
| 引用章节关键事实变化 | 强引用异常 | 重新生成视频或创建新版本 | 已发布视频保持冻结 |
| 视频项目使用旧快照 | 引用快照过期 | 重新检查引用 | 页面能解释旧快照来源 |
| 视频已发布后小说修改 | 已发布引用异常 | 查看发布记录和风险 | 不覆盖平台记录 |
| 只改未引用章节 | 无引用异常 | 继续小说创作 | 视频列表不制造误报 |

## P2 指标和样本口径矩阵

| 模块 | 最小可验收数据 | 完整产品数据 | 验收重点 |
| --- | --- | --- | --- |
| 热点分析 | 热点标题、题材、来源、人工判断、推荐方向 | 多平台趋势、样本拆解、机会点评分 | 可作为小说方向输入，但不强依赖外部服务 |
| 内容样例库 | 正向/反向样例标签、适用题材、判断标准 | 样例评分、审稿校准结果、模板版本 | AI 审稿不能只给抽象分 |
| 首条测试 | 标题、前 3 秒钩子、首屏字幕、测试结果 | 多平台 A/B、完播率、评论关键词 | 试写后可选，不改变小说完成状态 |
| 发布运营 | 平台、账号、链接、发布时间、24/48 小时数据、下一步决策 | API 同步、趋势分析、自动复盘 | 早期支持人工回填 |
| 自我成长 | 用户决策、审稿分、重写原因、发布表现 | 策略推荐、提示词调优、模型对比 | 数据先沉淀，智能优化后置 |
| AI 配置 | 策略版本、阈值、模型供应商和用途 | 配置后台、灰度、成本统计 | 核心流程读取策略，不写死阈值 |

## 小白体验验收清单

- 页面默认不展示完整状态机和内部字段。
- 每个关键页面都有“当前在哪一步”和“下一步建议”。
- 审稿默认只展示总分、评级、一句话结论、Top 3 问题和推荐动作。
- 高级选项默认折叠，但可以给后续扩展留入口。
- 任务不能只显示按钮 loading，必须有步骤、进度、失败原因和重试入口。
- 失败时告诉用户“还能不能继续”和“下一步点什么”。

# 视频引用异常

本文档补齐 `GAP-P1-006`：小说进入待视频化或已创建视频项目后，如果小说正文、章节摘要、审稿结果或章节范围发生变化，系统如何识别视频引用异常、提示用户、保护已生成或已发布视频。

本文档只定义需求规则，不讨论具体实现方案。

## 设计目标

- 视频项目引用小说时有稳定内容快照。
- 小说修改不会静默污染已生成视频。
- 用户能知道哪些视频可能受影响。
- 已发布视频不会被系统自动覆盖。
- 初期简单视频方案也能有引用保护。

## 引用对象

视频项目引用小说时，需要形成引用摘要：

- 小说 ID。
- 引用章节范围。
- 引用章节版本。
- 引用时的全书审稿报告。
- 引用时的视频化检查摘要。
- 引用创建时间。

视频项目后续展示时，应能说明“这个视频基于小说哪一段内容生成”。

## 引用状态

| 状态 | 含义 | 用户动作 |
| --- | --- | --- |
| `normal` | 引用内容与小说当前版本一致或无风险 | 可继续生成视频 |
| `info` | 小说有轻微变化，但不影响视频理解 | 可查看差异 |
| `warning` | 引用章节内容变化，建议同步视频素材 | 重新生成配音/字幕或确认忽略 |
| `blocking` | 引用正文失效或风险升高 | 暂停继续生成视频 |
| `resolved` | 异常已处理 | 可继续 |

小白默认只看到：

- 视频引用正常。
- 小说已修改，建议重新生成视频。
- 引用内容风险较高，请先处理。

## 异常触发场景

以下变化会触发视频引用异常：

- 被引用章节正式正文变化。
- 被引用章节摘要/特性卡片关键变化。
- 被引用章节被清空或重新生成。
- 被引用章节状态变为待处理。
- 全书审稿报告过期。
- 待视频化快照过期。
- 内容安全规则判定风险升高。
- 小说从待视频化回退到正文处理或全书质量检查。

以下变化通常只提示信息，不强阻塞：

- 未引用章节被修改。
- 被引用章节只做错别字或标点修正。
- 视频项目尚未生成任何音频、字幕或视频文件。

## 异常等级判断

| 变化类型 | 默认等级 |
| --- | --- |
| 未引用章节修改 | 无异常或 `info` |
| 引用章节轻微润色 | `info` |
| 引用章节正文内容变化 | `warning` |
| 引用章节被清空 | `blocking` |
| 引用章节进入待处理 | `blocking` |
| 全书审稿过期 | `warning` |
| 内容安全强风险 | `blocking` |
| 已发布视频引用内容变化 | `warning`，并提示已发布内容不会自动修改 |

## 处理动作

用户可选动作：

- 查看差异。
- 重新生成配音。
- 重新生成字幕。
- 重新渲染视频。
- 确认忽略本次变化。
- 返回小说处理章节问题。
- 标记视频项目停止继续。

规则：

- 未发布视频可以提示重新生成素材。
- 已发布视频不能自动覆盖，只能记录平台内容可能与当前小说不一致。
- 确认忽略必须填写原因。
- 阻塞级异常未处理时，禁止继续生成、渲染、确认当前视频和导出；仅填写原因不能解除 blocking，必须处理到 `normal` 或 `resolved` 后才能继续。

处理动作矩阵：

| 视频状态 | 异常等级 | 推荐动作 |
| --- | --- | --- |
| 未生成产物 | `info` | 查看差异或确认忽略 |
| 未生成产物 | `warning` | 重新检测、返回小说处理或确认继续 |
| 未生成产物 | `blocking` | 暂停创建后续产物，先处理小说或重新选择引用范围 |
| 已生成未发布 | `warning` | 从受影响产物开始重新生成，不覆盖旧产物 |
| 已生成未发布 | `blocking` | 暂停生成，处理异常后再继续 |
| 已发布 | `info` / `warning` | 保留已发布记录，建议创建新视频版本或新视频条目 |
| 已发布 | `blocking` | 停止基于旧引用继续生成，推荐复制新视频项目或新视频条目 |

已发布视频的异常处理重点是“提示和追溯”，不是自动修改平台内容。旧发布记录、旧视频文件版本、旧发布数据必须保留。

## 页面承载

小说列表：

- 展示视频引用异常摘要。
- 主推荐动作可变为“查看引用异常”。

小说详情视频关联区：

- 展示受影响视频项目。
- 展示受影响章节。
- 展示异常等级和推荐动作。

视频列表：

- 展示引用状态。
- 支持只看引用异常视频。

视频详情：

- 展示引用快照。
- 展示异常原因和处理动作。

## 需求验收标准

- 创建视频项目时能说明引用了哪些章节。
- 修改被引用章节后，视频项目能标记异常。
- 已发布视频不会被小说修改自动覆盖。
- 引用异常有等级、原因和推荐动作。
- 用户确认忽略异常必须记录原因。

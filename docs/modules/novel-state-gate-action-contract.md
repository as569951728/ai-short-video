# 状态、门禁、推荐动作总矩阵

本文档补齐 `GAP-P1-009`：统一小说系统的核心状态、派生状态、门禁判断和推荐动作推导，避免后端、前端、测试在“当前状态是什么、下一步按钮是什么、为什么不能继续”上出现口径漂移。

本文档只定义需求级契约，不讨论具体代码实现。

## 核心结论

小说系统只有三个核心保存状态：

| 字段 | 含义 | 是否作为事实源 |
| --- | --- | --- |
| `lifecycleStatus` | 小说是否仍在创作池中 | 是 |
| `creationStage` | 小说主链路推进到哪一步 | 是 |
| `stageStatus` | 当前阶段的处理状态 | 是 |

其他状态全部通过数据推导，不能作为新的事实源：

| 状态 | 推导来源 | 是否允许缓存 |
| --- | --- | --- |
| 页面展示状态 | 核心状态、任务、门禁、章节、视频引用 | 可以缓存，但必须可重算 |
| 推荐动作 | 核心状态、任务、门禁、章节、视频引用 | 可以返回快照，但不能作为业务判断事实源 |
| 质量门禁 | 审稿报告、策略版本、用户决策记录 | 不建议缓存为独立状态 |
| 章节汇总 | 章节正文、章节状态、审稿、影响案例 | 不建议缓存为独立状态 |
| 视频准备 | 完成确认、全书审稿、视频化快照、视频引用 | 可以保存快照，但快照必须有版本关系 |

## 字段口径统一

架构草案中出现过 `creationStatus`、`currentStep`、`completedSteps`、`currentRecommendedAction`、`archiveStatus`、`currentStage` 等字段。后续统一口径如下：

| 旧字段或草案字段 | 统一口径 |
| --- | --- |
| `creationStatus` | 不作为事实源，拆为 `creationStage` + `stageStatus` |
| `currentStage` | 不单独使用，统一为 `creationStage` |
| `archiveStatus` | 不单独使用，统一为 `lifecycleStatus=archived` |
| `currentStep` | 进度展示字段，可由 `creationStage` 推导或缓存 |
| `completedSteps` | 进度展示字段，可由已确认产物和阶段记录推导 |
| `currentRecommendedAction` | 推荐动作快照，可作为接口返回或短期缓存，不作为业务判断事实源 |
| `videoReferenceStatus` | 视频引用派生状态，由视频引用快照和异常记录推导 |

如果后续实现中为了列表性能缓存展示状态或推荐动作，必须满足：

- 缓存内容可由事实数据重新计算。
- 缓存需要记录计算时间和计算依据摘要。
- 任一触发重算事件发生后，缓存必须刷新或标记过期。
- 后端执行关键业务动作时不能只信缓存，必须重新校验事实源。

## 展示状态推导优先级

页面展示状态按以下优先级推导。命中高优先级状态后，不再展示低优先级状态为主状态。

| 优先级 | 判断域 | 示例展示状态 | 说明 |
| --- | --- | --- | --- |
| 1 | 生命周期 | 已删除、已归档、已暂停 | 生命周期覆盖所有创作动作 |
| 2 | 数据异常 | 数据异常、无法继续 | 必要产物缺失且无法自动补齐 |
| 3 | 主链路任务 | 生成中、审稿中、待确认结果、生成失败 | 任务优先于阶段 |
| 4 | 强阻塞门禁 | 章节待处理、全书审稿过期、内容高风险 | 阻塞完成或待视频化 |
| 5 | 当前阶段待确认 | 待选择方向、待确认设定、待确认大纲 | 用户需要确认 AI 产物 |
| 6 | 当前阶段可推进 | 待生成设定、待试写、待全书审稿 | 可以执行下一步任务 |
| 7 | 视频承接 | 待视频化、已创建视频、引用异常 | 小说主链路已完成后展示 |
| 8 | 默认查看 | 查看详情 | 无明确推进动作时展示 |

## 门禁总矩阵

门禁用于判断某个动作是否可以执行。门禁失败时，推荐动作必须指向最优先的阻塞来源。

| 目标动作 | 必须满足 | 常见阻塞原因 | 阻塞后推荐动作 |
| --- | --- | --- | --- |
| 生成小说方向 | 小说未暂停、未归档、未删除；无方向冲突任务；热点或默认题材可用 | 配置缺失、已有方向任务 | 查看任务进度或使用默认题材 |
| 确认方向 | 有方向候选；风险已展示；低分继续需原因 | 无候选、候选过期、原创化风险高 | 重新生成方向或优化方向 |
| 生成设定 | 方向已确认；无设定冲突任务 | 方向未确认、方向候选过期 | 选择小说方向 |
| 确认设定 | 有设定候选；必要字段完整；审稿风险已展示 | 设定缺字段、设定审稿阻塞 | 优化设定 |
| 生成全书大纲 | 设定当前版本已确认；无大纲冲突任务 | 设定未确认、设定已变化 | 确认或重新审查设定 |
| 确认全书大纲 | 有大纲候选；大纲审查不强阻塞 | 大纲支撑不了目标章节数 | 优化大纲 |
| 生成阶段大纲 | 全书大纲当前版本已确认 | 大纲未确认、阶段数量异常 | 确认大纲或调整阶段数量 |
| 确认阶段大纲 | 有阶段大纲候选；阶段范围合理 | 阶段太少或太多、阶段目标重复 | 调整阶段结构 |
| 生成章节目录 | 阶段大纲当前版本已确认 | 阶段大纲未确认 | 确认阶段大纲 |
| 确认章节目录 | 每章有目标、冲突、爽点、钩子和关键事实 | 章节重复、水剧情、前 3 章弱 | 优化章节目录 |
| 试写前三章 | 章节目录当前版本已确认；无试写冲突任务 | 章节目录未确认、任务冲突 | 确认章节目录或查看任务 |
| 确认试写通过 | 试写章节有正文、摘要和审稿；试写总评允许通过或用户确认风险 | 开篇低分、试写审稿缺失 | 优化试写章节 |
| 批量生成正文 | 试写已通过；章节目录当前有效；无正文生成冲突任务 | 试写未通过、上游版本变化 | 处理试写问题 |
| 进入全书审稿 | 所有计划章节有正文和摘要；无待处理章节；无未关闭影响案例 | 缺正文、章节待处理、影响未关闭 | 处理章节问题 |
| 确认小说完成 | 全书审稿有效；完成门禁允许；强风险已处理 | 全书审稿过期、低分阻塞、内容高风险 | 重新审稿或优化全书 |
| 确认待视频化 | 完成确认已通过；视频化检查通过；可生成引用快照 | 视频化检查失败、章节引用异常 | 查看待视频化检查 |
| 创建视频项目 | 小说可视频化；引用范围有效；无阻塞引用异常 | 小说未待视频化、引用范围无效 | 进入视频模块创建 |

## 推荐动作推导优先级

推荐动作必须由后端统一推导，前端只负责展示和触发对应动作。

| 优先级 | 命中条件 | 推荐动作类型 | 示例按钮 |
| --- | --- | --- | --- |
| 1 | `lifecycleStatus=paused` | 恢复 | 恢复小说 |
| 2 | `lifecycleStatus=archived` | 查看/恢复 | 查看归档原因 |
| 3 | 有主链路任务 `queued/processing` | 查看任务 | 查看生成进度 |
| 4 | 有主链路任务 `waiting_confirmation` | 确认结果 | 确认生成结果 |
| 5 | 有影响当前阶段的失败任务 | 失败处理 | 查看失败原因 |
| 6 | 有强阻塞门禁 | 修复 | 处理待处理章节 |
| 7 | 有待确认候选产物 | 确认 | 确认小说设定 |
| 8 | 当前阶段可开始任务 | 继续 | 生成全书大纲 |
| 9 | 小说可视频化 | 视频承接 | 去视频列表 |
| 10 | 默认 | 查看 | 查看详情 |

推荐动作对象字段沿用 `docs/modules/novel-recommended-actions.md`，但需要补充以下约束：

- `disabled=true` 时必须给出 `disabledReason`。
- `reasonText` 必须说明为什么推荐该动作。
- `target` 必须符合 `docs/modules/novel-surface-boundaries.md` 的承载边界。
- 会创建任务的动作必须声明 `taskType`。
- 高风险动作必须声明 `confirmRequired=true`。

## 禁用原因矩阵

当动作不可执行时，不能只隐藏按钮。系统需要返回禁用原因，让用户知道下一步该处理什么。

| 禁用原因 | 说明 | 用户可见文案方向 |
| --- | --- | --- |
| `lifecycle_not_active` | 小说已暂停、归档或删除 | 先恢复小说 |
| `conflict_task_exists` | 已有冲突任务 | 等当前任务完成或取消 |
| `required_asset_missing` | 缺少上游产物 | 先完成上一阶段 |
| `required_asset_stale` | 上游产物或候选已过期 | 基于最新内容重新生成 |
| `review_required` | 缺少必要审稿 | 先进行 AI 审稿 |
| `review_blocked` | 审稿结果阻塞 | 先处理审稿问题 |
| `chapter_pending` | 存在待处理章节 | 先处理章节 |
| `impact_unresolved` | 存在未关闭影响案例 | 先处理影响范围 |
| `content_risk_blocking` | 内容安全强风险 | 先处理风险内容 |
| `video_reference_blocking` | 视频引用强异常 | 先处理引用异常 |
| `permission_denied` | 无权限 | 联系管理员或切换账号 |
| `config_missing` | 模型、策略或模板不可用 | 检查系统配置 |
| `data_inconsistent` | 数据关系异常 | 进入详情查看异常 |

## 重算触发事件

以下事件发生后，必须重新计算展示状态和推荐动作：

- 小说核心状态变化。
- 生命周期变化：暂停、恢复、归档、删除、恢复归档。
- 任务创建、开始、完成、失败、取消、等待确认。
- 用户采用、放弃、继续优化、强制继续候选产物。
- 任一关键创作资产当前版本变化。
- 任一关键候选或审稿报告被标记过期。
- 章节正文、章节摘要、章节审稿、章节主状态变化。
- 影响评估创建、决策、处理、关闭。
- 全书审稿报告生成或过期。
- 完成确认记录生成。
- 视频化快照生成或过期。
- 视频项目创建。
- 视频引用异常生成、升级、忽略或关闭。
- 内容安全策略升级并影响当前小说。

## 阶段展示矩阵

以下矩阵用于验收后端返回的展示状态和主推荐动作。

| 核心条件 | 展示状态 | 主推荐动作 |
| --- | --- | --- |
| `lifecycleStatus=paused` | 已暂停 | 恢复小说 |
| `lifecycleStatus=archived` | 已归档 | 查看归档原因 |
| `creationStage=draft` 且无方向任务 | 待生成方向 | 生成小说方向 |
| `creationStage=direction` 且有方向候选 | 待选择方向 | 选择小说方向 |
| `creationStage=setting` + `not_started` | 待生成设定 | 生成小说设定 |
| `creationStage=setting` + `waiting_user` | 待确认设定 | 确认小说设定 |
| `creationStage=outline` + `not_started` | 待生成大纲 | 生成全书大纲 |
| `creationStage=chapter_plan` + `waiting_user` | 待确认章节目录 | 确认章节目录 |
| `creationStage=trial` + `processing` | 试写中 | 查看试写进度 |
| `creationStage=trial` + `blocked` | 试写需优化 | 优化试写章节 |
| `creationStage=body` 且有正文生成任务 | 正文生成中 | 查看正文进度 |
| `creationStage=body` 且存在待处理章节 | 章节待处理 | 处理待处理章节 |
| `creationStage=body` 且所有章节完成 | 待全书审稿 | 全书 AI 审稿 |
| `creationStage=full_review` + `blocked` | 全书需优化 | 优化全书问题 |
| `creationStage=completion_confirm` + `waiting_user` | 待确认完成 | 确认小说完成 |
| `creationStage=video_ready` 且无视频项目 | 待视频化 | 去视频列表 |
| 已创建视频项目且引用正常 | 已创建视频 | 查看视频项目 |
| 存在视频引用异常 | 引用异常 | 查看引用异常 |

说明：

- 小说列表行主按钮优先展示“详情”，不直接展示“创建视频项目”。
- `creationStage=video_ready` 表示小说已可被视频模块引用，不表示小说模块负责创建视频项目。
- 视频引用状态是派生状态，作为列表列、筛选项和详情摘要展示，不能和小说创作状态混成一个字段。

## 强阻塞门禁

以下门禁不能通过“接受风险继续”绕过：

- 小说已删除。
- 小说已归档且未恢复。
- 缺少当前正式方向、设定、大纲或章节目录。
- 计划章节缺正文。
- 章节缺当前摘要/特性卡片。
- 存在 `pending` 或 `processing` 章节。
- 存在未关闭的中等或严重影响案例。
- 全书审稿报告不存在或已过期。
- 当前全书审稿基于旧版本。
- 内容安全强风险。
- 创建视频项目时引用范围无效。
- 任务执行中上游版本已变化且任务结果基于旧版本。

质量低分、市场潜力低、轻微 AI 味、轻微视频化风险可以允许用户接受风险继续，但必须记录原因。

## 统一状态摘要服务

列表、小说详情、章节详情、任务进度和视频承接入口必须使用同一套状态摘要服务，例如 `calculateNovelStatusSummary(novelId)` 或等价业务方法。

统一状态摘要服务负责：

- 读取核心状态、任务、门禁、章节、影响案例、审稿报告、视频化快照和视频引用异常。
- 按本文档的优先级计算 `displayStatus`、`blockingReasons[]` 和 `recommendedAction`。
- 返回用户可读的下一步说明，而不是只返回机器状态。
- 标记计算依据、计算时间和计算版本，便于排查缓存和状态漂移。

调用要求：

- 小说列表、小说详情、章节详情中展示的主状态必须来自同一摘要结果。
- 高风险操作完成后必须重新计算状态摘要。
- 任务状态变化、候选采用、版本过期、审稿完成、视频引用异常变化后必须触发重算。
- 业务操作接口提交时必须重新执行门禁校验，不能只相信前端带回来的状态摘要。
- 如果为了性能使用缓存，缓存只能用于展示，不能作为业务动作放行依据。

## 接口返回契约

小说列表和小说详情接口建议返回同一套状态摘要：

```text
statusSummary
  lifecycleStatus
  creationStage
  stageStatus
  displayStatus
  displayStatusText
  currentStep
  completedSteps
  blockingReasons[]
  recommendedAction
  calculatedAt
  calculationVersion
```

规则：

- `blockingReasons[]` 可以为空，但不能省略。
- `recommendedAction` 即使禁用也要返回，便于页面告诉用户原因。
- `calculatedAt` 和 `calculationVersion` 用于解释推荐动作快照，不代表业务事实源。
- 关键操作接口提交时必须重新校验状态和门禁，不能只信列表接口返回的推荐动作。

## 验收口径

- 同一小说在列表、详情、章节工作台看到的主状态一致。
- 有冲突任务时，主按钮优先展示任务进度。
- 有强阻塞门禁时，不能展示普通推进按钮。
- 推荐动作能说明原因、承载位置和禁用原因。
- 暂停、归档状态下不展示生成类主按钮。
- 采用候选、任务完成、审稿过期、视频引用异常后推荐动作会刷新。
- 业务操作接口会重新校验门禁，不能依赖旧页面状态继续执行。
